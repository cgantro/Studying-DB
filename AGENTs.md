## 현재 운영 방향성
### 1. 지식과 답변 분리
- `CS공부/`는 지식 자체를 깊게 정리한다.
- 면접 답변은 `QnA/`에서 관리한다.
- 기술 개념을 바로 답변형 문장으로 만들지 않는다.

### 2. 프로젝트 중심 연결
- 모든 면접 답변은 가능하면 프로젝트 근거와 연결한다.
- 프로젝트 경험은 `문제 -> 원인 -> 해결 -> 결과 -> 배운 점` 흐름으로 정리한다.
- 기술 선택은 반드시 대안과 트레이드오프를 함께 포함한다.

### 3. 기업 맞춤 준비
- 공통 답변은 `QnA/`에 둔다.
- 회사 맞춤 답변은 `기업/<회사명>/`에 별도로 작성한다.
- 현재 우선 기업은 `기업/삼성SDS`다.
- 회사명을 바꿔도 통하는 범용 지원동기는 작성하지 않는다.

### 4. 상태/사실 기준 작성
- 프로젝트 상태는 `진행중`, `미완성`, `완료` 중 하나로 명시한다.
- 코드 근거 없는 내용은 `추정` 또는 `확인 필요`로 표시한다.
- 팀 전체 성과와 개인 기여를 구분한다.
- 성능 수치는 측정 조건과 함께 작성한다.

### 5. 네이밍/구조 원칙
- 한글 파일명과 한글 디렉토리를 우선 사용한다.
- 기존 폴더 체계는 유지한다.
- 새 문서를 만들기 전에 기존 문서와 중복되는지 확인한다.
- 중복 문서가 있으면 새로 만들기보다 기존 문서를 증분 수정한다.

## 작성 원칙
- 추상 표현보다 실제 경험, 구조, 수치, 판단 기준을 우선한다.
- 면접에서 바로 말할 수 있는 문장으로 정리한다.
- 긴 문장보다 짧고 명확한 문장을 사용한다.
- 기술 설명은 `개념 -> 왜 필요한가 -> 동작 원리 -> 장단점 -> 프로젝트 연결` 순서로 정리한다.
- 면접 답변은 `질문 의도 -> 핵심 답변 -> 근거 경험 -> 결과/교훈 -> 꼬리질문` 순서로 정리한다.
- 인성 답변도 주장만 쓰지 말고 실제 행동과 판단 기준을 포함한다.

## 작업 절차
1. 관련 폴더와 기존 문서를 먼저 확인한다.
2. 프로젝트 관련 내용은 `프로젝트/` 문서를 먼저 정리한다.
3. 이후 `QnA/` 또는 `기업/` 문서에서 면접 답변으로 재가공한다.
4. 회사 맞춤 답변은 공통 답변을 그대로 복사하지 말고 회사 요구사항에 맞게 재구성한다.
5. 마지막에 필요한 경우 아래 항목을 남긴다.
- `확인 필요`
- `추가 근거 필요`
- `면접 꼬리질문 후보`

## 폴더별 역할
### `CS공부/`
- CS, 언어, 프레임워크, DB, 네트워크, OS 지식을 정리한다.
- 면접 답변이 아니라 지식 원본을 작성한다.
- 개념의 장점뿐 아니라 한계, 대안, 사용 조건도 함께 정리한다.

### `QnA/`
- 기술 질문, 인성 질문, 경험 기반 질문을 관리한다.
- 실제 면접에서 말할 수 있는 답변 형태로 작성한다.
- 답변은 가능한 한 프로젝트 경험과 연결한다.

### `프로젝트/`
- 프로젝트 경험의 원본 DB다.
- 역할, 구조, 기술 선택, 문제 해결, 성능 개선, 협업 경험을 정리한다.
- 모든 면접 답변의 근거가 되는 공간이다.

### `기업/`
- 기업별 분석과 맞춤 면접 준비 문서를 관리한다.
- 기업의 사업, 제품, 직무 요구사항과 내 경험을 연결한다.
- 현재 우선순위는 `삼성SDS`다.

### `내 노션/정리본/`
- 과거 정리본과 원본 레퍼런스 공간이다.
- 직접 대규모 수정하지 않는다.
- 필요한 내용을 다른 폴더로 재구성해서 옮긴다.

3. 측정 방법(로그/툴/쿼리/프로파일러)
