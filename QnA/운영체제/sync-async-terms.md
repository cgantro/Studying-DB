# 동기적, 비동기적, 동기, 비동기, 동기화 Q&A

## 1. 트랩은 동기적이고 인터럽트는 비동기적이라는 뜻

트랩과 인터럽트에서 말하는 동기적/비동기적은 **사건이 현재 실행 중인 명령 흐름과 어떤 관계로 발생하는지**를 말한다.

```text
동기적 = 현재 실행 중인 명령 때문에 정해진 위치에서 발생
비동기적 = 현재 실행 중인 명령과 직접 상관없이 외부에서 발생
```

### 트랩은 동기적이다

트랩은 현재 실행 중인 명령 때문에 발생한다.

예를 들어 코드 흐름이 다음과 같다고 하자.

```text
1번 명령 실행
2번 명령 실행
3번 명령 실행
4번 명령 실행
```

2번 명령이 시스템 콜이라면 그 자리에서 커널로 진입한다.

```text
1번 명령 실행
2번 명령 실행: 시스템 콜 발생
        ↓
트랩 발생
        ↓
커널 진입
```

원인이 현재 실행 중인 명령 안에 있으므로 동기적이다.

대표 예시는 다음과 같다.

```text
시스템 콜
0으로 나누기
잘못된 메모리 접근
페이지 폴트
```

### 인터럽트는 비동기적이다

인터럽트는 현재 CPU가 실행 중인 명령과 직접 관계없이 외부에서 발생할 수 있다.

```text
1번 명령 실행
2번 명령 실행
갑자기 키보드 입력 발생
        ↓
인터럽트 발생
        ↓
CPU가 잠시 멈추고 인터럽트 처리
```

현재 CPU가 어떤 코드를 실행 중이었는지와 상관없이, 외부 장치나 타이머가 CPU에게 처리가 필요하다고 알리는 것이다.

대표 예시는 다음과 같다.

```text
키보드 입력
마우스 입력
디스크 I/O 완료
네트워크 패킷 도착
타이머 인터럽트
```

정리하면 다음과 같다.

```text
트랩
- 현재 실행 중인 명령 때문에 발생
- 어떤 명령에서 발생했는지 명확함
- 동기적

인터럽트
- 현재 명령 흐름과 별개로 외부에서 발생
- 실행 중간에 언제든 끼어들 수 있음
- 비동기적
```

한 줄로 말하면 다음과 같다.

```text
트랩 = 내가 실행한 명령 때문에 커널로 들어감
인터럽트 = 밖에서 일이 생겨서 CPU를 부름
```

## 2. 동기적/비동기적, 동기/비동기, 동기화는 같은 말인가

비슷해 보이지만 문맥에 따라 의미가 다르다.

```text
동기적/비동기적 = 사건이 현재 실행 흐름과 맞물려 발생하는가
동기/비동기 = 요청한 작업의 완료를 기다리는가
동기화 = 여러 실행 흐름이 공유 자원을 안전하게 쓰도록 순서를 맞추는 것
```

## 3. 동기적/비동기적

트랩과 인터럽트에서 주로 쓰는 표현이다. 사건이 현재 실행 중인 명령 때문에 발생하는지, 외부에서 독립적으로 발생하는지를 구분한다.

```text
0으로 나누기
-> 현재 명령 때문에 발생
-> 동기적

키보드 입력
-> 외부에서 갑자기 발생
-> 비동기적
```

## 4. 동기/비동기

I/O나 함수 호출에서 자주 쓰는 표현이다. 요청한 작업이 끝날 때까지 기다리는지, 요청만 해두고 나중에 완료를 받는지를 구분한다.

동기 파일 읽기는 다음과 같다.

```text
파일 읽기 요청
        ↓
다 읽을 때까지 기다림
        ↓
다음 코드 실행
```

비동기 파일 읽기는 다음과 같다.

```text
파일 읽기 요청
        ↓
바로 다음 코드 실행
        ↓
나중에 완료 알림 받음
```

정리하면 다음과 같다.

```text
동기 = 작업 완료를 기다림
비동기 = 요청만 하고 완료는 나중에 받음
```

## 5. 동기화

동기화는 멀티스레드나 병렬 처리에서 자주 쓰는 표현이다. 여러 실행 흐름이 같은 공유 자원을 동시에 건드려서 결과가 꼬이지 않도록 조절하는 것이다.

예를 들어 두 스레드가 같은 `count` 값을 증가시킨다고 하자.

```text
스레드 A: count 증가
스레드 B: count 증가
```

동시에 실행되면 값이 꼬일 수 있다. 그래서 mutex, lock, semaphore, atomic operation 같은 방법으로 보호한다.

```text
동기화 = 공유 자원 접근 순서를 안전하게 맞추는 것
```

## 6. 전체 정리

```text
동기적/비동기적
- 사건 발생 방식
- 현재 실행 중인 명령과 관련 있는가
- 예: 트랩은 동기적, 인터럽트는 비동기적

동기/비동기
- 작업 처리 방식
- 요청한 작업 완료를 기다리는가
- 예: 동기 파일 읽기, 비동기 파일 읽기

동기화
- 공유 자원 제어
- 여러 실행 흐름이 충돌하지 않게 순서를 맞추는 것
- 예: mutex, lock, semaphore
```

가장 짧게 정리하면 다음과 같다.

```text
동기적/비동기적 = 사건이 언제, 어디서 발생하느냐
동기/비동기 = 작업 완료를 기다리느냐
동기화 = 여러 작업의 충돌을 막느냐
```
