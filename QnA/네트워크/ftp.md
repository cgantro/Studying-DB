# FTP Q&A

## 1. FTP란 무엇인가

FTP는 **File Transfer Protocol**의 약자다. 이름 그대로 네트워크를 통해 파일을 주고받기 위한 응용 계층 프로토콜이다.

```text
FTP = File Transfer Protocol
역할 = 클라이언트와 서버 사이에서 파일 업로드/다운로드
```

예를 들어 내 컴퓨터에서 서버로 파일을 올리거나, 서버에 있는 파일을 내 컴퓨터로 내려받을 때 사용할 수 있다.

```text
내 컴퓨터 -> 서버: 파일 업로드
서버 -> 내 컴퓨터: 파일 다운로드
```

## 2. FTP는 어디 계층인가

FTP는 TCP/IP 모델에서 응용 계층 프로토콜이다.

```text
응용 계층: FTP, HTTP, SMTP, DNS
전송 계층: TCP, UDP
인터넷 계층: IP
링크 계층: Ethernet, Wi-Fi
```

FTP는 신뢰성 있는 전송이 필요하므로 TCP를 사용한다.

```text
FTP
↓
TCP
↓
IP
```

## 3. FTP의 특징: 제어 연결과 데이터 연결

FTP의 중요한 특징은 연결을 두 종류로 나누어 사용한다는 것이다.

```text
제어 연결
- 명령어와 응답을 주고받는 연결
- 예: 로그인, 파일 목록 요청, 다운로드 명령

데이터 연결
- 실제 파일 데이터가 오가는 연결
- 예: 파일 내용, 디렉터리 목록
```

보통 제어 연결은 TCP 21번 포트를 사용한다.

```text
FTP control connection: TCP 21
```

파일을 실제로 전송할 때는 별도의 데이터 연결을 사용한다.

## 4. 예시 흐름

클라이언트가 FTP 서버에서 파일을 다운로드한다고 하자.

```text
1. 클라이언트가 FTP 서버의 21번 포트에 연결
2. 사용자 이름과 비밀번호로 로그인
3. 클라이언트가 "파일 목록 보여줘" 또는 "이 파일 다운로드할게" 명령 전송
4. 서버가 데이터 연결을 통해 파일 목록 또는 파일 데이터 전송
5. 명령과 응답은 제어 연결에서 계속 오감
```

흐름을 단순화하면 다음과 같다.

```text
제어 연결:
클라이언트 -> 서버: 로그인
클라이언트 -> 서버: download file.txt
서버 -> 클라이언트: OK

데이터 연결:
서버 -> 클라이언트: file.txt 내용
```

## 5. Active mode와 Passive mode

FTP에는 데이터 연결을 여는 방식에 따라 active mode와 passive mode가 있다.

### Active mode

Active mode에서는 클라이언트가 서버에게 "내 이 포트로 접속해"라고 알려주고, 서버가 클라이언트 쪽으로 데이터 연결을 건다.

```text
클라이언트 -> 서버: 내 포트로 데이터 연결해
서버 -> 클라이언트: 데이터 연결 시도
```

문제는 클라이언트가 방화벽이나 NAT 뒤에 있으면 서버가 클라이언트로 직접 접속하기 어려울 수 있다는 점이다.

### Passive mode

Passive mode에서는 서버가 "내 이 포트로 접속해"라고 알려주고, 클라이언트가 서버 쪽으로 데이터 연결을 건다.

```text
클라이언트 -> 서버: passive mode 요청
서버 -> 클라이언트: 이 포트로 접속해
클라이언트 -> 서버: 데이터 연결
```

현대 환경에서는 클라이언트가 NAT나 방화벽 뒤에 있는 경우가 많아서 passive mode가 더 자주 쓰인다.

## 6. FTP의 보안 문제

기본 FTP는 사용자 이름, 비밀번호, 파일 내용이 암호화되지 않은 평문으로 전송될 수 있다.

```text
기본 FTP
- 계정 정보 평문 전송 가능
- 파일 내용 평문 전송 가능
- 도청에 취약
```

그래서 보안이 필요한 환경에서는 보통 다음 대안을 사용한다.

```text
FTPS
- FTP에 TLS/SSL 암호화를 붙인 방식

SFTP
- SSH 기반 파일 전송 프로토콜
- 이름은 비슷하지만 FTP와는 별개의 프로토콜
```

## 7. FTP 정리

```text
FTP
- File Transfer Protocol
- 파일 업로드/다운로드용 응용 계층 프로토콜
- TCP 사용
- 제어 연결과 데이터 연결을 나누어 사용
- 기본 FTP는 암호화되지 않아 보안에 취약
```

한 줄로 정리하면, **FTP는 서버와 클라이언트 사이에서 파일을 주고받기 위한 오래된 파일 전송 프로토콜**이다.
