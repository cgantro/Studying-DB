# TCP와 UDP의 통신 상태 Q&A

## 1. 양쪽이 통신 상태를 맞춘다는 뜻

TCP에서 "양쪽이 통신 상태를 맞춘다"는 말은, 통신하는 두 컴퓨터가 데이터를 주고받기 전에 서로 다음 정보를 확인하고 관리한다는 뜻이다.

```text
너 받을 준비 됐어?
나 보낼 준비 됐어.
이 번호부터 데이터 보낼게.
받으면 확인 응답 보내줘.
중간에 빠진 데이터 있으면 다시 보낼게.
```

즉, TCP는 단순히 데이터를 던지는 것이 아니라 **서로 연결 상태를 만들고, 데이터가 잘 도착했는지 계속 확인하면서 통신**한다.

## 2. TCP는 어떤 상태를 맞추는가

TCP는 대표적으로 다음 상태를 맞춘다.

```text
연결이 맺어졌는지
상대가 받을 준비가 되었는지
데이터 순서 번호가 어디부터 시작하는지
어디까지 받았는지
얼마나 더 받을 수 있는지
연결을 언제 종료할지
```

예를 들어 TCP 연결은 보통 3-way handshake로 시작한다.

```text
클라이언트 -> 서버: SYN
서버 -> 클라이언트: SYN + ACK
클라이언트 -> 서버: ACK
```

이 과정은 단순히 인사하는 것이 아니라, 양쪽이 "이제 TCP 연결이 만들어졌고, 이 번호부터 데이터를 주고받자"는 상태를 맞추는 과정이다.

## 3. 예시

친구에게 택배를 보낸다고 생각해보자.

TCP는 이런 방식이다.

```text
나: 지금 받을 수 있어?
친구: 응, 받을 수 있어.
나: 그럼 1번 상자부터 보낼게.
친구: 1번 받았어. 2번 보내.
나: 2번 보낼게.
친구: 2번이 안 왔어. 다시 보내.
```

즉, TCP는 상대가 받았는지 확인하고, 빠진 것이 있으면 다시 보내고, 순서도 맞춘다.

## 4. UDP는 상태를 맞추지 않는가

UDP는 TCP처럼 연결 상태를 만들지 않는다. 데이터를 보낼 때 상대와 미리 연결을 맺거나, 상대가 받았는지 확인하지 않는다.

UDP는 이런 느낌이다.

```text
나: 데이터 보냄
나: 또 데이터 보냄
나: 또 데이터 보냄
```

상대가 받았는지, 순서가 바뀌었는지, 중간에 빠졌는지는 UDP 자체가 보장하지 않는다.

그래서 UDP를 connectionless, 즉 비연결형이라고 한다.

## 5. TCP와 UDP 비교

```text
TCP
- 연결형
- 통신 전에 연결 상태를 맞춤
- 순서 보장
- 손실 시 재전송
- ACK로 수신 확인
- 신뢰성 높음
- 상대적으로 무거움

UDP
- 비연결형
- 연결 상태를 거의 맞추지 않음
- 순서 보장 없음
- 재전송 보장 없음
- 수신 확인 없음
- 빠르고 단순함
- 신뢰성은 애플리케이션이 필요하면 직접 처리
```

## 6. 한 줄 정리

```text
TCP에서 상태를 맞춘다
= 양쪽이 연결 여부, 데이터 순서, 수신 확인, 재전송 정보를 관리하면서 통신한다는 뜻

UDP에서 상태를 맞추지 않는다
= 그냥 데이터를 보내고, 도착 확인이나 순서 관리는 기본적으로 하지 않는다는 뜻
```
