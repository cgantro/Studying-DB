## 1. 프로그램 메모리 구조

- 질문: 프로그램 메모리 구조의 핵심 개념을 면접 답변처럼 1분 안에 설명해 보세요.
- 답변: 프로그램 메모리 구조는 프로그램 실행 시 코드와 데이터가 배치되는 영역입니다. 핵심은 변수 수명과 할당 위치를 모르면 누수, dangling pointer, stack overflow를 설명하기 어렵기 때문입니다. 그래서 코드는 text, 전역/정적 데이터는 data/BSS, 호출 정보는 stack, 동적 객체는 heap에 배치됩니다. 면접에서는 정의만 말하지 말고, 이 개념이 어떤 위험을 줄이고 어떤 기준으로 성능이나 안정성에 영향을 주는지까지 연결하면 좋습니다.
- 답변 키워드: 프로그램 메모리 구조, 스택/힙, 데이터/BSS, 수명, 할당 위치, 누수, 정의, 핵심 목적, 면접식 요약
- 관련 문서: [[CS공부/10_언어/01_메모리 구조/01_메모리 구조|메모리 구조]]

## 2. 스택 영역

- 질문: 스택 영역이/가 필요한 이유와 해결하려는 문제는 무엇인가요?
- 답변: 스택 영역이 필요한 이유는 변수 수명과 할당 위치를 모르면 누수, dangling pointer, stack overflow를 설명하기 어렵기 때문입니다. 이를 해결하기 위해 시스템은 코드는 text, 전역/정적 데이터는 data/BSS, 호출 정보는 stack, 동적 객체는 heap에 배치됩니다 결과적으로 애플리케이션이나 사용자는 하드웨어, 저장소, 네트워크, 런타임의 세부 구현을 모두 직접 다루지 않고도 일관된 규칙 위에서 동작할 수 있습니다.
- 답변 키워드: 스택 영역, 스택/힙, 데이터/BSS, 수명, 할당 위치, 누수, 문제 배경, 도입 이유, 대안 비교

## 5. BSS 영역

- 질문: BSS 영역에서 성능 병목이나 비용이 커지는 지점은 어디인가요?
- 답변: BSS 영역에서 성능 병목은 주로 heap 할당, stack 깊이, GC/해제 비용, cache locality에서 커집니다. 병목을 확인할 때는 먼저 처리량, 지연 시간, 메모리 사용량처럼 관측 가능한 지표를 잡고, memory leak, dangling pointer, stack overflow, 초기화 오해 가능성을 하나씩 좁혀야 합니다. 개선은 가장 큰 비용을 만든 구간을 측정으로 확인한 뒤 적용해야 합니다.
- 답변 키워드: BSS 영역, 스택/힙, 데이터/BSS, 수명, 할당 위치, 누수, 병목 지점, 비용, 측정 지표
- 관련 문서: [[CS공부/10_언어/01_메모리 구조/01_메모리 구조|메모리 구조]]
