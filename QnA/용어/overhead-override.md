# 오버헤드와 오버라이드 Q&A

## 1. 오버헤드란 무엇인가

오버헤드(overhead)는 어떤 일을 하기 위해 필요한 **부가 비용**을 뜻한다.

여기서 비용은 꼭 돈이 아니라 다음을 포함한다.

```text
시간
CPU 사용량
메모리 사용량
네트워크 트래픽
관리 비용
```

예를 들어 문맥 교환은 오버헤드가 있다.

```text
프로세스 A 중단
A의 레지스터 상태 저장
프로세스 B의 상태 복원
캐시/TLB 영향 발생
B 실행
```

이 과정은 실제 사용자 작업을 처리하는 시간이 아니라, 작업을 바꾸기 위해 드는 부가 비용이다. 그래서 문맥 교환 오버헤드라고 한다.

또 다른 예시는 함수 호출 오버헤드다.

```text
함수 인자 전달
반환 주소 저장
스택 프레임 생성
함수 종료 후 복귀
```

함수 안의 실제 계산 외에 호출 자체를 위해 드는 비용이 있다.

정리하면 다음과 같다.

```text
오버헤드 = 본 작업을 하기 위해 추가로 드는 비용
```

## 2. 오버라이드란 무엇인가

오버라이드(override)는 객체지향 프로그래밍에서 **부모 클래스의 메서드를 자식 클래스가 다시 정의하는 것**을 뜻한다.

예를 들어 Java 코드로 보면 다음과 같다.

```java
class Animal {
    void sound() {
        System.out.println("some sound");
    }
}

class Dog extends Animal {
    @Override
    void sound() {
        System.out.println("bark");
    }
}
```

`Animal`에는 `sound()` 메서드가 있다. `Dog`는 `Animal`을 상속받았지만, `sound()`를 자기 방식으로 다시 정의했다. 이것이 오버라이드다.

```text
부모 메서드: Animal.sound()
자식이 재정의: Dog.sound()
```

오버라이드를 하면 부모 타입으로 다루더라도 실제 객체의 메서드가 실행될 수 있다.

```java
Animal a = new Dog();
a.sound(); // bark
```

정리하면 다음과 같다.

```text
오버라이드 = 상속 관계에서 부모 메서드를 자식이 다시 정의하는 것
```

## 3. 오버헤드와 오버라이드 차이

```text
오버헤드
- 부가 비용
- 성능, 자원 사용량, 처리 시간 이야기에서 자주 사용
- 예: 문맥 교환 오버헤드, 네트워크 오버헤드

오버라이드
- 부모 메서드를 자식 클래스에서 재정의
- 객체지향 프로그래밍에서 사용
- 예: Animal.sound()를 Dog.sound()로 재정의
```

한 줄로 정리하면 다음과 같다.

```text
오버헤드 = 추가로 드는 비용
오버라이드 = 상속받은 기능을 다시 정의하는 것
```
