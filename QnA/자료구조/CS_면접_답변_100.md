각 답변은 질문 번호와 1:1로 대응하며, 면접에서 말할 수 있는 핵심 논리와 관련 문서 백링크를 함께 정리했습니다.

## 1. 배열

- 질문: 배열의 핵심 개념을 면접 답변처럼 1분 안에 설명해 보세요.
- 답변: 배열은 연속 메모리에 값을 나열하는 구조입니다. 핵심은 인덱스로 빠르게 접근해야 하는 데이터를 효율적으로 저장할 필요가 있기 때문입니다. 그래서 base address, index, element size로 위치를 계산해 O(1)에 접근합니다. 면접에서는 정의만 말하지 말고, 이 개념이 어떤 위험을 줄이고 어떤 기준으로 성능이나 안정성에 영향을 주는지까지 연결하면 좋습니다.
- 답변 키워드: 배열, 연속 메모리, 인덱스 접근, 삽입/삭제 비용, 캐시 지역성, 크기 관리, 정의, 핵심 목적, 면접식 요약
- 관련 문서: [[CS공부/08_자료구조/01_배열|배열]]

## 2. 동적 배열

- 질문: 동적 배열이/가 필요한 이유와 해결하려는 문제는 무엇인가요?
- 답변: 동적 배열이 필요한 이유는 인덱스로 빠르게 접근해야 하는 데이터를 효율적으로 저장할 필요가 있기 때문입니다. 이를 해결하기 위해 시스템은 base address, index, element size로 위치를 계산해 O(1)에 접근합니다. 결과적으로 애플리케이션이나 사용자는 하드웨어, 저장소, 네트워크, 런타임의 세부 구현을 모두 직접 다루지 않고도 일관된 규칙 위에서 동작할 수 있습니다.
- 답변 키워드: 동적 배열, 연속 메모리, 인덱스 접근, 삽입/삭제 비용, 캐시 지역성, 크기 관리, 문제 배경, 도입 이유, 대안 비교
- 관련 문서: [[CS공부/08_자료구조/01_배열|배열]]

## 5. 인덱스 접근

- 질문: 인덱스 접근에서 성능 병목이나 비용이 커지는 지점은 어디인가요?
- 답변: 인덱스 접근에서 성능 병목은 주로 중간 삽입/삭제, 재할당, cache miss, 큰 복사 비용에서 커집니다. 병목을 확인할 때는 먼저 처리량, 지연 시간, 메모리 사용량처럼 관측 가능한 지표를 잡고, 범위 초과, 재할당 비용, 중간 삽입 비용, sparse 데이터 낭비 가능성을 하나씩 좁혀야 합니다. 개선은 가장 큰 비용을 만든 구간을 측정으로 확인한 뒤 적용해야 합니다.
- 답변 키워드: 인덱스 접근, 연속 메모리, 삽입/삭제 비용, 캐시 지역성, 크기 관리, 병목 지점, 비용, 측정 지표
- 관련 문서: [[CS공부/08_자료구조/01_배열|배열]]
