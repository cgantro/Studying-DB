### 정의
AI 거버넌스는 AI를 조직 내에서 누가, 어떤 데이터로, 어떤 목적에 어느 권한까지 어떤 검증과 책임 체계 아래 사용할 지 정하는 관리/통제 체계

### 왜 중요해졌나
예전 AI는 결과를 참고하는 수준이 많았음(추천, 분류)
지금은 AI가 문서 작성,코드 작성, 리뷰 등등의 역할까지 맡음
SDS에서도 데이터 유출 방지, 반복 업무 자동화, 등등

| 영역             | 핵심 질문                      | 개발 관점 예시                            |
| -------------- | -------------------------- | ----------------------------------- |
| **목적·정책**      | 이 AI를 왜 쓰는가? 어디까지 허용할 것인가? | AI 코드리뷰는 조언만 할지, MR 차단까지 할지         |
| **책임 구조**      | 문제가 생기면 누가 책임지는가?          | 모델 운영자, 서비스 담당자, 보안 담당자 분리          |
| **데이터 거버넌스**   | 어떤 데이터를 넣어도 되는가?           | 개인정보, 사내 문서, 고객 데이터 마스킹             |
| **모델·프롬프트 관리** | 어떤 모델과 프롬프트를 쓰는가?          | 프롬프트 버전관리, 모델 변경 이력 기록              |
| **권한·도구 통제**   | AI가 어떤 시스템을 호출할 수 있는가?     | DB 조회만 허용, 결제·삭제·배포는 금지             |
| **평가·검증**      | 답변이 정확하고 안전한가?             | 테스트셋, hallucination 평가, 보안 테스트      |
| **로그·감사**      | 나중에 추적 가능한가?               | 입력, 출력, 호출 도구, 사용자, 시간 기록           |
| **모니터링**       | 운영 중 성능·위험이 변하지 않는가?       | 실패율, 금칙어, 개인정보 노출, 응답 품질 모니터링       |
| **사고 대응**      | 문제가 나면 어떻게 멈추고 복구하는가?      | kill switch, rollback, human review |
