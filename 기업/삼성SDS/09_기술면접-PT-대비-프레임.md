# 기술면접/PT 대비 프레임 (삼성SDS)

출처:
- `내 노션/정리본/SDS 면접 전체 정리`
- `내 노션/정리본/SDS 면접`

주의:
- 위 노션 문서에는 외부 링크/수치가 섞여 있고, 최신성도 보장되지 않는다. 이 문서는 “대비 프레임”만 정리한다.
- 매출/실적/시장 수치 같은 내용은 면접에서 단정하지 말고, 필요하면 지원 시점 기준으로 다시 확인한다.

## 1) 면접에서 자주 나오는 축(내 경험 기반 정리)
- 회사/직무: 왜 SDS, 어떤 영역(Brity/FabriX/클라우드/물류 등), 기대와 다른 일을 맡으면?
- 프로젝트: 역할/기여, 기술 선택 이유, 장애/병목 원인-해결-대안, 운영 관점(권한/로그/복구)
- 개인/인성: 갈등/피드백, 실패 경험, 강점/약점 관리, 협업을 “행동”으로 설명

## 2) 설계형 질문 답변 프레임(고정)
- 문제 정의(비즈니스/사용자) → 제약(보안/권한/운영/규모) → 경계(컴포넌트/책임) → 핵심 흐름
- 병목/장애 포인트 → 관측(로그/메트릭/트레이싱) → 복구(재시도/롤백/백오프) → 대안/트레이드오프

## 3) 기술 PT 주제 분류(체크리스트)
네트워크/인프라:
- TCP vs UDP, WebSocket/WebRTC, 재전송/순서/혼잡, NAT/방화벽
- 로드밸런싱, 장애 격리, 타임아웃/리트라이

데이터/백엔드:
- 트랜잭션/격리수준, 인덱스, 캐시(장단점), CQRS/비동기 처리
- 큐/스트리밍(카프카 등)을 “왜” 쓰는지(운영/확장/재처리)

운영/품질:
- 배포 전략(롤링/블루그린), 관측성(로그/메트릭), 알람 기준
- 권한/감사 로그, 외부 연동 실패 시 정책(재시도/보상/수동 처리)

## 4) 내 프로젝트로 연결하는 한 문장
- “실시간 흐름에서 최신 상태를 유지하고, 장애가 나면 재현 가능한 형태로 좁혀서 고치는 경험”을 Brity 같은 협업 솔루션의 운영 품질에 연결한다.

근거 프로젝트(요약):
- RobotPal: 스트리밍 캡처/인코딩/전송 병목을 단계로 분리해 개선
- Autowing: 실시간 메시징 경계(MQTT↔WebSocket)와 운영/인증/배포까지 E2E로 정리
- mm-bot: 배치 적재/실시간 질의 분리로 실패 격리
