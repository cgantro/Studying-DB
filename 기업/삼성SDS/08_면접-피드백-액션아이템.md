# 면접 피드백/액션아이템 (삼성SDS)

출처:
- `내 노션/정리본/2024 04-25 면접 피드백`
- `내 노션/정리본/컨설팅 요약`

## 1) 답변 퀄리티 피드백 → 수정 방향
- 암기 느낌:
  - 문장을 외우지 말고 “경험에서 꺼내는 구조”로 바꾼다.
  - 질문을 받으면 “내가 겪은 문제/원인/해결/대안”부터 말한다.
- B2B 관점:
  - 기능 설명보다 운영/장애/보안/권한/유지보수 관점으로 말한다.

## 2) 기술 PT/설계형 대비(프레임)
- 문제 정의(비즈니스 문제) → 가정/제약 → 아키텍처 → 병목/장애 포인트 → 관측/복구 → 대안/트레이드오프

## 3) Python 질문 대비(스탠스)
- “주력 언어”보다 “문제 해결을 위해 필요한 도구를 고른다”가 핵심.
- Python은 자동화/로그 분석/데이터 처리/AI API 연결로 설명한다.

## 4) 기본기 준비 범위(깊이 기준)
- Java 17/JVM, 동시성, 네트워크(TCP/UDP/DNS/로드밸런싱), DB 인덱스/트랜잭션, Redis 캐시/대안, CI/CD, 관측(로그/메트릭)
- 단순 정의가 아니라 “왜 필요했고, 어떤 트레이드오프가 있는지”로 말한다.

## 5) AI 활용 답변(구체화 방향)
- “AI를 썼다”가 아니라 “AI가 실수하지 않게 가드레일을 어떻게 뒀는가”를 말한다.
- 예시(면접용 방향):
  - 규칙 문서(AGENTS.md)로 수정 범위/금지사항을 명시
  - 테스트/리뷰 체크리스트를 생성해 누락을 줄임
- 수치/시간 단축 같은 정량 근거는 **확인 필요**면 면접용 답변에서는 제외.

## 6) 이번 주 실행 체크리스트(연습용)
- 30초/1분/2분 버전으로: 자기소개, 지원동기, 프로젝트 2개, 협업/갈등, 실패 1개
- 각 답변마다:
  - “대안 1개”와 “왜 안 했는지”를 1문장으로 준비
  - “운영에서 어떻게 깨질 수 있는지”를 1문장으로 준비
