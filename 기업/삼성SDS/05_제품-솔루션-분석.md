# 제품/솔루션 분석 (FabriX/Brity 중심)

작성 기준:
- 1차 근거는 `내 노션/정리본/SDS 기업 분석`, `삼성SDS 지원 전략 - Brity 중심` 메모를 재구성한 것.
- 제품 세부 기능/수치(정확도 %, 라이브러리 개수 등)는 **확인 필요**. 면접에서는 “숫자 단정” 대신 “방향/가치/내 경험 연결”로 말한다.

## 1) FabriX (기업용 생성형 AI/에이전트 플랫폼)
### 내가 이해한 제품 역할
- 기업 데이터(업무/문서/시스템)를 안전하게 통합해, 생성형 AI 또는 에이전트를 “업무에 붙일 수 있게” 만드는 플랫폼 성격.

### 면접 연결 포인트
- “RAG/에이전트”의 핵심은 모델 선택보다 데이터 경계, 권한, 감사/로그, 운영성.
- 내가 말할 수 있는 경험 연결:
  - mm-bot: 배치 적재(임베딩/쓰기)와 실시간 질의(읽기) 분리로 latency/실패 격리
  - Autowing/RobotPal: 실시간 흐름에서 병목/정합성 문제를 단계 분리로 해결

### 예상 꼬리질문
- 기업 데이터 연동에서 가장 어려운 건 무엇인가(권한/스키마/품질/변경 대응)?
- “에이전트가 잘못된 답/행동”을 하면 어떤 안전장치를 둘 건가(HITL, 정책, 로그/재현)?

## 2) Brity Copilot (협업 + AI)
### 내가 끌리는 지점
- 협업 도구는 “동시성/권한/알림/검색/이력”이 얽혀서, 기능 추가보다 운영 품질이 핵심.
- AI가 붙을수록 “맥락 단절/정보 불일치”를 줄이는 방향이 중요해진다고 본다.

### 내 경험 연결
- Autowing: 상태 전이/승인 흐름을 구조로 강제(요청→추천→승인)해서 운영 안전성을 올린 경험
- RobotPal: 최신 프레임 우선(프레임 드롭) 같은 “실시간 품질 기준”을 명확히 둔 경험

## 3) Brity Automation (업무 자동화)
### 내가 이해한 제품 역할
- 반복 작업을 자동화하는 RPA를 넘어 “업무 프로세스 전체”를 자동화하는 방향.

### 면접 연결 포인트
- 자동화에서 중요한 건 “정확도”보다 실패 시 복구, 재시도, 감사/로그, 롤백.
- “자동화할 만한 것”과 “사람이 개입해야 하는 것(Human-in-the-Loop)”을 나누는 기준이 필요.

## 4) 내가 SDS에서 노리는 자리(한 문장)
- Brity 같은 협업 솔루션에서 실시간 이벤트/상태 정합성을 깨지 않게 유지하고, 운영에서 재현 가능한 구조와 관측 지표를 갖춘 백엔드 개발.
