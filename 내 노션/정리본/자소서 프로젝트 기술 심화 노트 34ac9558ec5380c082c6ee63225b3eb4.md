- 안정성이 왜 중요한지

- 특정 기술과 대립하는 기술을 비교하고 선택 이유를 설명할 수 있는지

- 공백기/휴학/학습 흐름을 어떻게 설명할지

## 2. 프로젝트 심화 - AutowingCar

### 핵심 포인트

- 자율주행 견인차 관제 백엔드
- 단순 데이터 수집 서버가 아니라 향후 차량이 늘어나도 안정적으로 동작하는 실시간 관제 구조를 목표로 함
- 최신 상태 조회와 장기 저장이 하나의 흐름에 묶일 경우 병목이 생길 수 있었음
- 비동기 환경에서 응답 순서가 어긋나며 상태 불일치 가능성이 있었음

### 설명 포인트

- 최신 상태 조회와 장기 저장 분리
- Redis와 DB 역할 분리
- ACK 기반 상태 전이
- 상태 롤백 방지
- 확장 상황까지 고려한 설계

### Brity 연결 해석

- 협업 솔루션도 결국 사용자의 상태, 메시지, 문서, 이벤트 흐름이 어긋나면 안 됨
- 따라서 이 경험은 Brity처럼 여러 흐름이 동시에 존재하는 서비스와 연결 가능

## 3. 프로젝트 심화 - RobotPal

### 핵심 포인트

- 카메라 캡처/인코딩과 렌더 루프 간 경쟁으로 인해 드랍 발생
- 병목을 분석하고 Producer-Consumer 구조 및 멀티워커 비동기 처리로 재설계
- 프레임 드랍률을 개선

### 면접 방어 포인트

- 이 수치는 전체 렌더링 FPS가 아니라 **인코딩/큐 병목을 모델링한 마이크로 벤치마크**라는 점을 먼저 인정
- 절대값보다 동일 조건의 상대 비교에 의미가 있음
- GPU readback 병목까지 다 증명한 것은 아니며, CPU 측 스트리밍 병목에 집중한 측정임

### 절대 피할 말

- 실제 서비스 FPS라고 단정
- 어떤 환경에서도 동일하다고 주장
- GPU 병목까지 모두 증명했다고 과장

## 4. 협업/인성 심화 노트

### 밴드/베이스

- 베이스 7년, 밴드 10년
- 단순 취미가 아니라 오랫동안 한 가지를 꾸준히 파고든 성실성의 근거
- 공연 준비, 장소 섭외, 일정 조율 경험은 협업/조율 능력 사례로 활용

### 인정받지 못했을 때

- 감정적으로 대응하기보다, 결과와 근거를 명확히 설명하고 협업 과정에서 신뢰를 쌓는 방향
- 내 기여를 드러내되 팀 성과를 우선하는 태도

## 5. AI / 거버넌스 / 문서화

### 관심 주제

- AI 사용 경험
- AI 활용 서비스
- 윤표봇
- LLM을 활용한 자동화
- 하네스 엔지니어링
- 문서화 체계화
- AI 결과물 검증 체계

### 말할 포인트

- AI는 생산성을 높일 수 있지만, 기업에서는 출처 추적, 유사도 검사, 저작권 귀속, 보안, 검증 가능성이 중요함
- 단순히 “AI 좋다”가 아니라 **체계 안에서 검증 가능하게 써야 한다**는 관점
- 문서화는 협업, 유지보수, 자동화, AI 활용의 기반이라는 점 강조
