# SDS 면접 전체 정리

## 1. 면접 개요

- 실무면접은 문제풀이형과 설계형이 함께 나오고, 잘할 수 있는 문제를 선택해서 푸는 형태가 있었다.
- 이력서 기반 질문도 함께 받아서 생각보다 문제 해설 시간이 부족할 수 있다.
- 역량면접은 자소서 경험 기반으로 진행되었고, 기술면접은 제시된 주제 중 하나를 선택해 구조를 설계하고 설명하는 방식이 있었다.

## 2. 면접에서 자주 나오는 질문 축

### 회사/직무

- 우리 회사 들어오면 무엇을 하고 싶은가
- 기대와 다른 일을 맡게 되면 어떻게 할 것인가
- 삼성SDS에서 어떤 직무를 하고 싶은가
- 입사 후 하고 싶은 일이 무엇인가
- 파견 근무에 대해 어떻게 생각하는가
- 삼성SDS가 어떤 신규 사업을 통해 발전하면 좋을지

### 프로젝트

- 프로젝트 설명
- 정확히 어떤 역할을 맡았는가
- 기술을 왜 그렇게 선택했는가
- 프로젝트에서 구조 개선/플랫폼 고도화를 어떻게 했는가
- 병목 분석과 성능 개선을 어떻게 했는가
- 안정적으로 데이터가 흐르는 구조를 어떻게 만들었는가

### 개인/인성

- 개발자가 되려는 이유
- 인생에서 가장 큰 성취감을 느낀 경험
- 내가 한 일에 대해 인정을 받지 못하면 어떻게 할 것인가
- 취미는 무엇인가
- 불공평한 일을 겪은 경험
- 팀 프로젝트 갈등 상황 및 해결
- 무임승차 팀원 경험이 있는가
- 또래상담, 밴드 활동, 조율 경험이 실무에서 어떻게 쓰일 수 있는가

## 3. 기술면접/PT면접 키워드

### 네트워크/인프라

- HTTPS로 www 입력 시 IP를 얻는 과정
- DNS 서버를 쓰는 이유
- TCP와 UDP 차이
- 로드밸런싱 알고리즘 설명
- 스케일업과 스케일아웃 차이 및 선택 이유
- 오토스케일 시 실제로 서버를 어떻게 늘리고, 늘어난 서버끼리 어떻게 통신하는가
- Stateless하게 설계하는 이유

### 데이터/백엔드

- DB 인덱스 설정 기준
- Redis를 캐시로 사용하는 이유
- Redis 대신 쓸 수 있는 대안
- Master-Slave 구조
- 읽기 서버 확장 시 고려사항
- MongoDB가 좋은 이유와 MySQL을 선택한 이유
- PostgreSQL은 고려했는가

### 운영/품질

- CI/CD를 왜 썼는가
- 프로젝트 안정성을 어떻게 확보했는가
- 부하 테스트 툴을 써봤는가
- 클라우드 전환 시 트래픽 급증 대응 방안
- 생성형 AI 관련 생각

## 4. 삼성SDS 면접 준비 방향

- 삼성SDS는 B2B 회사 특성이 강하기 때문에 단순 구현보다 **안정성, 운영성, 확장성, 유지보수성** 관점이 중요하다.
- 기술을 썼다는 사실보다 **왜 그 기술을 선택했는지**, **대안 대비 장점이 무엇인지**를 설명할 수 있어야 한다.
- 프로젝트 답변은 **문제 정의 → 선택 이유 → 적용 방식 → 결과** 흐름으로 준비한다.
- 기술 질문은 깊게 들어올 수 있으므로 **사용 기술의 한계**도 함께 준비한다.

## 5. 주의할 리스크

- 성적이 낮았던 과목(C+) 질문 가능성
- “역량을 키우겠다” 같은 미래형 표현보다 이미 보여준 역량으로 말할 것
- AI 관련 문제의식은 있었지만 해결책이 추상적이면 약해 보일 수 있음
- 기술적 쿠션어를 쓰면 꼬리질문에서 무너질 수 있으니 구조적으로 답변 준비 필요
