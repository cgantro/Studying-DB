# 전공지식

1. **시간 복잡도 (Time Complexity):** 알고리즘이 문제를 해결하는 데 걸리는 시간의 척도.
2. **공간 복잡도 (Space Complexity):** 알고리즘이 문제를 해결하는 데 필요한 메모리 공간의 척도.
3. **빅-O 표기법 (O(N)):** 시간 및 공간 복잡도를 표현하는 수학적 표기법 (상한).
4. **최적화(Optimization):** 코드 및 알고리즘의 성능을 향상시키는 과정.
5. **상수 시간 (O(1)):** 입력 크기에 관계없이 항상 일정한 시간 소요.
6. **로그 시간 (O(logN)):** 입력 크기가 증가할수록 시간이 매우 느리게 증가.
7. **선형 시간 (O(N)):** 입력 크기에 비례하여 시간이 선형적으로 증가.
8. **선형-로그 시간 (O(NlogN)):** 정렬 알고리즘에서 흔히 보임.
9. **제곱 시간 (O(N2)):** 입력 크기의 제곱에 비례하여 시간이 증가.
10. **지수 시간 (O(2N)):** 입력 크기가 1 증가할 때마다 시간이 2배씩 증가 (매우 비효율적).
11. **배열 (Array):** 순차적 데이터 저장, 인덱스를 통한 빠른 접근, 크기 고정.
12. **연결 리스트 (Linked List):** 동적 크기, 삽입/삭제 용이, 탐색 속도 느림.
13. **스택 (Stack):** LIFO(Last-In, First-Out) 구조. (함수 호출 스택, 실행 취소)
14. **큐 (Queue):** FIFO(First-In, First-Out) 구조. (메시지 큐, BFS)
15. **트리 (Tree):** 계층적 데이터 표현.
16. **이진 트리 (Binary Tree):** 각 노드가 최대 두 개의 자식 노드를 가짐.
17. **이진 탐색 트리 (Binary Search Tree, BST):** 효율적인 검색, 삽입, 삭제.
18. **힙 (Heap):** 우선순위 큐 구현에 사용되는 완전 이진 트리. (최대 힙, 최소 힙)
19. **그래프 (Graph):** 노드와 엣지로 구성된 데이터 구조. (네트워크, 소셜 관계)
20. **해시 테이블 (Hash Table):** 키-값 쌍 저장, 평균적으로 O(1) 탐색 속도.
21. **해시 충돌 (Hash Collision):** 다른 키가 같은 해시 값을 가질 때 발생.
22. **충돌 해결 기법:** 체이닝, 개방 주소법(선형 조사, 제곱 조사, 이중 해싱).
23. **정렬 알고리즘:** 버블, 선택, 삽입, 퀵, 병합, 힙 정렬 등. 각 알고리즘의 시간 복잡도, 안정성, 제자리 정렬 여부.
24. **탐색 알고리즘:** 선형 탐색, 이진 탐색.
25. **너비 우선 탐색 (BFS):** 최단 경로 탐색에 유리.
26. **깊이 우선 탐색 (DFS):** 모든 노드 방문, 특정 경로 탐색에 유리.
27. **동적 계획법 (Dynamic Programming):** 중복 계산을 피하여 효율성을 높이는 기법. (피보나치 수열, 최단 경로)
28. **분할 정복 (Divide and Conquer):** 큰 문제를 작은 문제로 나누어 해결. (병합 정렬, 퀵 정렬)
29. **그리디 알고리즘 (Greedy Algorithm):** 각 단계에서 최적의 선택을 하여 전체 최적해를 찾는 기법.
30. **백트래킹 (Backtracking):** 해를 찾는 도중 해가 아니면 되돌아가 다른 경로 탐색.
31. **재귀 (Recursion):** 함수가 자신을 호출하여 문제를 해결하는 기법. (메모리 스택 오버플로우 주의)
32. **반복 (Iteration):** 루프를 사용하여 문제를 해결.
33. **비트 연산 (Bitwise Operations):** 정수 단위의 빠른 연산으로 성능 최적화.
34. **스트링 빌더/버퍼 (StringBuilder/StringBuffer):** 문자열 조작 시 `String` 객체 생성 오버헤드 감소.
35. **불변 객체 (Immutable Object):** 객체 생성 후 상태 변경 불가, 스레드 안전성 향상, 예상치 못한 동작 방지.
36. **컬렉션 프레임워크 (Java Collections Framework):** `ArrayList`, `LinkedList`, `HashMap`, `HashSet` 등 각 컬렉션의 내부 동작 및 성능 특성.
37. **제네릭스 (Generics):** 타입 안정성 확보 및 형변환 오버헤드 감소.
38. **스트림 API (Java Stream API):** 컬렉션 데이터 처리를 위한 선언적이고 효율적인 방법. (병렬 처리 가능)
39. **람다 표현식 (Lambda Expressions):** 익명 함수를 통해 코드 간결화 및 함수형 프로그래밍 도입.
40. **널 포인터 예외 (NullPointerException) 방지:** `Optional` 클래스, 방어적 코딩.
41. **캐시 지역성 (Locality of Reference):** CPU 캐시 활용을 위한 코드 패턴 (시간 지역성, 공간 지역성).
    - 시간 → 최근 참조된 주소 → 곧 참조 가능
    - 공간 → 기억장치 내에 서로 인접하여 저장되있는 데이터 → 연속적 참조 가능
42. **버퍼링 (Buffering):** I/O 작업 시 데이터 덩어리(블록) 단위 처리로 성능 향상.
43. **정적 팩토리 메서드 (Static Factory Method):** 생성자 대신 객체 생성 시 유연성 및 성능 이점.
    - 이름을 가질 수 있다.
    - 호출 될 때마다 인스턴스를 새로 생성안해도 된다.
    - 반환 타입의 하위 타입 객체를 반환 가능하다.
44. **프리미티브 타입 vs Wrapper 클래스:** 메모리 사용 및 성능 차이.
    - 래퍼 : 기본형을 객체로 변환함.
        - null을 넣거나, 메서드 사용 가능
        - heap 메모리 저장 → 느림, 값 직접 변경 불가능
        - 제네릭 가능
    - 기본형 : 빠름, 변경 가능
45. **코드 리팩토링 (Refactoring):** 코드의 외부 동작 변경 없이 내부 구조 개선을 통한 유지보수성 및 효율성 향상.
46. **디버깅 기법:** 효율적인 문제 진단 및 해결 능력.
47. **테스트 주도 개발 (TDD):** 테스트 코드를 먼저 작성하여 견고하고 효율적인 코드 설계 유도.
48. **벤치마킹 (Benchmarking):** 코드 또는 시스템의 성능을 측정하고 비교하는 과정.
49. **프로파일링 (Profiling):** 애플리케이션의 CPU, 메모리, I/O 사용량 등 성능 병목 지점을 식별하는 도구 사용 능력.
50. **컨테이너 (Container):** Docker와 같은 컨테이너 기술을 사용하여 애플리케이션을 가볍고 이식 가능하게 패키징, 효율적인 자원 사용 및 배포.

### **2. 데이터베이스 최적화 및 관리**

1. **인덱스(Index):** 검색 속도 향상
    - `Unique Index`
        - 조회를 자주 하는 경우
        - 삽입, 수정, 삭제가 빈번하지 “않은” 경우
        - 중복도가 낮은 경우
2. **인덱스 종류:** B-Tree, Hash, Full-text, Spatial 등.
3. **인덱스 생성 기준:** WHERE 절, JOIN 조건, ORDER BY, GROUP BY 컬럼.
4. **인덱스 남용의 문제점:** 삽입, 수정, 삭제 시 성능 저하, 추가적인 디스크 공간 소모.
5. **쿼리 최적화(Query Optimization):** SQL 쿼리 실행 계획 분석(`EXPLAIN` 명령)
    - MYSQL에서는 최적의 실행 계획을 수립해주는 Optimizer가 내부적으로 동작한다.
    - EXPLAIN → Query Plan을 사용자가 알아보기 쉽게 결과 Set으로 보여준다.
6. **조인(JOIN) 최적화:** 적절한 조인 방식 선택(INNER, LEFT, RIGHT), 조인 순서.
7. **서브쿼리(Subquery) vs 조인:** 성능 특성 비교 및 적절한 사용.
8. **불필요한 데이터 조회 최소화:** `SELECT *` 지양, 필요한 컬럼만 선택.
9. **LIMIT/OFFSET 사용 시 주의사항:** 대량 데이터 페이징 시 성능 문제.
10. **정규화(Normalization):** 데이터 중복 제거, 무결성 향상 (1NF, 2NF, 3NF, BCNF).
11. **역정규화(Denormalization):** 조회 성능 향상을 위해 의도적으로 중복 허용, 데이터 일관성 관리 유의.
12. **트랜잭션(Transaction):** 논리적인 작업 단위.
13. **ACID 속성:** 원자성(Atomicity), 일관성(Consistency), 고립성(Isolation), 지속성(Durability).
14. **트랜잭션 격리 수준(Isolation Level):** Read Uncommitted, Read Committed, Repeatable Read, Serializable.
15. **동시성 제어(Concurrency Control):** 잠금(Locking), 다중 버전 동시성 제어(MVCC) 등.
16. **데드락(Deadlock):** 상호 배타적 잠금으로 인한 교착 상태, 회피/탐지/복구 전략.
17. **데이터베이스 연결 풀(Connection Pool):** DB 커넥션 재사용, 오버헤드 감소 (HikariCP, Apache DBCP).
18. **Prepared Statement:** SQL 인젝션 방지, 쿼리 파싱 비용 절감으로 성능 향상.
19. **데이터 타입 선택:** 저장 공간 및 연산 효율성을 고려한 적절한 데이터 타입(INT, BIGINT, VARCHAR, TEXT 등) 사용.
20. **캐싱(Caching):** 데이터베이스 접근 횟수를 줄여 성능 향상 (Application Cache, Database Cache).
21. **NoSQL 데이터베이스:** 관계형 DB의 한계(확장성, 유연성) 보완 (MongoDB, Cassandra, Redis).
22. **Redis:** 인메모리 데이터 스토어, 빠른 읽기/쓰기, 다양한 자료구조 지원 (문자열, 해시, 리스트, 셋, 정렬된 셋).
    - **프로젝트 연관:** `member_service_analysis.txt`의 `delegateAccessToken / delegateRefreshToken` 개선안에서 Redis를 사용한 Refresh Token 저장 방식으로 언급됨. TTL(Time-To-Live) 기능을 이용한 토큰 만료 관리.
23. **샤딩(Sharding):** 대규모 데이터를 여러 DB 인스턴스에 분산 저장하여 확장성 및 성능 향상.
24. **레플리케이션(Replication):** DB 복제본을 생성하여 읽기 부하 분산 및 고가용성 확보.
25. **데이터베이스 마이그레이션 도구:** Flyway, Liquibase 등 스키마 변경 이력 관리.
26. **백업 및 복구 전략:** 데이터 유실 방지를 위한 주기적 백업 및 복구 계획.
27. **쿼리 튜닝:** 느린 쿼리 식별 및 개선 과정.
28. **Explain Plan:** 쿼리 실행 계획을 분석하여 성능 병목 지점 파악.
29. **벌크성 연산(Bulk Operations):** 여러 건의 데이터를 한 번의 쿼리로 처리하여 DB 부하 감소.
30. **페이징(Paging):** 대량 데이터 조회 시 응답 속도 향상 (LIMIT, OFFSET).
31. **MVCC (Multi-Version Concurrency Control):** 동시성 제어 메커니즘, 락 없이 읽기 가능.
32. **데이터베이스 프로파일링 도구:** DB 성능 모니터링 및 튜닝.
33. **UUID vs Long ID:** ID 생성 방식에 따른 성능 및 저장 공간 차이.
34. **데이터베이스 보안:** SQL 인젝션 방지, 접근 제어, 데이터 암호화.
35. **스키마 설계:** 확장성과 성능을 고려한 테이블 및 컬럼 설계.
36. **트리거(Trigger):** 특정 이벤트 발생 시 자동 실행되는 SQL 코드.
37. **스토어드 프로시저(Stored Procedure):** 일련의 SQL 문을 저장하여 재사용 및 성능 향상.
38. **뷰(View):** 복잡한 쿼리를 추상화하여 사용 편의성 및 보안 강화.
39. **클러스터링(Clustering):** 여러 서버를 하나의 시스템처럼 묶어 고가용성 및 확장성 제공.
40. **데이터베이스 모니터링:** 시스템 자원, 쿼리 성능 등 주기적 모니터링.
41. **ORM(Object-Relational Mapping):** 객체 지향 언어와 관계형 데이터베이스 매핑. (JPA, Hibernate)
42. **N+1 문제:** ORM 사용 시 발생하는 성능 저하 문제 및 해결 방안(Fetch Join, Entity Graph).
43. **영속성 컨텍스트(Persistence Context):** JPA에서 엔티티를 관리하는 논리적 개념.
44. **Dirty Checking:** JPA에서 엔티티 변경 감지를 통한 자동 업데이트.
45. **캐스케이드(Cascade):** 엔티티 상태 변화가 연관 엔티티에 전파되는 설정.
46. **고아 객체(Orphan Removal):** 부모 엔티티 제거 시 자식 엔티티 자동 제거.
47. **데이터베이스 트러블슈팅:** 느린 쿼리, 연결 오류 등 문제 해결.
48. **컬럼 타입 변경:** 기존 데이터 손실 없이 컬럼 타입 변경 전략.
49. **데이터베이스 클라우드 서비스:** AWS RDS, Azure SQL Database, Google Cloud SQL 등.
50. **DBMS 선택:** MySQL, PostgreSQL, Oracle 등 각 DBMS의 특징 및 장단점.

### **3. 운영체제 및 시스템 자원 관리**

1. **프로세스(Process):** 실행 중인 프로그램, 독립적인 메모리 공간.
2. **스레드(Thread):** 프로세스 내의 실행 흐름, 프로세스 자원 공유.
3. **멀티태스킹(Multitasking):** 여러 프로세스가 동시에 실행되는 것처럼 보이는 기술.
4. **멀티스레딩(Multithreading):** 하나의 프로세스 내에서 여러 스레드가 동시에 실행.
5. **동시성(Concurrency) vs 병렬성(Parallelism):** 개념과 차이점.
6. **CPU 스케줄링:** CPU 자원을 효율적으로 할당하는 방법(FCFS, SJF, Round Robin, 우선순위).
7. **메모리 관리:** 프로그램이 메모리를 효율적으로 사용할 수 있도록 관리하는 기법.
8. **가상 메모리(Virtual Memory):** 실제 물리 메모리보다 큰 메모리를 사용하는 것처럼 보이게 하는 기술.
9. **페이징(Paging):** 가상 메모리를 고정된 크기(페이지)로 나누어 관리.
10. **세그멘테이션(Segmentation):** 가상 메모리를 가변적인 크기(세그먼트)로 나누어 관리.
11. **스와핑(Swapping):** 메모리의 일부 내용을 디스크로 옮기는 기술.
12. **캐시 메모리(Cache Memory):** CPU와 주기억장치 사이의 빠른 임시 저장 공간.
13. **캐시 지역성(Locality of Reference):** 데이터 접근 패턴이 캐시 성능에 미치는 영향.
14. **IPC(Inter-Process Communication):** 프로세스 간 통신 방법(파이프, 소켓, 공유 메모리, 메시지 큐).
15. **동시성 문제(Concurrency Issues):** Race Condition (경쟁 조건), Deadlock (교착 상태).
16. **상호 배제(Mutual Exclusion):** 공유 자원에 대한 동시 접근 방지.
17. **뮤텍스(Mutex):** 상호 배제를 위한 동기화 메커니즘, 락(Lock) 방식.
18. **세마포어(Semaphore):** 자원 개수를 세는 동기화 메커니즘.
19. **모니터(Monitor):** 상호 배제와 조건 동기화를 지원하는 동기화 메커니즘.
20. **생산자-소비자 문제(Producer-Consumer Problem):** 동시성 문제 예시.
21. **임계 구역(Critical Section):** 공유 자원에 접근하는 코드 영역.
22. **I/O 효율성:** 버퍼링, 논블로킹 I/O, 비동기 I/O를 통한 I/O 성능 향상.
23. **파일 시스템(File System):** 파일 저장 및 관리 방식(FAT, NTFS, ext4 등).
24. **파일 디스크립터(File Descriptor):** 운영체제가 파일을 관리하는 정수 값.
25. **커널(Kernel):** 운영체제의 핵심 부분, 하드웨어 자원 관리.
26. **시스템 콜(System Call):** 사용자 프로그램이 커널 기능을 요청하는 방법.
27. **CPU 바운드(CPU-bound) vs I/O 바운드(I/O-bound):** 작업 특성에 따른 성능 병목 지점.
28. **스레드 풀(Thread Pool):** 스레드 생성/소멸 오버헤드를 줄여 자원 재사용.
29. **데몬(Daemon):** 백그라운드에서 실행되는 프로세스.
30. **환경 변수:** 시스템 전체 또는 특정 프로세스에 영향을 미치는 설정 값.
31. **권한 관리:** 파일, 디렉토리, 프로세스에 대한 접근 권한 제어.
32. **커널 스레드 vs 사용자 스레드:** 스레드 구현 방식의 차이.
33. **컨텍스트 스위칭(Context Switching):** CPU가 다른 프로세스/스레드로 전환 시 발생하는 오버헤드.
34. **가상화(Virtualization):** 하나의 물리적 하드웨어에서 여러 운영체제를 실행하는 기술.
35. **컨테이너화(Containerization):** Docker 등 격리된 환경에서 애플리케이션 실행.
36. **CGroup (Control Group):** 리눅스 커널 기능, 프로세스 그룹의 자원 사용량 제한 및 모니터링.
37. **네임스페이스(Namespace):** 리눅스 커널 기능, 프로세스에 격리된 환경 제공.
38. **SWAP 공간:** 물리 메모리가 부족할 때 디스크를 메모리처럼 사용하는 공간.
39. **페이지 교체 알고리즘:** FIFO, LRU, LFU, OPT 등.
40. **스레드 세이프티(Thread Safety):** 멀티스레드 환경에서 데이터 일관성을 유지하는 코드 작성.
41. **원자적 연산(Atomic Operations):** 중간에 중단되지 않고 한 번에 완료되는 연산.
42. **볼라타일(Volatile) 키워드:** 멀티스레드 환경에서 변수의 가시성 보장.
43. **JIT 컴파일러 (Just-In-Time Compiler):** Java 바이트코드를 런타임에 기계어로 변환하여 성능 향상.
44. **GC(Garbage Collection):** Java의 자동 메모리 관리. (Generational GC, Minor/Major GC)
45. **GC 튜닝:** GC 알고리즘(G1, CMS, Parallel), 힙(Heap) 크기 조절 등을 통한 성능 최적화.
46. **프로세스 동기화:** 생산자-소비자 문제, 식사하는 철학자 문제.
47. **커널 모드 vs 사용자 모드:** 권한에 따른 실행 모드.
48. **시스템 모니터링 도구:** `top`, `htop`, `free`, `iostat`, `vmstat` 등.
49. **파일 권한(Permission):** rwx (읽기, 쓰기, 실행) 개념.
50. **쉘 스크립트:** 자동화된 작업 처리 및 시스템 관리.

### **4. 네트워크 효율성 및 아키텍처**

1. **HTTP (Hypertext Transfer Protocol):** 웹 통신 프로토콜, Stateless 특징.
2. **TCP/IP (Transmission Control Protocol/Internet Protocol):** 인터넷 통신의 핵심 프로토콜 스위트.
3. **OSI 7계층 vs TCP/IP 4계층:** 각 계층의 역할과 기능.
4. **HTTP/1.1 vs HTTP/2:** 다중화(Multiplexing), 서버 푸시(Server Push), 헤더 압축 등 성능 개선 사항.
5. **RESTful API:** 자원 기반 설계, HTTP 메서드 활용, Stateless 원칙 준수.
6. **Stateless (무상태):** 서버가 클라이언트의 상태를 저장하지 않음. 확장성 및 부하 분산에 유리.
7. **Cacheable (캐시 가능):** HTTP 캐시 메커니즘을 통한 성능 향상.
8. **CORS (Cross-Origin Resource Sharing):** 다른 도메인 간 자원 공유를 위한 보안 정책.
9. **API Gateway:** 여러 마이크로서비스의 단일 진입점, 인증/인가, 로깅, 라우팅 등 기능 제공.
10. **마이크로서비스 아키텍처 (MSA):** 독립적인 배포 및 확장 용이, 각 서비스의 자원 효율화.
11. **로드 밸런싱 (Load Balancing):** 여러 서버에 요청 분산 (L4, L7), 시스템 부하 감소, 고가용성.
    - **프로젝트 연관:** `project_analysis_report.txt`의 "EC2 단독 사용" 문제점과 "Docker + ECS/Fargate 자동화 배포" 개선 방안과 연결될 수 있습니다.
12. **CDN (Content Delivery Network):** 정적 파일(이미지, CSS, JS)을 사용자 근처 서버에서 제공, 로딩 속도 향상.
13. **TCP 3-way Handshake & 4-way Handshake:** 연결 설정 및 해제 과정.
14. **UDP (User Datagram Protocol):** 비연결성, 비신뢰성, 빠른 통신. (스트리밍, 게임)
15. **HTTPS (HTTP Secure):** SSL/TLS를 통한 보안 통신.
16. **SSL/TLS 핸드셰이크:** 보안 연결 설정 과정.
17. **소켓 통신 (Socket Communication):** 클라이언트-서버 간 직접 통신.
18. **웹소켓 (WebSocket):** 양방향 실시간 통신 (채팅, 게임).
19. **서버리스 아키텍처 (Serverless Architecture):** 클라우드 공급자가 서버 인프라 관리, 사용량 기반 과금, 개발자가 코드만 집중.
20. **클라우드 컴퓨팅:** IaaS, PaaS, SaaS 개념 및 장단점.
21. **AWS EC2:** 가상 서버 인스턴스, 유연한 컴퓨팅 자원.
    - **프로젝트 연관:** `이력서.pdf`와 `project_analysis_report.txt`에 명시된 배포 환경.
22. **AWS RDS:** 관계형 데이터베이스 서비스, 관리형 DB, 자동 백업, 확장성.
    - **프로젝트 연관:** `이력서.pdf`와 `project_analysis_report.txt`에 명시된 DB 환경.
23. **VPC (Virtual Private Cloud):** AWS 내에서 격리된 가상 네트워크 환경.
24. **보안 그룹 (Security Group):** 인스턴스에 대한 네트워크 트래픽 제어.
25. **IAM (Identity and Access Management):** AWS 리소스에 대한 접근 권한 관리.
26. **도커 (Docker):** 컨테이너 기반 가상화, 애플리케이션 패키징 및 배포 표준화.
27. **컨테이너 오케스트레이션 (Container Orchestration):** 다수의 컨테이너 관리 (Kubernetes, AWS ECS/Fargate).
    - **프로젝트 연관:** `project_analysis_report.txt`의 "Docker + ECS/Fargate 자동화 배포" 개선 방안.
28. **CI/CD (Continuous Integration/Continuous Deployment):** 개발 효율성 및 배포 자동화.
29. **비동기 통신:** 논블로킹 I/O, 메시지 큐 (Kafka, RabbitMQ)를 활용한 시스템 간 느슨한 결합.
30. **분산 시스템(Distributed System):** 여러 컴퓨터에서 협력하여 하나의 작업을 수행.
31. **API 버전 관리:** API 변경 시 호환성 유지를 위한 전략.
32. **Gateway API:** 특정 API의 보안, 라우팅, 속도 제한 등을 관리.
33. **Reverse Proxy:** 클라이언트 요청을 서버로 전달, 보안 및 로드 밸런싱.
34. **Forward Proxy:** 클라이언트가 외부 네트워크에 접근할 때 경유.
35. **방화벽(Firewall):** 네트워크 보안, 트래픽 필터링.
36. **DDoS 방어:** 분산 서비스 거부 공격 방어 전략.
37. **세션(Session):** 클라이언트-서버 간 연결 상태 유지 (Stateless REST API와 대조).
38. **쿠키(Cookie):** 클라이언트 측에 저장되는 작은 데이터 조각.
39. **토큰(Token):** 인증/인가를 위한 경량화된 자격 증명. (JWT)
40. **OAuth 2.0:** 제3자 서비스 접근 권한 위임 표준.
    - **프로젝트 연관:** `이력서.pdf`와 `project_analysis_report.txt`에 Google OAuth2 연동 언급.
41. **OpenID Connect:** OAuth 2.0 위에 구축된 인증 레이어.
42. **메시지 브로커(Message Broker):** 비동기 통신을 위한 메시지 큐 시스템.
43. **서비스 디스커버리(Service Discovery):** 마이크로서비스 환경에서 서비스 위치를 찾는 메커니즘.
44. **서킷 브레이커(Circuit Breaker):** 분산 시스템에서 장애 전파 방지.
45. **폴리시(Policy):** 분산 시스템에서 실패한 요청을 자동으로 재시도.
46. **API 문서화:** Swagger/OpenAPI를 통한 API 명세.
47. **데이터 전송 포맷:** JSON, XML.
48. **gRPC:** 고성능 RPC 프레임워크, HTTP/2 기반, Protobuf 사용.
49. **웹 캐싱:** 브라우저, 프록시, CDN을 통한 웹 콘텐츠 캐싱.
50. **WebSocket vs HTTP Long Polling:** 실시간 통신 방식 비교.

15. **MyBatis:** SQL Mapper 프레임워크, 복잡한 쿼리 작성에 용이.
16. **스프링 시큐리티 (Spring Security):** 강력한 인증 및 인가 프레임워크.

18. **JWT (JSON Web Token):** 토큰 기반 인증/인가 방식. (Header, Payload, Signature)

46. **전략 패턴:** 알고리즘을 캡슐화하여 유연하게 변경.
47. **프록시 패턴:** 실제 객체에 대한 접근 제어.
48. **테스트 코드 (JUnit, Mockito, MockMvc):** 코드 품질 보증 및 리팩토링 용이성.
    - **프로젝트 연관:** `project_analysis_report.txt`의 "테스트 코드 미확인" 문제점.
49. **컨테이너화 (Docker, Docker Compose):** 개발/배포 환경 일관성 유지.
50. **메시지 큐 (Kafka, RabbitMQ):** 비동기 통신, 이벤트 기반 아키텍처.

### **6. 성능 최적화 및 모니터링**

1. **성능 지표(Metrics):** CPU 사용률, 메모리 사용량, I/O 대기 시간, 네트워크 대역폭, 응답 시간, 처리량, 동시 사용자 수, 에러율 등.
2. **병목 현상(Bottleneck):** 시스템 성능을 저하시키는 특정 자원 또는 코드 영역.
3. **프로파일링(Profiling):** 애플리케이션의 런타임 동작을 분석하여 성능 병목 지점을 식별하는 과정.
4. **프로파일링 도구:** JProfiler, VisualVM, YourKit, Spring Boot Actuator.
5. **APM(Application Performance Monitoring):** 애플리케이션 성능을 지속적으로 모니터링하고 분석하는 시스템. (Pinpoint, New Relic, Dynatrace)
6. **로그 분석:** Logback, ELK Stack (Elasticsearch, Logstash, Kibana) 등을 통한 로그 수집 및 분석.
    - **프로젝트 연관:** `member_service_analysis.txt`의 "예외 로그 ELK 연동 고려" 개선안.
7. **분산 추적(Distributed Tracing):** 분산 시스템에서 요청의 흐름을 추적하여 성능 문제 진단. (Zipkin, Jaeger)
8. **경쟁 조건(Race Condition) 진단:** 멀티스레드 환경에서 공유 자원 접근 문제 진단.
9. **데드락(Deadlock) 진단:** 스레드 덤프(Thread Dump) 분석.
10. **메모리 누수(Memory Leak) 진단:** 힙 덤프(Heap Dump) 분석, 메모리 프로파일러 사용.
11. **GC 로그 분석:** 가비지 컬렉션의 효율성 및 잠재적 문제 파악.
12. **벤치마킹(Benchmarking):** 특정 기능 또는 시스템의 성능을 표준화된 방법으로 측정. (JMH)
13. **스트레스 테스트(Stress Test):** 시스템이 최대 부하를 견딜 수 있는지 확인.
14. **부하 테스트(Load Test):** 예상되는 사용자 부하에서 시스템 성능 측정.
15. **성능 테스트(Performance Test):** 응답 시간, 처리량 등 성능 지표 측정.
16. **스케일 업(Scale Up):** 서버의 물리적 자원(CPU, RAM) 증설.
17. **스케일 아웃(Scale Out):** 서버 대수를 늘려 부하 분산.
18. **수평 확장(Horizontal Scaling):** 스케일 아웃과 동일, 분산 시스템에서 주로 사용.
19. **모니터링 대시보드:** Grafana, Prometheus 등을 활용한 시각화.
20. **경고/알림 시스템(Alerting System):** 성능 임계치 초과 시 자동 알림.
21. **실시간 모니터링:** 시스템의 현재 상태를 실시간으로 파악.
22. **예측적 모니터링:** 과거 데이터를 기반으로 미래 성능 예측.
23. **쿼리 모니터링:** 데이터베이스 쿼리 성능 분석.
24. **JVM 모니터링:** JVM 내부 상태(힙 사용량, 스레드 수) 모니터링.
25. **WAS 모니터링:** 웹 애플리케이션 서버의 성능 지표 모니터링.
26. **네트워크 모니터링:** 트래픽, 대역폭, 지연 시간 모니터링.
27. **디스크 I/O 모니터링:** 디스크 읽기/쓰기 성능 분석.
28. **CPU 사용률 모니터링:** CPU 사용 패턴 분석.
29. **메모리 사용률 모니터링:** 물리/가상 메모리 사용량 분석.
30. **오토 스케일링(Auto Scaling):** 클라우드 환경에서 트래픽에 따라 자동으로 서버 증감.
31. **서버 자원 할당 최적화:** CPU 코어 수, 메모리 크기 등 적절한 할당.
32. **JVM 힙 덤프(Heap Dump) 분석:** 메모리 누수 원인 파악.
33. **스레드 덤프(Thread Dump) 분석:** 애플리케이션의 현재 스레드 상태, 데드락 파악.
34. **JMX (Java Management Extensions):** Java 애플리케이션 모니터링 및 관리.
35. **GC 튜닝 파라미터:** `Xms`, `Xmx`, `XX:NewRatio` 등.
36. **Keep-Alive:** HTTP 연결 재사용으로 TCP 핸드셰이크 오버헤드 감소.
37. **Connection Pool 튜닝:** 최대 연결 수, 최소 연결 수, 유휴 시간 등 설정.
38. **SQL Cache:** 데이터베이스 쿼리 결과 캐싱.
39. **Prepared Statement Cache:** JDBC 드라이버에서 PreparedStatement 재사용.
40. **바이트코드 최적화:** 불필요한 바이트코드 생성 방지.
41. **JIT 컴파일러 옵션:** 성능 향상을 위한 JIT 컴파일러 설정.
42. **AIO (Asynchronous I/O):** I/O 작업을 비동기로 처리하여 메인 스레드 블로킹 방지.
43. **Event-driven Architecture:** 이벤트 기반으로 시스템 구성, 높은 확장성과 반응성.
44. **Bulk Insert/Update:** 대량 데이터 처리 시 성능 향상.
45. **Connection Draining:** 서버 종료 시 진행 중인 요청 안전하게 처리 후 연결 해제.
46. **Graceful Shutdown:** 서비스 중단 시 진행 중인 요청 처리 후 정상 종료.
47. **Health Check:** 시스템 상태를 주기적으로 확인하여 문제 발생 시 트래픽 분산 중단.
48. **Circuit Breaker Pattern:** 분산 시스템에서 서비스 장애가 전체로 확산되는 것을 방지.
49. **Timeout 설정:** 응답 지연으로 인한 자원 소모 방지.
50. **Rate Limiting:** 특정 기간 동안 요청 수를 제한하여 서비스 과부하 방지.
