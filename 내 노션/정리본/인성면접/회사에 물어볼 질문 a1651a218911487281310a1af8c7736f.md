# 회사에 물어볼 질문

1. **솔루션 아키텍처와 기술 스택에 관한 질문:**
    - "회사에서 사용하고 있는 솔루션의 기술 스택은 무엇인가요?" → 이미 나와 있음
    - "솔루션의 아키텍처와 모듈화에 대해 설명해주실 수 있나요?" → 물어볼 만 한듯
2. **통합 관리와 연동에 대한 이해:**
    - "솔루션은 어떻게 다양한 전자상거래 시스템과 통합되고 있나요?" →
    - "외부 시스템과의 효과적인 연동을 위해 어떤 접근 방식이나 도구를 사용하고 계신가요?" → 이건 진짜 좋은 듯
3. **보안 및 데이터 처리에 대한 질문:**
    - "개발 프로세스에서 보안 측면을 어떻게 고려하고 계신가요?" → 좋은 듯
    - "클라이언트의 데이터를 어떻게 안전하게 처리하고 있나요?" → 좋은 듯
4. **고객 요구사항과 협업에 대한 질문:**
    - "고객이나 클라이언트의 요구사항을 수집하고 구현하는 과정에 대해 설명해주세요."
    - "개발팀과 다른 부서들 간의 원활한 협업이 이루어지고 있나요?"
5. **새로운 기능 도입과 기술 향상에 관한 질문:**
    - "새로운 기술이나 기능을 도입하는 데 회사가 어떤 접근 방식을 가지고 있나요?"
    - "개발자들이 지속적으로 기술 습득과 성장을 할 수 있도록 회사가 어떤 지원을 제공하나요?"
6. **사용자 경험(UX) 및 성능 최적화에 관한 질문:**
    - "솔루션의 사용자 경험을 개선하기 위해 어떠한 노력을 기울이고 있나요?"
    - "성능 최적화와 관련된 개발적인 고려 사항이 어떤 것들이 있나요?"
7. **프로젝트 관리 및 일정에 관한 질문:**
    - "개발 프로젝트의 일정과 관리 방식에 대해 설명해주실 수 있나요?"
    - "프로젝트에서 발생하는 문제를 해결하는 데 어떤 절차를 따르고 있나요?"
