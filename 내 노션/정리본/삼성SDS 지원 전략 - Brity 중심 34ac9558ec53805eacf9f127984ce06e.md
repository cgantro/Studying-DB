# 삼성SDS 지원 전략 - Brity 중심

## 1. 지원 방향 한 줄 정리

나는 삼성SDS에 지원하지만, 그중에서도 **Brity 계열 협업 솔루션** 쪽이 가장 잘 맞는다고 정리한다.

## 2. 왜 Brity인가

### 내가 맞는 이유

- 여러 사용자가 동시에 쓰는 환경에서 상태가 어긋나지 않도록 구조를 설계하는 데 강점이 있다.
- 단순 기능 추가보다 **안정적으로 동작하는 구조**, **데이터 흐름**, **병목 완화**, **운영 가능한 형태**를 중요하게 생각한다.
- Brity 같은 협업 솔루션은 단순 채팅 기능이 아니라 여러 기능이 동시에 얽히므로 이런 역량이 중요하다.

### Brity에서 끌리는 포인트

- 기업용 협업 솔루션
- 여러 사용자가 동시에 쓰는 환경
- 메신저, 메일, 미팅, 문서, AI 기능이 연결되는 구조
- B2B 서비스 특성상 안정성, 권한, 운영성, 신뢰성이 중요함

## 3. Brity 기준으로 내 경험 연결

### 실시간성/정합성

- AutowingCar에서 ACK 기반 상태 전이, Redis 분리 등을 통해 상태 불일치와 병목을 줄인 경험
- 여러 흐름이 비동기로 들어오는 환경에서 **신뢰할 수 있는 상태 관리** 경험이 있음

### 성능/구조 개선

- RobotPal에서 병목 구간을 분석해 구조를 비동기로 재설계한 경험
- 단순히 느리다에서 끝나지 않고, 어디가 병목인지 분리해서 개선함

### 협업/소통

- 밴드 활동, 공연 준비, 장소 섭외, 조율 경험
- 또래상담/실습코치 경험
- 협업 솔루션 회사에 맞게 **사람 간 소통을 돕는 도구**에 관심이 있다는 흐름으로 연결 가능

## 4. Brity 지원 시 강조할 메시지

- 저는 여러 사용자가 동시에 쓰는 환경에서 데이터가 끊기지 않고 상태가 일관되게 유지되도록 구조를 설계하는 일에 강점이 있습니다.
- 그래서 실시간성과 협업성이 중요한 Brity 같은 솔루션에서, 기능 추가 자체보다도 실제 업무 환경에서 안정적으로 동작하는 구조를 만드는 데 기여하고 싶습니다.
- 특히 AI가 협업 솔루션에 붙는 시대에는 단순히 기능이 많아지는 것이 아니라, 정보가 정확히 연결되고 사용자가 신뢰할 수 있는 경험을 만드는 것이 더 중요하다고 생각합니다.

## 5. 예상 질문과 답변 방향

### 왜 삼성SDS인가

- B2B 서비스
- 실제 기업 업무 환경을 다루는 제품
- 안정성/운영성/확장성이 중요한 시스템
- 내 경험과 맞닿아 있음

### 왜 Brity인가

- 협업 솔루션은 여러 사용자가 동시에 쓰며, 기능 간 연결성과 데이터 정합성이 중요함
- 내가 해온 프로젝트도 결국 실시간 데이터 흐름과 상태 일관성을 다룬 경험임

### 입사 후 하고 싶은 일

- Brity 계열 서비스에서 협업 흐름이 끊기지 않도록 만드는 개발
- AI 기능이 들어가더라도 실제 업무에 신뢰성 있게 연결되는 구조 설계
- 안정성, 운영성, 성능 개선에 기여

### 기대와 다른 일을 맡게 되면

- Brity를 희망하지만, 삼성SDS 전체 사업 구조 안에서 필요한 업무를 이해하고 빠르게 적응할 수 있음
- 다만 장기적으로는 협업 솔루션/실시간 사용자 서비스 쪽에서 강점을 더 크게 발휘하고 싶음

- 즉, 물류/스마트팩토리 내용은 삭제가 아니라 **비교군**이다.
