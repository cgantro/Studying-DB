# 1분 자기소개

안녕하세요. 삼성SDS 지원자 홍윤표입니다.
저는 효율과 안정성을 함께 고려하며 구조를 개선하는 개발자입니다.

두 가지 프로젝트를 통해 이런 역량을 쌓아 왔습니다. 자율 토잉카 프로젝트에서는 실시간 텔레메트리 처리 구조를 개선해 500대 규모 환경에서도 종단 간 전송 지연을 14~28ms 수준으로 유지했습니다. 또 시뮬레이션 프로젝트에서는 병목 구간을 분석하고 처리 구조를 분리해 프레임 드랍률을 약 90% 줄였고, 실제 환경에서도 5% 이상의 FPS 개선을 이끌어냈습니다.

이런 경험을 통해 저는 문제를 구조적으로 보고 성능과 안정성을 함께 개선하는 역량을 쌓아 왔습니다. 이를 바탕으로 삼성SDS에서도 안정적으로 운영되는 시스템과 플랫폼 개발에 기여하겠습니다.

1. 실시간 데이터 흐름을 구체적으로 어떻게?
    - 딥뱐
        
        제가 했던 프로젝트들은 공통적으로
        
        **실시간으로 들어오는 데이터를 바로 반영해야 하는 환경** 이었습니다.
        
        자율 토잉카 프로젝트에서는 차량 상태 데이터를 받아 관제 화면에 빠르게 보여줘야 했고, 시뮬레이션 프로젝트에서는 렌더링 결과를 스트리밍하는 과정에서 처리 지연이 생기면 바로 프레임 드랍으로 이어졌습니다.
        
        그래서 저는 단순히 데이터를 받는 것보다,
        
        **어떤 흐름은 바로 반영하고 어떤 흐름은 분리해서 처리해야 안정적으로 동작하는지**
        
        를 고민해왔습니다. 그 경험 때문에 실시간 처리에서는 속도 자체보다도
        
        **상태가 꼬이지 않고 지속적으로 운영될 수 있는 구조**가 중요하다고 생각하게 됐습니다.
        
2. 본인이 생각하는 효율이란?
    - 답
        
        제가 말하는 효율은 단순히 **빨리 돌아가는 것**만 의미하지는 않습니다.
        
        저는 효율을 **같은 자원으로 더 안정적으로 처리하고, 병목이나 불필요한 대기를 줄여 실제 운영 품질을 높이는 것**이라고 생각합니다.
        
        예를 들어 실시간 시스템에서 응답 속도가 빨라도 상태가 꼬이거나, 특정 구간이 막혀 전체 흐름이 흔들리면 그건 좋은 효율이라고 보기 어렵습니다.
        
        그래서 저는 속도와 함께 안정성, **처리 흐름, 자원 사용 방식**까지 같이 보는 편입니다.
        
3. 스마트팩토리나 MES를 직접 해본 건 아닌데, 그 점은 괜찮을까요?
    - 답
        
        직접적인 제조 도메인 경험이 많다고 말하기는 어렵습니다.
        
        다만 저는 실시간 데이터가 들어오고, 그 데이터를 안정적으로 반영해야 하며, 운영 중에도 신뢰할 수 있어야 하는 시스템을 여러 번 다뤘습니다.
        
        MES나 AHMS도 결국 현장 데이터를 실행 시스템과 연결하는 구조라고 생각하기 때문에, 도메인은 빠르게 배우고 제 강점인
        
        **실시간 처리와 상태 관리 관점**
        
        으로 기여할 수 있다고 생각합니다.

### 2안

안녕하세요, 삼성SDS 지원자 홍윤표입니다.

저는 **기준을 분명히 세우되, 사람들과 호흡을 맞추는 과정을 중요하게 생각하는 사람**입니다.

대학 시절 밴드 활동을 오래 했고, 밴드 마스터 역할도 맡았습니다. 합주를 하다 보면 각자 연주 스타일도 다르고 실력 차이도 있어서, 그냥 제 생각만 밀어붙이면 분위기도 무너지고 결과도 잘 나오지 않았습니다. 그래서 저는 어떤 부분이 맞지 않는지 먼저 기준을 정리하고, 그걸 상대가 받아들일 수 있는 방식으로 전달하려고 많이 노력했습니다.

그 과정을 겪으면서 좋은 결과는 혼자 잘해서 나오는 게 아니라, 서로 다른 사람들을 같은 방향으로 움직이게 할 때 나온다는 걸 배웠습니다. 지금도 개발할 때 문제를 구조적으로 보는 편이지만, 결국 협업 안에서 기준을 맞추고 흐름을 정리하는 태도는 이 경험에서 많이 만들어졌다고 생각합니다.
