### **Why LG CNS**

```html
LG CNS에 지원하게 된 계기는 단순히 큰 회사여서가 아니라, '기술로 세상을 바꾼다'는 철학을 실제로 구현하고 있다고 느꼈기 때문입니다. 
특히 세종스마트시티 같은 프로젝트를 보면서, 저도 개발자로서 사람들의 일상을 더 편리하게 만드는 의미 있는 일을 하고 싶다는 생각이 확고해졌습니다.

입사 후 제가 꿈꾸는 모습은 두 가지입니다. 
첫째, 고객이 진짜 필요로 하는 것이 무엇인지 정확히 파악하고, 그것을 기술로 해결할 수 있는 개발자가 되고 싶습니다. 
둘째, 언젠가는 사람들에게 "이 시스템 덕분에 정말 편해졌어"라는 말을 들을 수 있는, 실질적인 변화를 만드는 사람이 되는 것입니다. 
```

**경험**

```html
저는 학부 시절, 전공 과목들을 단순히 이론에 머무르지 않고 직접 구현하며 익히는 데 집중했습니다.

자료구조 강의에서 연결 리스트, 스택, B+트리를 직접 구현하면서 자료구조가 내부적으로 어떻게 동작하는지 명확히 이해할 수 있었습니다. 이 과정은 단순한 과제를 넘어서 데이터베이스 인덱스 최적화처럼 실제 백엔드 성능을 개선하는 원리를 이해하는 데 큰 도움이 되었습니다.

운영체제 수업에서는 원형 연결 리스트와 LRU 알고리즘을 활용해 버퍼 캐시 관리 프로그램을 만들었습니다. 이 경험을 통해 제한된 메모리를 효율적으로 다루는 방법을 배웠고, 이는 향후 안정적인 캐싱 설계를 고민하는 데 기반이 될 것이라 생각합니다.

저는 항상 전공 과제에서 배운 내용을 실제 개발에 어떻게 활용할 수 있을지 고민해 왔습니다. LG CNS의 DX 백엔드 개발자로서, 지금까지 쌓아온 지식을 바탕으로 시스템 성능을 높이고 효율적인 솔루션을 만들어 기여하고 싶습니다.
```

**개발 프로젝트 경험**

```html
팀 프로젝트에서 해외 거주자에게 한국 간식을 소개하는 플랫폼의 백엔드 개발을 맡았습니다. Java 11, Spring Boot, Spring Security로 시스템을 구축했고, 인프라는 AWS RDS와 EC2를 기반으로 구성했습니다.

저는 사용자 인증·인가 기능 구현, 데이터베이스 인프라 설계, 성능 테스트를 담당했습니다. AWS RDS 기반 MySQL 환경을 구축하고 회원가입·로그인 기능을 개발했으며, JWT 기반 인증과 Google OAuth2 소셜 로그인 연동을 통해 핵심 인증 로직을 직접 구현했습니다.

개발 과정에서 로그인 요청 처리 속도가 점점 느려지는 문제가 있었습니다. DB와 인증 서버가 분리된 구조에서 로그인 시마다 DB에 접근해야 했기 때문에, 동시 접속자가 늘어날수록 응답 속도가 저하되는 상황이었습니다.

이를 해결하기 위해 MySQL 인덱스와 쿼리를 최적화해 DB 부하를 줄였고, Redis를 캐싱 시스템으로 도입해 자주 조회하는 정보를 메모리에 저장했습니다. 또한 Spring Security 필터와 Redis를 연동해 만료·갱신 토큰을 안정적으로 관리하는 로직을 구현했습니다.

아울러, 제가 직접 k6를 활용해 부하 테스트를 설계하고 실행했습니다. 시나리오 기반으로 동시 접속자가 늘어날 때의 응답 속도와 서버 안정성을 검증했고, 성능 개선 효과를 수치로 확인할 수 있었습니다.

그 결과 로그인 API의 평균 응답 속도를 500ms에서 150ms로 줄였고, 50명 이상이 동시에 접속하는 환경에서도 안정적인 인증 처리를 유지할 수 있었습니다. 이 경험을 통해 인증·인가 기능 개발, DB 인프라 설계, 캐싱 기반 성능 개선, 성능 테스트 도구를 활용한 안정성 검증 역량을 모두 직접 체득할 수 있었습니다.
```
