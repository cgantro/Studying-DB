# 1️⃣ 전체적인 자기소개 (약 750~800자 / 70~80%)

서버와 시스템은 사용자에게 직접 드러나지는 않지만, 서비스의 안정성과 신뢰성을 결정하는 기반이라고 생각합니다. 저는 시스템 개발 경험이 길지는 않지만, 이러한 기반 영역을 이해하고 책임질 수 있는 개발자가 되기 위해 기초부터 차근차근 역량을 쌓아왔습니다.

광운대학교에서 소프트웨어를 전공하며 자료구조, 운영체제, 시스템 프로그래밍 과목을 통해 프로세스 구조와 메모리 관리 등 시스템의 기본 동작 원리를 학습했습니다. 이후 로봇 시뮬레이터 프로젝트에서 C++ 기반 애플리케이션을 설계하며, 기능 구현에 그치지 않고 성능 저하의 원인을 분석하고 개선하는 경험을 했습니다. 이미지 인코딩 과정에서 처리 흐름을 점검하고 라이브러리를 교체하며 멀티 스레딩을 적용해 성능을 개선한 경험은, 문제를 구조적으로 바라보는 개발 관점을 갖게 한 계기였습니다.

또한 Spring 기반 서버 개발과 AWS 환경에서의 배포 경험을 통해 서버 애플리케이션이 실제로 운영되는 흐름을 이해할 수 있었습니다. 현재는 SSAFY에서 임베디드 교육을 이수고 있습니다. 아직 배워야 할 부분이 많지만, 기본을 소홀히 하지 않는 자세로 안정적인 서버 개발자로 성장하고자 합니다.

# 2️⃣ 성장과정 (약 600자)

어릴 때부터 결과보다 과정에 더 관심이 많았습니다. 무언가가 잘 되면 왜 그렇게 동작하는지, 문제가 생기면 어디에서부터 꼬였는지를 알고 싶어 했습니다. 컴퓨터를 사용하면서도 단순히 쓰는 데서 그치지 않고, 직접 분해해 보거나 고장이 난 이유를 찾아보려 했습니다. 이런 경험을 반복하며 빠르게 답을 얻는 것보다 원리를 이해하는 과정이 더 중요하다고 느끼게 되었고, 자연스럽게 소프트웨어를 전공으로 선택하게 되었습니다.

고등학교 시절부터 약 10년간 밴드 활동을 하며 하나의 결과물이 완성되기까지의 과정을 꾸준히 경험했습니다. 합주를 하다 보면 음의 균형이 맞지 않거나 흐름이 어긋나는 경우가 많았는데, 그럴 때마다 감각에만 의존하기보다 무엇이 문제인지부터 정리하려 했습니다. 파트별로 의견을 나누고 연습 방식을 조정하며, 시간이 걸리더라도 끝까지 맞춰 나가는 경험을 통해 문제를 외면하지 않는 태도를 기를 수 있었습니다.

이러한 태도는 대학 전공 수업과 프로젝트를 수행하면서도 이어졌습니다. 오류나 성능 문제를 마주했을 때 단순히 동작하게 만드는 데서 멈추기보다, 왜 문제가 발생했는지를 확인하고 해결하려는 습관이 자리 잡았습니다. 이러한 경험은 안정성이 중요한 서버 개발 직무에서도 꾸준히 도움이 될 수 있는 기반이라고 생각합니다.

# 3️⃣ 성격의 장단점 / 가치관 (약 600자)

저의 장점은 문제를 구조와 근거를 중심으로 해결하려는 태도입니다. 오류나 성능 저하가 발생했을 때 추측에 의존하기보다 수치, 코드 흐름을 기준으로 원인을 분해하며 접근합니다. 로봇 시뮬레이터 프로젝트에서 프레임 저하 문제가 발생했을 때도 병목 구간을 분석한 뒤 라이브러리 교체와 멀티스레딩 적용을 통해 성능을 개선한 경험이 있습니다. 이 과정에서 문제를 차분히 분석하고 검증하는 개발 방식이 몸에 배게 되었습니다.

반면, 의견을 전달할 때 다소 직설적으로 표현하는 경향이 있었습니다. 밴드 활동 중 피드백을 주고받는 과정에서 의도와 달리 전달이 거칠게 느껴질 수 있다는 점을 인식하게 되었고, 이후 표현 방식을 조절하려 노력했습니다. 또한 수학학원에서 초·중등 학생을 가르치며 상대의 이해 수준에 맞춰 말을 순화하고 설명하는 연습을 지속했습니다. 이러한 경험을 통해 현재는 정확성을 유지하면서도 상대를 고려한 소통을 중요하게 생각하고 있습니다.

# 4️⃣ 학교생활 (약 600자)

학교생활 동안 전공 수업과 프로젝트를 통해 소프트웨어 전반에 대한 기초 체력을 쌓았습니다. 운영체제, 자료구조, 시스템 프로그래밍 수업을 통해 프로세스 구조, 메모리 관리, 동기화 개념을 학습했으며, 단순 이론에 그치지 않고 직접 구현하며 이해하고자 노력했습니다. 이러한 학습 방식은 이후 프로젝트 수행 시 구조를 먼저 고민하는 습관으로 이어졌습니다.

또한 6년간 밴드 동아리에서 활동하며 밴드 마스터 역할을 맡아 공연 기획, 일정 조율, 구성원 간 의견 조정을 담당했습니다. 팀 단위 결과물의 완성도를 책임지며 다양한 성향의 구성원들과 협업하는 과정에서 소통과 조율의 중요성을 체감했습니다. 특히 타 대학 팀과 함께 자작곡을 제작해 가요제에 참가하며 의견을 조율하고 완성도를 높인 끝에 대상을 수상한 경험은, 목표를 공유하고 끝까지 완성해 나가는 태도의 중요성을 깨닫는 계기가 되었습니다.

# 5️⃣ 지원동기 및 포부 (약 600자)

보안 솔루션은 눈에 잘 띄지 않더라도, 문제가 없을 때 당연하게 잘 동작하는 것이 가장 중요하다고 생각합니다. 저는 사용자에게 직접 드러나지 않는 서버 영역에서 이러한 안정성을 지켜 나가는 역할에 매력을 느껴 PNP시큐어에 지원하게 되었습니다.

프로젝트를 진행하며 기능을 빠르게 구현하는 것보다, 코드가 어떤 흐름으로 동작하고 있는지를 먼저 이해하려 노력해 왔습니다. 문제가 발생했을 때도 감에 의존해 판단하기보다 로그와 실행 흐름을 하나씩 따라가며 원인을 확인하는 방식에 익숙해졌습니다. 이러한 경험을 통해 개발 과정에서 무엇을 우선적으로 확인해야 하는지에 대한 나름의 기준을 세울 수 있었습니다.

입사 후에는 기존 시스템과 코드의 맥락을 충분히 이해하는 데 집중하고 싶습니다. 작은 수정이라도 왜 필요한지, 어디에 영향을 줄 수 있는지를 먼저 고민하며 신중하게 접근하겠습니다. 장기적으로는 서버 구조와 보안 요구사항을 함께 이해하는 개발자로 성장해, PNP시큐어의 솔루션이 안정적으로 운영되는 데 기여하고 싶습니다.
