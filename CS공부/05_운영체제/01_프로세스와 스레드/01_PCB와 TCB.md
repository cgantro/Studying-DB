# PCB와 TCB

## 한 줄 정의
[[용어 모음집/PCB|PCB]](Process Control Block)는 **[[용어 모음집/프로세스|프로세스]]를 관리하기 위해 [[용어 모음집/운영체제|운영체제]]가 저장하는 정보 묶음**이고, [[용어 모음집/TCB|TCB]](Thread Control Block)는 **[[용어 모음집/스레드|스레드]]를 관리하기 위해 저장하는 정보 묶음**이다.

## PCB에 들어가는 정보
- 프로세스 ID(PID)
- 프로세스 상태: ready, running, waiting 등
- 프로그램 카운터(PC)
- [[용어 모음집/CPU|CPU]] [[용어 모음집/레지스터|레지스터]] 값
- 메모리 관리 정보: [[용어 모음집/페이지 테이블|페이지 테이블]], 주소 공간 정보
- 열린 파일 정보
- 우선순위와 [[용어 모음집/스케줄링|스케줄링]] 정보
- 부모/자식 프로세스 관계

## TCB에 들어가는 정보
- 스레드 ID
- 스레드 상태
- 프로그램 카운터와 레지스터 값
- 스택 포인터
- 스케줄링 정보

스레드는 같은 프로세스 안에서 코드, 데이터, 힙, 파일 정보를 공유하지만 각자 실행 흐름이 다르므로 PC, 레지스터, 스택은 따로 관리해야 한다.

## 예시
프로세스 안에 스레드 3개가 있으면 PCB는 프로세스 전체 자원 정보를 들고, 각 스레드의 TCB는 실행 위치와 스택 정보를 따로 가진다.

```text
Process
├─ PCB: 주소 공간, 열린 파일, 프로세스 상태
├─ TCB 1: PC, register, stack
├─ TCB 2: PC, register, stack
└─ TCB 3: PC, register, stack
```

## 문맥 교환과의 관계
[[용어 모음집/문맥 교환|문맥 교환]]이 일어나면 운영체제는 현재 실행 중인 프로세스나 스레드의 레지스터 값, PC 등을 PCB/TCB에 저장하고, 다음 실행 대상의 PCB/TCB에서 값을 복원한다.

## 자주 헷갈리는 점
- PCB는 프로세스 단위 정보, TCB는 스레드 단위 정보다.
- 스레드는 자원을 공유하지만 실행 흐름은 독립적이므로 TCB가 필요하다.
- 문맥 교환 비용은 PCB/TCB 저장과 복원, [[용어 모음집/캐시|캐시]] 영향까지 포함해 생각해야 한다.

## 면접 포인트
PCB는 프로세스의 실행 상태와 자원 정보를 저장하는 [[용어 모음집/커널|커널]] 자료구조이고, TCB는 스레드의 실행 상태를 저장한다. 문맥 교환 시 이 정보들이 저장되고 복원된다.

## 보강 설명
프로세스와 스레드 주제는 실행 단위와 자원 소유 단위를 구분하는 것이 중요하다. 프로세스는 주소 공간과 자원을 갖고, 스레드는 그 안에서 CPU를 받는 실행 흐름으로 동작한다.

## 구체적인 예시
예를 들어 브라우저는 여러 프로세스로 탭을 분리하고, 각 프로세스 안에서 렌더링이나 네트워크 작업을 여러 스레드로 나눌 수 있다. 문맥 교환 때는 CPU 상태가 저장되고 다른 실행 흐름의 상태가 복원된다.

- 이름이 비슷한 용어는 “무엇을 관리하는가”, “언제 발생하는가”, “어떤 비용을 줄이는가”로 구분한다.
