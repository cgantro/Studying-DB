# 버퍼 캐시 스풀링

## 한 줄 정의
버퍼, [[용어 모음집/캐시|캐시]], [[용어 모음집/스풀링|스풀링]]은 **느린 [[용어 모음집/입출력 장치|입출력 장치]]와 빠른 프로그램 사이의 속도 차이를 줄이고 작업을 안정적으로 처리하기 위한 [[용어 모음집/운영체제|운영체제]] 기법**이다.

---

## 핵심
- 버퍼는 일시적으로 데이터를 모아 둔다
- 캐시는 자주 쓰는 데이터를 다시 빠르게 쓰기 위해 보관한다
- 스풀링은 느린 장치 작업을 큐에 쌓아 순서대로 처리한다
- 세 기법 모두 I/O 비용을 줄이지만 의미가 다르다

---

## 버퍼
버퍼는 데이터가 이동하는 중간에 잠시 머무는 공간이다.

```text
프로그램 → 버퍼 → 장치
장치 → 버퍼 → 프로그램
```

### 왜 필요한가
- 생산자와 소비자의 속도 차이를 흡수한다
- 작은 쓰기를 모아 큰 쓰기로 바꿀 수 있다
- I/O 호출 횟수를 줄일 수 있다

---

## 캐시
캐시는 다시 쓸 가능성이 높은 데이터를 빠른 곳에 보관한다.

### 예시
- 파일을 다시 읽을 때 디스크 대신 페이지 캐시에서 읽음
- 자주 접근하는 메타데이터를 메모리에 보관

---

## 스풀링
스풀링은 느린 장치 작업을 바로 실행하지 않고 큐에 쌓아 순서대로 처리하는 방식이다.

```text
여러 프로그램의 출력 요청
   ↓
스풀 큐
   ↓
프린터가 순서대로 처리
```

대표 예시는 프린터 작업이다.

---

## 비교표

| 구분 | 목적 | 예시 |
|---|---|---|
| 버퍼 | 속도 차이 흡수 | 파일 쓰기 버퍼 |
| 캐시 | 재사용 성능 향상 | 페이지 캐시 |
| 스풀링 | 느린 장치 작업 큐잉 | 프린터 큐 |

---

## 주의할 점
- 버퍼에만 있고 실제 디스크에 쓰이지 않은 데이터는 장애 시 사라질 수 있다
- 캐시는 최신성과 일관성을 관리해야 한다
- 스풀링 큐가 밀리면 지연이 커진다

---

## 면접 포인트
> 버퍼는 데이터 이동 중 속도 차이를 흡수하는 임시 공간이고, 캐시는 자주 쓰는 데이터를 빠르게 재사용하기 위한 저장 공간입니다. 스풀링은 프린터처럼 느린 장치 작업을 큐에 쌓아 순서대로 처리하는 방식입니다. 셋 다 I/O 성능과 안정성을 위해 쓰이지만 목적이 다릅니다.

---

## 체크리스트
- [ ] 버퍼와 캐시 차이를 설명할 수 있다
- [ ] 스풀링이 어떤 상황에서 필요한지 안다
- [ ] 버퍼링과 장애 시 데이터 손실 가능성을 연결할 수 있다
- [ ] 페이지 캐시가 파일 I/O 성능에 영향을 준다는 점을 안다
---

## 보강 설명
캐시 주제는 “빠른 [[용어 모음집/CPU|CPU]]와 느린 메모리 사이의 간격을 어떻게 줄이는가”가 핵심이다. [[용어 모음집/지역성|지역성]]이 좋은 접근은 캐시 적중률을 높이고, [[용어 모음집/캐시 미스|캐시 미스]]나 일관성 비용은 성능을 떨어뜨린다.

## 구체적인 예시
예를 들어 배열을 앞에서부터 순서대로 읽으면 한 캐시 라인에 들어온 주변 원소를 이어서 쓰기 때문에 빠르다. 반대로 큰 배열을 무작위로 접근하거나 여러 코어가 같은 캐시 라인을 계속 수정하면 미스와 일관성 트래픽이 늘어난다.
