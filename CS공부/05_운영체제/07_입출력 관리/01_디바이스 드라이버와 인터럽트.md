# 디바이스 드라이버와 인터럽트

## 한 줄 정의
[[용어 모음집/디바이스 드라이버|디바이스 드라이버]]는 **[[용어 모음집/운영체제|운영체제]]가 특정 하드웨어 장치를 제어할 수 있게 해 주는 [[용어 모음집/커널|커널]] 코드**이고, [[용어 모음집/인터럽트|인터럽트]]는 **장치가 운영체제에게 처리할 사건을 알리는 신호**다.

---

## 핵심
- 운영체제는 모든 장치의 세부 명령을 직접 알 수 없다
- 드라이버가 장치별 차이를 숨긴다
- 인터럽트는 I/O 완료와 오류를 알리는 데 사용된다
- 커널은 인터럽트를 받아 대기 중인 작업을 이어간다

---

## 드라이버가 하는 일
- 장치 초기화
- 장치 상태 확인
- 데이터 읽기/쓰기 요청 전달
- 인터럽트 처리
- 오류 처리
- 커널의 공통 I/O 인터페이스와 장치 연결

---

## I/O 완료 흐름

```text
프로세스가 read 요청
   ↓
커널이 드라이버 호출
   ↓
드라이버가 장치에 작업 지시
   ↓
프로세스는 대기 상태
   ↓
장치 작업 완료
   ↓
인터럽트 발생
   ↓
커널이 프로세스 깨움
```

---

## 인터럽트 처리에서 주의할 점
- 빠르게 처리해야 한다
- 긴 작업은 나중에 처리할 큐로 넘긴다
- 공유 자료구조 [[용어 모음집/동기화|동기화]]가 필요하다
- 너무 많은 인터럽트는 성능 저하를 만들 수 있다

---

## 자주 하는 실수
- 드라이버를 일반 사용자 프로그램처럼 생각함
  - 바로잡기: 디바이스 드라이버는 하드웨어를 제어하는 커널 수준 코드이며, 일반 사용자 프로그램보다 시스템 안정성에 직접 영향을 준다.
- 인터럽트가 오면 항상 즉시 모든 처리를 끝낸다고 생각함
  - 바로잡기: 인터럽트 핸들러는 빠르게 필요한 상태만 처리하고, 오래 걸리는 작업은 커널 작업 큐나 후속 처리로 넘겨 시스템 응답성을 유지한다.
- 장치별 차이를 운영체제 본체가 모두 직접 처리한다고 생각함
  - 바로잡기: 운영체제는 공통 인터페이스를 제공하고, 장치별 세부 제어는 드라이버가 담당한다.
- 드라이버 버그가 시스템 전체 안정성에 영향을 줄 수 있음을 놓침
  - 바로잡기: 디바이스 드라이버는 하드웨어를 제어하는 커널 수준 코드이며, 일반 사용자 프로그램보다 시스템 안정성에 직접 영향을 준다.

---

## 면접 포인트
> 디바이스 드라이버는 장치별 제어 방식을 운영체제의 공통 인터페이스에 연결하는 커널 코드입니다. I/O 요청이 들어오면 드라이버가 장치에 작업을 지시하고, 장치가 완료되면 인터럽트로 커널에 알립니다. 커널은 이를 처리해 대기 중인 [[용어 모음집/프로세스|프로세스]]를 깨우거나 후속 작업을 진행합니다.

---

## 체크리스트
- [ ] 드라이버의 역할을 설명할 수 있다
- [ ] I/O 요청부터 완료 인터럽트까지 흐름을 안다
- [ ] 인터럽트 핸들러가 짧아야 하는 이유를 안다
- [ ] 드라이버가 커널 안정성과 연결됨을 안다
---

## 보강 설명
입출력 주제는 [[용어 모음집/CPU|CPU]]가 느린 장치와 어떻게 협력하는지 보는 영역이다. CPU가 계속 기다릴지, 장치가 인터럽트로 알릴지, [[용어 모음집/DMA|DMA]]로 데이터 이동을 맡길지에 따라 효율이 달라진다.

## 구체적인 예시
예를 들어 디스크에서 큰 파일을 읽을 때 CPU가 바이트마다 직접 옮기면 비효율적이다. DMA를 사용하면 장치가 메모리로 직접 데이터를 옮기고, 완료되면 인터럽트로 CPU에게 알린다.
