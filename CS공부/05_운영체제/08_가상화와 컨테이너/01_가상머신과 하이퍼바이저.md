# 가상머신과 하이퍼바이저

## 한 줄 정의
[[용어 모음집/가상머신|가상머신]]은 **물리 컴퓨터처럼 보이는 가상의 실행 환경**이고, [[용어 모음집/하이퍼바이저|하이퍼바이저]]는 **여러 가상머신이 하드웨어를 나눠 쓰도록 관리하는 계층**이다.

---

## 핵심
- VM은 별도 [[용어 모음집/운영체제|운영체제]]를 실행할 수 있다
- 하이퍼바이저가 [[용어 모음집/CPU|CPU]], 메모리, 디스크, 네트워크를 [[용어 모음집/가상화|가상화]]한다
- VM은 격리가 강하지만 상대적으로 무겁다
- 클라우드 인프라의 기본 단위로 자주 쓰인다

---

## 구조

```text
물리 하드웨어
   ↓
하이퍼바이저
   ├─ VM 1: Guest OS + App
   ├─ VM 2: Guest OS + App
   └─ VM 3: Guest OS + App
```

---

## 하이퍼바이저 역할
- VM 생성과 종료
- CPU 시간 배분
- 메모리 할당과 격리
- 가상 디스크와 네트워크 제공
- VM 간 격리 유지

---

## Type 1과 Type 2

| 구분 | 설명 |
|---|---|
| Type 1 | 하드웨어 위에서 직접 실행되는 하이퍼바이저 |
| Type 2 | 기존 운영체제 위에서 실행되는 하이퍼바이저 |

Type 1은 서버/클라우드 환경에서, Type 2는 개인 개발/테스트 환경에서 자주 떠올리면 쉽다.

---

## 장점과 단점

| 항목 | 내용 |
|---|---|
| 장점 | 강한 격리, 다른 OS 실행 가능, 복제/이동 편리 |
| 단점 | 부팅과 실행이 무겁고 자원 사용량이 큼 |

---

## 면접 포인트
> 가상머신은 하드웨어를 가상화해 독립된 컴퓨터처럼 동작하는 실행 환경이고, 하이퍼바이저는 여러 VM이 물리 자원을 나눠 쓰도록 관리합니다. VM은 각자 게스트 OS를 실행할 수 있어 격리가 강하지만 [[용어 모음집/컨테이너|컨테이너]]보다 무겁고 시작이 느릴 수 있습니다.

---

## 체크리스트
- [ ] VM이 별도 OS를 실행할 수 있음을 안다
- [ ] 하이퍼바이저 역할을 설명할 수 있다
- [ ] Type 1과 Type 2 차이를 이해적으로 안다
- [ ] VM의 장단점을 말할 수 있다
---

## 보강 설명
가상화와 컨테이너 주제는 하나의 물리 자원을 여러 격리된 실행 환경이 나눠 쓰게 하는 방법이다. VM은 하드웨어 수준 추상화가 강하고, 컨테이너는 [[용어 모음집/커널|커널]]을 공유하며 [[용어 모음집/프로세스|프로세스]] 환경을 격리한다.

## 구체적인 예시
예를 들어 VM은 게스트 OS를 따로 실행하지만 컨테이너는 호스트 커널을 공유한다. 컨테이너는 [[용어 모음집/namespace|namespace]]로 보이는 자원을 분리하고 [[용어 모음집/cgroup|cgroup]]으로 사용량을 제한한다.

## 헷갈릴 때 구분 기준
- 이름이 비슷한 용어는 “무엇을 관리하는가”, “언제 발생하는가”, “어떤 비용을 줄이는가”로 구분한다.
- 성능 관련 개념은 항상 병목과 오버헤드를 같이 본다.
- 운영체제 개념은 사용자 프로그램 관점과 커널 내부 관점을 나누어 설명한다.
