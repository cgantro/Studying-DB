# 컨테이너 namespace cgroup

## 한 줄 정의
[[용어 모음집/컨테이너|컨테이너]]는 **호스트 [[용어 모음집/커널|커널]]을 공유하면서 [[용어 모음집/프로세스|프로세스]] 실행 환경을 격리하는 방식**이고, [[용어 모음집/namespace|namespace]]와 [[용어 모음집/cgroup|cgroup]]은 **격리와 자원 제한을 담당하는 핵심 커널 기능**이다.

---

## 핵심
- 컨테이너는 VM처럼 별도 커널을 띄우지 않는다
- namespace는 프로세스가 보는 자원 범위를 분리한다
- cgroup은 [[용어 모음집/CPU|CPU]], 메모리, I/O 같은 자원 사용량을 제한하고 측정한다
- 컨테이너 이미지는 실행에 필요한 [[용어 모음집/파일 시스템|파일 시스템]] 묶음이다

---

## 컨테이너 구조

```text
호스트 OS 커널
   ├─ 컨테이너 A: 격리된 프로세스들
   ├─ 컨테이너 B: 격리된 프로세스들
   └─ 컨테이너 C: 격리된 프로세스들
```

각 컨테이너는 독립된 환경처럼 보이지만 커널은 공유한다.

---

## namespace
namespace는 프로세스가 보는 이름 공간을 분리한다.

| 구분 기준                         | 분리되는 것          |     |
| ----------------------------- | --------------- | --- |
| PID namespace                 | 프로세스 번호 공간      |     |
| Mount namespace               | 파일 시스템 마운트      |     |
| Network namespace             | 네트워크 인터페이스/라우팅  |     |
| [[용어 모음집/IPC\|IPC]] namespace | 프로세스 간 통신 자원    |     |
| UTS namespace                 | hostname 같은 식별자 |     |
| User namespace                | 사용자/권한 매핑       |     |

---

## cgroup
cgroup은 자원 사용을 제한하고 관찰한다.

- CPU 사용량 제한
- 메모리 사용량 제한
- I/O 대역폭 제한
- 프로세스 그룹 단위 통계 수집

---

## VM과 다른 점

| 항목  | 컨테이너      | VM                |          |
| --- | --------- | ----------------- | -------- |
| 커널  | 호스트 커널 공유 | 각 VM이 별도 커널       |          |
| 시작  | 빠름        | 느린 편              |          |
| 격리  | 커널 기능 기반  | 하드웨어 [[용어 모음집/가상화 | 가상화]] 기반 |
| 크기  | 작음        | 큼                 |          |

---

## 자주 하는 실수
- 컨테이너를 작은 VM이라고만 생각함
  - 바로잡기: 컨테이너는 호스트 커널을 공유하는 프로세스 격리이고, VM은 게스트 OS와 가상 하드웨어를 포함하는 더 무거운 격리다.
- 컨테이너마다 커널이 따로 있다고 생각함
  - 바로잡기: 컨테이너는 보통 호스트 커널을 공유하며, namespace와 cgroup으로 보이는 자원과 사용량을 제한한다.
- namespace와 cgroup 역할을 섞음
  - 바로잡기: Namespace는 프로세스가 보는 자원 이름 공간을 분리하고, cgroup은 CPU/메모리/I/O 사용량을 제한한다.
- 이미지와 컨테이너를 같은 것으로 생각함
  - 바로잡기: 이미지는 실행 템플릿이고, 컨테이너는 그 이미지에서 만들어져 실행 중인 인스턴스다.

---

## 면접 포인트
> 컨테이너는 호스트 커널을 공유하면서 프로세스 실행 환경을 격리하는 방식입니다. namespace는 PID, 네트워크, 파일 시스템처럼 프로세스가 보는 자원 공간을 분리하고, cgroup은 CPU와 메모리 같은 자원 사용량을 제한하고 측정합니다. 그래서 컨테이너는 VM보다 가볍고 빠르지만 커널을 공유한다는 특성을 이해해야 합니다.

---

## 체크리스트
- [ ] 컨테이너가 호스트 커널을 공유한다는 점을 안다
- [ ] namespace와 cgroup 차이를 설명할 수 있다
- [ ] 컨테이너와 이미지 차이를 안다
- [ ] VM과 컨테이너 차이를 말할 수 있다
---

## 보강 설명
가상화와 컨테이너 주제는 하나의 물리 자원을 여러 격리된 실행 환경이 나눠 쓰게 하는 방법이다. VM은 하드웨어 수준 추상화가 강하고, 컨테이너는 커널을 공유하며 프로세스 환경을 격리한다.

## 구체적인 예시
예를 들어 VM은 게스트 OS를 따로 실행하지만 컨테이너는 호스트 커널을 공유한다. 컨테이너는 namespace로 보이는 자원을 분리하고 cgroup으로 사용량을 제한한다.
