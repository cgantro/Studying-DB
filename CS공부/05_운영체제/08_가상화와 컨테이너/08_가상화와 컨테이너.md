# 가상화와 컨테이너

## 한 줄 정의
가상화와 컨테이너는 **하나의 물리 자원을 여러 실행 환경이 독립적으로 쓰는 것처럼 나누어 보이게 하는 기술**이다.

쉽게 말하면, 한 컴퓨터 위에서 여러 컴퓨터나 여러 격리된 실행 공간이 있는 것처럼 만들어 **배포, 격리, 자원 제어를 쉽게 하는 방식**이다.

---

## 왜 배우는가
- 클라우드, Docker, Kubernetes의 기본 이해과 연결된다.
- [[용어 모음집/운영체제|운영체제]]의 자원 격리, [[용어 모음집/스케줄링|스케줄링]], 메모리 관리와 이어진다.
- VM과 컨테이너의 차이는 면접과 실무에서 자주 나온다.
- 가상 메모리와 시스템 가상화를 헷갈리지 않기 위해 필요하다.

---

## 전체 흐름

```text
물리 하드웨어
   ↓
운영체제 또는 하이퍼바이저
   ↓
격리된 실행 환경
   ├─ 가상머신
   └─ 컨테이너
```

---

## 핵심 개념
- **가상머신(VM)**: 하드웨어까지 가상화해 별도 OS를 실행하는 환경
- **하이퍼바이저**: VM을 만들고 관리하는 계층
- **컨테이너**: 같은 [[용어 모음집/커널|커널]]을 공유하면서 [[용어 모음집/프로세스|프로세스]] 실행 환경을 격리하는 방식
- **namespace**: 프로세스가 보는 자원 이름 공간을 분리
- **cgroup**: [[용어 모음집/CPU|CPU]], 메모리 같은 자원 사용량을 제한/측정
- **이미지**: 컨테이너 실행에 필요한 파일 묶음

---

## VM과 컨테이너 비교

| 항목 | VM | 컨테이너 |
|---|---|---|
| 격리 단위 | 하드웨어/OS 수준 | 프로세스/커널 기능 수준 |
| 커널 | 각 VM이 별도 커널 사용 | 호스트 커널 공유 |
| 무게 | 상대적으로 무거움 | 상대적으로 가벼움 |
| 시작 속도 | 느린 편 | 빠른 편 |
| 격리 강도 | 강한 편 | 설정과 커널에 의존 |

---

## 운영체제와 연결
- 스케줄러는 VM/컨테이너의 실행 시간을 나눈다
- 메모리 관리는 각 환경에 제한과 격리를 제공한다
- [[용어 모음집/파일 시스템|파일 시스템]]은 이미지와 레이어 구조로 연결된다
- 네트워크는 가상 인터페이스와 namespace로 격리될 수 있다

---

## 면접 포인트
> 가상머신은 하드웨어를 가상화해 각 VM이 별도 운영체제를 실행하는 방식이고, 컨테이너는 호스트 커널을 공유하면서 프로세스 실행 환경을 격리하는 방식입니다. VM은 격리가 강하지만 무겁고, 컨테이너는 가볍고 빠르지만 커널 공유 특성을 이해해야 합니다. 컨테이너 격리에는 namespace와 cgroup이 핵심입니다.

## 보강 설명
가상화와 컨테이너 주제는 하나의 물리 자원을 여러 격리된 실행 환경이 나눠 쓰게 하는 방법이다. VM은 하드웨어 수준 추상화가 강하고, 컨테이너는 커널을 공유하며 프로세스 환경을 격리한다.

## 구체적인 예시
예를 들어 VM은 게스트 OS를 따로 실행하지만 컨테이너는 호스트 커널을 공유한다. 컨테이너는 namespace로 보이는 자원을 분리하고 cgroup으로 사용량을 제한한다.
