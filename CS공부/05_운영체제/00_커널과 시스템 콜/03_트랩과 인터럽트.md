# 트랩과 인터럽트

## 한 줄 정의
==[[용어 모음집/트랩|트랩]]은 **현재 실행 중인 명령 흐름에서 동기적으로 [[용어 모음집/커널|커널]]에 진입하는 사건**이고, [[용어 모음집/인터럽트|인터럽트]]는 **외부 장치나 타이머가 비동기적으로 [[용어 모음집/CPU|CPU]]에게 알리는 사건**이다.==

---

## 핵심
- 트랩은 실행 흐름과 직접 관련된 사건이다
- 인터럽트는 외부에서 언제든 들어올 수 있다
- [[용어 모음집/시스템 콜|시스템 콜]]은 보통 트랩을 통해 커널로 들어간다
- 타이머 인터럽트는 [[용어 모음집/스케줄링|스케줄링]]과 연결된다
- 장치 인터럽트는 I/O 완료 처리와 연결된다

---

## 트랩 예시
- 시스템 콜 요청
- 0으로 나누기 같은 예외
- [[용어 모음집/페이지 폴트|페이지 폴트]]
- 디버거 브레이크포인트

실행 중인 명령 때문에 발생하므로 동기적이라고 한다.

---

## 인터럽트 예시
- 키보드 입력
- 디스크 I/O 완료
- 네트워크 패킷 도착
- 타이머 틱

현재 명령과 직접 관계없이 외부에서 발생하므로 비동기적이라고 한다.

---

## 비교표

| 항목                       | 트랩            | 인터럽트         |     |
| ------------------------ | ------------- | ------------ | --- |
| 발생 원인                    | 현재 명령 실행      | 외부 장치/타이머    |     |
| 타이밍                      | 동기적           | 비동기적         |     |
| 예시                       | 시스템 콜, 페이지 폴트 | 키보드, 디스크 완료  |     |
| [[용어 모음집/운영체제\|운영체제]] 연결 | 예외 처리, 커널 진입  | 스케줄링, I/O 처리 |     |

---

## 처리 흐름

```text
사건 발생
   ↓
현재 상태 저장
   ↓
커널 핸들러 실행
   ↓
필요한 처리 수행
   ↓
복귀 또는 다른 작업으로 전환
```

---

## 자주 하는 실수
- 트랩과 인터럽트를 모두 같은 의미로 씀
  - 바로잡기: 트랩은 실행 중인 명령 때문에 동기적으로 발생하고, 인터럽트는 외부 장치나 타이머가 비동기적으로 알리는 사건이다.
- 페이지 폴트를 단순 에러로만 봄
  - 바로잡기: 페이지 폴트는 잘못된 접근일 수도 있지만, 요구 페이징이나 COW처럼 정상적인 메모리 로딩 과정에서도 발생한다.
- 타이머 인터럽트와 스케줄링의 관계를 놓침
  - 바로잡기: 타이머 인터럽트는 커널이 주기적으로 CPU를 회수해 스케줄러를 실행할 수 있게 만든다.
- 인터럽트 핸들러가 오래 실행되어도 된다고 생각함
  - 바로잡기: 인터럽트 핸들러는 가능한 짧게 처리하고, 오래 걸리는 작업은 별도 작업 큐나 후반부 처리로 넘기는 것이 일반적이다.

---

## 면접 포인트
> 트랩은 현재 실행 중인 명령 때문에 동기적으로 발생하는 커널 진입 사건이고, 인터럽트는 외부 장치나 타이머에서 비동기적으로 발생하는 사건입니다. 시스템 콜은 트랩을 통해 커널에 진입하고, 타이머 인터럽트는 스케줄링, 장치 인터럽트는 I/O 완료 처리와 연결됩니다.

- [ ] 장치 인터럽트와 I/O 완료 처리를 연결할 수 있다

## 보강 설명
입출력 주제는 CPU가 느린 장치와 어떻게 협력하는지 보는 영역이다. CPU가 계속 기다릴지, 장치가 인터럽트로 알릴지, [[용어 모음집/DMA|DMA]]로 데이터 이동을 맡길지에 따라 효율이 달라진다.

## 구체적인 예시
예를 들어 디스크에서 큰 파일을 읽을 때 CPU가 바이트마다 직접 옮기면 비효율적이다. DMA를 사용하면 장치가 메모리로 직접 데이터를 옮기고, 완료되면 인터럽트로 CPU에게 알린다.
