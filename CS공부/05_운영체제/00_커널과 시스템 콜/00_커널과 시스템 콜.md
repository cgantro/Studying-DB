# 커널과 시스템 콜

## 한 줄 정의
커널과 시스템 콜은 **사용자 프로그램이 하드웨어와 [[용어 모음집/운영체제|운영체제]] 자원을 안전하게 쓰도록 중간에서 통제하는 핵심 구조**다.

쉽게 말하면, 프로그램이 파일을 열거나 네트워크를 쓰거나 메모리를 더 달라고 할 때 **운영체제에게 정식으로 요청하는 문**이 시스템 콜이고, 그 요청을 처리하는 중심부가 커널이다.

---

## 왜 배우는가
- 운영체제의 권한 분리와 보호를 이해하는 출발점이다.
- 파일, [[용어 모음집/프로세스|프로세스]], 메모리, 네트워크 작업은 대부분 시스템 콜로 이어진다.
- 사용자 모드와 커널 모드를 구분해야 보안과 안정성을 설명할 수 있다.
- 인터럽트, 트랩, [[용어 모음집/문맥 교환|문맥 교환]]과 자연스럽게 연결된다.

---

## 전체 흐름

```text
사용자 프로그램
   ↓ 시스템 콜 요청
사용자 모드 → 커널 모드 전환
   ↓
커널이 권한 검사와 자원 처리
   ↓
결과 반환
커널 모드 → 사용자 모드 복귀
   ↓
프로그램 계속 실행
```

---

## 핵심 개념
- **커널(kernel)**: 운영체제의 핵심 실행부
- **사용자 모드(user mode)**: 일반 프로그램이 실행되는 제한된 모드
- **커널 모드(kernel mode)**: 하드웨어와 핵심 자원에 접근 가능한 특권 모드
- **시스템 콜(system call)**: 사용자 프로그램이 커널 기능을 요청하는 인터페이스
- **트랩(trap)**: 의도적으로 커널에 진입하는 동기적 예외
- **인터럽트(interrupt)**: 외부 장치나 타이머가 [[용어 모음집/CPU|CPU]]에게 알리는 비동기 사건

---

## 왜 직접 접근을 막는가
일반 프로그램이 하드웨어와 메모리에 마음대로 접근하면 시스템 전체가 위험해진다.

- 다른 프로세스 메모리 침범 가능
- 파일 권한 우회 가능
- 장치 상태 파괴 가능
- 운영체제 자체가 망가질 수 있음

그래서 운영체제는 권한을 나누고, 위험한 작업은 커널을 통해서만 하게 만든다.

---

## 시스템 콜 예시

| 작업 | 시스템 콜 이해 |
|---|---|
| 파일 열기 | 커널에게 파일 접근 요청 |
| 프로세스 생성 | 새 실행 단위 생성 요청 |
| 메모리 할당 | 주소 공간 확장 또는 매핑 요청 |
| 네트워크 송수신 | 소켓을 통한 입출력 요청 |
| 시간 확인 | 커널이 관리하는 시간 정보 요청 |

---

## 면접 포인트
> 커널은 운영체제의 핵심으로 하드웨어와 자원을 관리하고, 시스템 콜은 사용자 프로그램이 커널 기능을 요청하는 공식 인터페이스입니다. 일반 프로그램은 사용자 모드에서 제한된 권한으로 실행되고, 파일 I/O나 프로세스 생성처럼 특권이 필요한 작업은 시스템 콜을 통해 커널 모드로 전환해 처리합니다.

---

## 체크리스트
- [ ] 커널과 사용자 프로그램의 역할을 구분할 수 있다
- [ ] 사용자 모드와 커널 모드가 왜 나뉘는지 설명할 수 있다
- [ ] 시스템 콜 흐름을 말할 수 있다
- [ ] 트랩과 인터럽트의 차이를 이해적으로 안다
- [ ] 파일/프로세스/메모리 작업이 시스템 콜과 연결됨을 안다

---

## 초보자용 한 줄 기억법
커널과 시스템 콜은 **"프로그램이 위험한 일을 직접 하지 못하게 운영체제에게 허락받는 구조"**다.

## 보강 설명
커널과 시스템 콜 주제는 “사용자 프로그램이 왜 하드웨어를 직접 제어하지 못하고 운영체제를 거치는가”가 핵심이다. 보호와 안정성을 위해 권한을 나누고, 필요한 작업은 시스템 콜로 요청한다.

## 구체적인 예시
예를 들어 파일 읽기는 단순 함수 호출처럼 보여도 내부적으로는 사용자 모드에서 커널 모드로 전환되어 권한 검사, [[용어 모음집/파일 시스템|파일 시스템]] 처리, 디스크 I/O 요청이 진행된다.
