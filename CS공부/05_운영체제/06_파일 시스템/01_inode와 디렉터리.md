# inode와 디렉터리

## 한 줄 정의
[[용어 모음집/inode|inode]]는 **파일의 메타데이터와 데이터 블록 위치를 저장하는 [[용어 모음집/파일 시스템|파일 시스템]] 자료구조**이고, [[용어 모음집/디렉터리|디렉터리]]는 **파일 이름을 inode 번호에 매핑하는 특별한 파일**이다.

## inode에 들어가는 정보
- 파일 종류: 일반 파일, 디렉터리, [[용어 모음집/심볼릭 링크|심볼릭 링크]] 등
- 파일 크기
- 소유자와 그룹
- 권한
- 생성/수정/접근 시간
- 데이터 블록 위치
- 링크 수

중요한 점은 일반적으로 inode가 파일 이름 자체를 저장하지 않는다는 것이다.

## 디렉터리의 역할
디렉터리는 이름과 inode 번호의 목록을 가진다.

```text
name        inode
----------------
hello.txt   105
src         208
README.md   311
```

사용자가 `hello.txt`를 열면 파일 시스템은 디렉터리에서 이름을 찾고, 해당 inode를 따라가 파일 메타데이터와 데이터 블록 위치를 확인한다.

## 경로 탐색 예시
`/home/user/a.txt`를 찾는 과정:

```text
/ 디렉터리 inode
→ home 이름 조회
→ user 이름 조회
→ a.txt 이름 조회
→ a.txt inode 확인
→ 데이터 블록 읽기
```

## 하드 링크와 inode
[[용어 모음집/하드 링크|하드 링크]]는 서로 다른 파일 이름이 같은 inode를 가리키는 구조다. 그래서 이름은 달라도 실제 파일 내용과 메타데이터는 공유된다.

## 자주 헷갈리는 점
- 파일 이름은 inode가 아니라 디렉터리 엔트리에 저장된다.
- 같은 inode를 여러 이름이 가리킬 수 있다.
- 디렉터리도 파일 시스템 입장에서는 이름 목록을 담는 특별한 파일이다.

## 면접 포인트
inode는 파일의 메타데이터와 데이터 위치를 저장하고, 디렉터리는 파일 이름을 inode 번호에 연결한다. 경로 탐색은 각 디렉터리에서 이름을 조회하며 inode를 따라가는 과정이다.

## 보강 설명
파일 시스템 주제는 사람이 보는 파일 이름과 저장 장치의 실제 블록 배치를 연결하는 구조를 이해하는 것이 중요하다. 이름, inode, 데이터 블록, 권한, 복구 정보가 나뉘어 관리된다.

## 구체적인 예시
예를 들어 `/home/me/a.txt`를 열면 각 디렉터리에서 이름을 조회해 inode를 찾고, inode가 가리키는 데이터 블록을 읽는다. 저널링은 중간에 전원이 꺼져도 구조를 복구하기 쉽게 만든다.
