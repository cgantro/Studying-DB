# 링크 권한 저널링

## 한 줄 정의
링크는 **파일 이름과 실제 파일 데이터를 연결하는 방식**이고, 권한은 **누가 읽고 쓰고 실행할 수 있는지 정하는 규칙**이며, [[용어 모음집/저널링|저널링]]은 **[[용어 모음집/파일 시스템|파일 시스템]] 변경 작업을 기록해 장애 후 복구를 쉽게 하는 기법**이다.

## 하드 링크
[[용어 모음집/하드 링크|하드 링크]]는 다른 이름이 같은 [[용어 모음집/inode|inode]]를 가리키게 만든다.

특징:
- 원본과 링크가 같은 inode를 공유한다.
- 한쪽 이름을 삭제해도 링크 수가 남아 있으면 데이터는 유지된다.
- 일반적으로 [[용어 모음집/디렉터리|디렉터리]]에는 제한된다.

## 심볼릭 링크
[[용어 모음집/심볼릭 링크|심볼릭 링크]]는 다른 경로를 가리키는 별도 파일이다.

특징:
- 원본과 inode가 다르다.
- 원본 경로가 사라지면 깨진 링크가 될 수 있다.
- 다른 파일 시스템을 가리킬 수 있다.

## 권한
유닉스 계열 권한은 보통 소유자, 그룹, 기타 사용자에 대해 읽기, 쓰기, 실행 권한을 둔다.

```text
rwx r-x r--
소유자: 읽기/쓰기/실행
그룹:   읽기/실행
기타:   읽기
```

- `r`: read
- `w`: write
- `x`: execute

## 저널링
파일 시스템 작업 중 전원이 꺼지면 메타데이터가 어중간한 상태로 남을 수 있다. 저널링 파일 시스템은 변경 내용을 먼저 로그에 기록하고, 이후 실제 구조에 반영해 복구 가능성을 높인다.

```text
저널에 작업 기록
→ 실제 파일 시스템 변경
→ 완료 표시
```

## 예시
파일을 생성하는 중 장애가 발생하면 디렉터리 엔트리는 생겼는데 inode 초기화가 덜 되었거나, inode는 생겼는데 디렉터리에 연결되지 않은 상태가 될 수 있다. 저널링은 이런 중간 상태를 복구할 단서를 제공한다.

## 자주 헷갈리는 점
- 하드 링크는 같은 inode를 공유하고, 심볼릭 링크는 경로를 저장하는 별도 파일이다.
- 권한의 `x`는 파일에서는 실행, 디렉터리에서는 진입/탐색 가능 여부와 관련된다.
- 저널링은 데이터 손실을 완전히 막는 기능이 아니라 파일 시스템 일관성 복구를 돕는 기능이다.

## 면접 포인트
하드 링크는 같은 inode를 여러 이름이 가리키는 것이고, 심볼릭 링크는 경로를 가리키는 별도 파일이다. 저널링은 변경 내용을 로그로 남겨 장애 후 파일 시스템 일관성을 복구하기 쉽게 한다.

## 보강 설명
파일 시스템 주제는 사람이 보는 파일 이름과 저장 장치의 실제 블록 배치를 연결하는 구조를 이해하는 것이 중요하다. 이름, inode, 데이터 블록, 권한, 복구 정보가 나뉘어 관리된다.

## 구체적인 예시
예를 들어 `/home/me/a.txt`를 열면 각 디렉터리에서 이름을 조회해 inode를 찾고, inode가 가리키는 데이터 블록을 읽는다. 저널링은 중간에 전원이 꺼져도 구조를 복구하기 쉽게 만든다.
