# 페이지 교체와 지역성

## 한 줄 정의
[[용어 모음집/페이지 교체|페이지 교체]]는 **물리 메모리가 부족할 때 어떤 페이지를 내보낼지 결정하는 과정**이고, [[용어 모음집/지역성|지역성]]은 **프로그램이 가까운 시간이나 가까운 주소를 반복해서 접근하는 경향**이다.

## 왜 페이지 교체가 필요한가
필요한 페이지를 메모리에 올려야 하는데 빈 프레임이 없으면 기존 페이지 중 하나를 디스크로 내보내야 한다. 이때 어떤 페이지를 내보내느냐가 성능에 큰 영향을 준다.

## 대표 알고리즘
| 알고리즘 | 기준 | 특징 |
|---|---|---|
| FIFO | 가장 오래 전에 들어온 페이지 | 단순하지만 성능이 나쁠 수 있음 |
| [[용어 모음집/LRU|LRU]] | 가장 오래 사용하지 않은 페이지 | 지역성을 잘 활용 |
| Clock | 참조 비트를 활용한 LRU 근사 | 구현 비용과 성능 균형 |
| Optimal | 앞으로 가장 늦게 쓸 페이지 | 이론적 기준, 실제 구현 어려움 |

## 지역성
- 시간 지역성: 최근 사용한 페이지를 다시 사용할 가능성
- 공간 지역성: 접근한 주소 근처를 사용할 가능성

LRU는 시간 지역성을 활용하는 대표적인 정책이다.

## 예시
반복문이 작은 배열을 계속 훑는다면 같은 페이지들이 반복 사용된다. 이 경우 지역성이 좋고 page fault가 적다. 반대로 메모리보다 훨씬 큰 데이터를 무작위로 접근하면 page fault가 늘 수 있다.

## Dirty page
수정된 페이지를 내보낼 때는 디스크에 기록해야 하므로 비용이 크다. 수정되지 않은 페이지는 그냥 버리고 나중에 다시 읽으면 되기 때문에 상대적으로 저렴하다.

## 자주 헷갈리는 점
- FIFO는 간단하지만 항상 좋은 정책은 아니다.
- LRU는 이상적이지만 정확히 구현하려면 접근 기록 관리 비용이 크다.
- page fault 횟수가 많으면 [[용어 모음집/CPU|CPU]]보다 디스크 I/O가 병목이 된다.

## 면접 포인트
페이지 교체는 메모리 부족 시 희생 페이지를 고르는 문제다. 지역성을 활용하는 LRU 계열이 효과적이며, 실제 시스템은 비용 때문에 Clock 같은 근사 알고리즘을 사용하기도 한다.

## 보강 설명
[[용어 모음집/캐시|캐시]] 주제는 “빠른 CPU와 느린 메모리 사이의 간격을 어떻게 줄이는가”가 핵심이다. 지역성이 좋은 접근은 캐시 적중률을 높이고, [[용어 모음집/캐시 미스|캐시 미스]]나 일관성 비용은 성능을 떨어뜨린다.

## 구체적인 예시
예를 들어 배열을 앞에서부터 순서대로 읽으면 한 캐시 라인에 들어온 주변 원소를 이어서 쓰기 때문에 빠르다. 반대로 큰 배열을 무작위로 접근하거나 여러 코어가 같은 캐시 라인을 계속 수정하면 미스와 일관성 트래픽이 늘어난다.
