# 페이징과 세그멘테이션

## 한 줄 정의
[[용어 모음집/페이징|페이징]]은 **주소 공간을 고정 크기 페이지로 나누는 방식**이고, [[용어 모음집/세그멘테이션|세그멘테이션]]은 **코드, 데이터, 스택처럼 의미 있는 가변 크기 구역으로 나누는 방식**이다.

## 페이징
[[용어 모음집/가상 메모리|가상 메모리]]와 물리 메모리를 같은 크기의 블록으로 나눈다.

- 가상 메모리 블록: page
- 물리 메모리 블록: frame

장점:
- 외부 단편화를 줄일 수 있다.
- 메모리 할당과 회수가 비교적 단순하다.
- 가상 메모리 구현에 적합하다.

단점:
- 페이지 내부에서 남는 공간인 내부 단편화가 생길 수 있다.
- [[용어 모음집/페이지 테이블|페이지 테이블]] 관리 비용이 필요하다.

## 세그멘테이션
프로그램을 논리적 단위로 나눈다.

예시:
- code segment
- data segment
- heap segment
- stack segment

장점:
- 프로그램의 논리 구조와 잘 맞는다.
- 세그먼트별 보호와 공유를 표현하기 좋다.

단점:
- 세그먼트 크기가 달라 외부 단편화가 생길 수 있다.
- 가변 크기 관리가 어렵다.

## 비교표
| 구분 | 페이징 | 세그멘테이션 |
|---|---|---|
| 분할 기준 | 고정 크기 | 논리적 의미, 가변 크기 |
| 단편화 | 내부 단편화 가능 | 외부 단편화 가능 |
| 관리 | 페이지 테이블 | 세그먼트 테이블 |
| 장점 | 할당 관리 쉬움 | 논리 구조 표현 좋음 |

## 예시
배열 데이터가 여러 페이지에 나뉘어 저장될 수 있다. 프로그래머는 연속된 가상 주소로 보지만 실제 물리 메모리에서는 여러 프레임에 흩어져 있을 수 있다.

## 자주 헷갈리는 점
- 페이징은 물리 메모리가 연속적이지 않아도 [[용어 모음집/프로세스|프로세스]]에는 연속된 주소처럼 보이게 한다.
- 세그멘테이션은 의미 단위 분할이라 보호와 공유를 설명하기 좋다.
- 현대 시스템은 페이징 중심으로 가상 메모리를 구현하는 경우가 많다.

## 면접 포인트
페이징은 고정 크기 분할로 외부 단편화를 줄이고, 세그멘테이션은 논리적 가변 크기 분할로 프로그램 구조를 잘 표현한다. 페이징은 내부 단편화, 세그멘테이션은 외부 단편화와 연결된다.

## 보강 설명
메모리 관리 주제는 가상 주소와 물리 주소를 분리하고, 제한된 물리 메모리를 효율적으로 쓰는 방법을 다룬다. 주소 변환, [[용어 모음집/페이지 폴트|페이지 폴트]], 교체 정책, [[용어 모음집/커널|커널]] 할당자가 서로 연결된다.

## 구체적인 예시
예를 들어 프로세스가 가상 주소에 접근하면 TLB나 페이지 테이블을 통해 물리 프레임을 찾는다. 해당 페이지가 메모리에 없으면 페이지 폴트가 발생하고 [[용어 모음집/운영체제|운영체제]]가 디스크에서 페이지를 가져올 수 있다.
