# 페이지 테이블과 TLB

## 한 줄 정의
[[용어 모음집/페이지 테이블|페이지 테이블]]은 **가상 주소의 페이지 번호를 물리 메모리의 프레임 번호로 바꾸기 위한 표**이고, [[용어 모음집/TLB|TLB]]는 **최근 주소 변환 결과를 저장하는 빠른 [[용어 모음집/캐시|캐시]]**다.

## 주소 변환 흐름
```text
가상 주소
→ 페이지 번호 + 오프셋 분리
→ 페이지 테이블에서 프레임 번호 조회
→ 프레임 번호 + 오프셋
→ 물리 주소
```

오프셋은 페이지 안에서의 위치이므로 변하지 않고, 페이지 번호만 프레임 번호로 바뀐다.

## TLB가 필요한 이유
메모리에 접근할 때마다 페이지 테이블을 메모리에서 읽으면 실제 데이터 접근 전에 추가 메모리 접근이 필요하다. TLB는 최근 변환 결과를 저장해 이 비용을 줄인다.

```text
TLB hit:   빠르게 물리 주소 획득
TLB miss:  페이지 테이블 조회 후 TLB 갱신
```

## 예시
페이지 크기가 4KB라면 가상 주소는 페이지 번호와 0~4095 범위의 오프셋으로 나눌 수 있다. 페이지 테이블이 `가상 페이지 5 → 물리 프레임 20`이라고 알려주면, 가상 페이지 5의 100번째 바이트는 물리 프레임 20의 100번째 바이트가 된다.

## 페이지 테이블 엔트리 정보
- 물리 프레임 번호
- valid bit
- 권한 정보: read/write/execute
- dirty bit
- accessed bit

## 자주 헷갈리는 점
- 페이지 테이블은 [[용어 모음집/프로세스|프로세스]]마다 별도로 존재할 수 있다.
- TLB는 페이지 테이블 자체가 아니라 주소 변환 결과를 캐싱한다.
- TLB miss와 page fault는 다르다. TLB miss는 변환 캐시 실패이고, page fault는 해당 페이지가 메모리에 없거나 접근 권한 문제가 있는 경우다.

## 면접 포인트
가상 주소는 페이지 번호와 오프셋으로 나뉘며, 페이지 테이블은 페이지 번호를 프레임 번호로 변환한다. TLB는 이 변환 결과를 캐싱해 주소 변환 비용을 줄인다.

## 보강 설명
메모리 관리 주제는 가상 주소와 물리 주소를 분리하고, 제한된 물리 메모리를 효율적으로 쓰는 방법을 다룬다. 주소 변환, 페이지 폴트, 교체 정책, [[용어 모음집/커널|커널]] 할당자가 서로 연결된다.

## 구체적인 예시
예를 들어 프로세스가 가상 주소에 접근하면 TLB나 페이지 테이블을 통해 물리 프레임을 찾는다. 해당 페이지가 메모리에 없으면 페이지 폴트가 발생하고 [[용어 모음집/운영체제|운영체제]]가 디스크에서 페이지를 가져올 수 있다.
