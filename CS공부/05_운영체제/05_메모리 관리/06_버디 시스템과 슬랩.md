# 버디 시스템과 슬랩

## 한 줄 정의
[[용어 모음집/버디 시스템|버디 시스템]]은 **물리 메모리를 2의 거듭제곱 크기 블록으로 나누고 합치며 관리하는 방식**이고, [[용어 모음집/슬랩 할당자|슬랩 할당자]]는 **자주 쓰는 [[용어 모음집/커널|커널]] 객체를 빠르게 재사용하기 위한 메모리 할당 방식**이다.

## 버디 시스템
메모리를 큰 블록에서 시작해 요청 크기에 맞을 때까지 절반씩 나눈다. 해제할 때는 인접한 buddy 블록이 비어 있으면 다시 합친다.

```text
1024KB
→ 512KB + 512KB
→ 256KB + 256KB + 512KB
```

장점:
- 블록 분할과 병합 규칙이 단순하다.
- 인접한 빈 블록을 빠르게 합칠 수 있다.

단점:
- 요청 크기가 2의 거듭제곱과 맞지 않으면 내부 단편화가 생길 수 있다.

## 슬랩 할당자
커널에서는 [[용어 모음집/프로세스|프로세스]] 구조체, 파일 객체, [[용어 모음집/inode|inode]] 같은 작은 객체를 자주 만들고 지운다. 슬랩 할당자는 이런 객체를 일정 크기 [[용어 모음집/캐시|캐시]]에 보관해 빠르게 재사용한다.

```text
cache: task_struct 전용
cache: inode 전용
cache: file 객체 전용
```

## 왜 슬랩이 필요한가
매번 일반 메모리 할당자로 작은 객체를 새로 만들면 초기화와 할당 비용이 크다. 슬랩은 이미 준비된 객체 공간을 재사용해 속도를 높이고 단편화를 줄인다.

## 비교표
| 구분 | 버디 시스템 | 슬랩 할당자 |
|---|---|---|
| 대상 | 물리 페이지/큰 블록 관리 | 커널 객체 관리 |
| 방식 | 2의 거듭제곱 분할/병합 | 객체 캐시 재사용 |
| 장점 | 병합이 단순 | 작은 객체 할당 빠름 |
| 단점 | 내부 단편화 가능 | 캐시 관리 비용 |

## 자주 헷갈리는 점
- 버디 시스템은 주로 페이지 단위의 큰 메모리 관리와 관련된다.
- 슬랩은 버디 시스템 위에서 작은 커널 객체를 효율적으로 관리하는 계층으로 이해하면 좋다.
- 둘은 경쟁 관계라기보다 서로 다른 수준의 할당 문제를 해결한다.

## 면접 포인트
버디 시스템은 메모리를 2의 거듭제곱 블록으로 나누고 buddy를 병합해 관리한다. 슬랩 할당자는 자주 생성되는 커널 객체를 캐시해 할당과 해제를 빠르게 한다.

## 보강 설명
메모리 관리 주제는 가상 주소와 물리 주소를 분리하고, 제한된 물리 메모리를 효율적으로 쓰는 방법을 다룬다. 주소 변환, [[용어 모음집/페이지 폴트|페이지 폴트]], 교체 정책, 커널 할당자가 서로 연결된다.

## 구체적인 예시
예를 들어 프로세스가 가상 주소에 접근하면 [[용어 모음집/TLB|TLB]]나 [[용어 모음집/페이지 테이블|페이지 테이블]]을 통해 물리 프레임을 찾는다. 해당 페이지가 메모리에 없으면 페이지 폴트가 발생하고 [[용어 모음집/운영체제|운영체제]]가 디스크에서 페이지를 가져올 수 있다.
