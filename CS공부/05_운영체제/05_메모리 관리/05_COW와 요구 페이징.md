# COW와 요구 페이징

## 한 줄 정의
[[용어 모음집/요구 페이징|요구 페이징]](demand paging)은 **페이지가 실제로 필요할 때 메모리에 올리는 방식**이고, [[용어 모음집/COW|COW]](Copy-On-Write)는 **복사한 것처럼 보이게 공유하다가 쓰기가 발생할 때 실제 복사하는 최적화 기법**이다.

## 요구 페이징
[[용어 모음집/프로세스|프로세스]] 시작 시 모든 페이지를 메모리에 올리지 않는다. 처음 접근하는 순간 page fault를 발생시키고, [[용어 모음집/운영체제|운영체제]]가 필요한 페이지를 메모리에 로드한다.

장점:
- 시작 시간이 줄어들 수 있다.
- 실제로 쓰지 않는 페이지를 올리지 않아 메모리를 아낀다.

단점:
- 첫 접근 시 page fault 비용이 발생한다.

## COW
프로세스를 복사할 때 모든 메모리를 즉시 복사하지 않고, 부모와 자식이 같은 물리 페이지를 읽기 전용으로 공유한다. 둘 중 하나가 쓰기를 시도하면 그때 해당 페이지를 복사한다.

```text
fork 직후: 부모와 자식이 같은 페이지 공유
쓰기 발생: page fault
운영체제: 해당 페이지를 복사하고 쓰기 허용
```

## fork 예시
유닉스 계열에서 `fork()`는 부모 프로세스의 주소 공간을 자식에게 복사한 것처럼 보이게 한다. 하지만 실제로는 COW를 사용해 대부분의 페이지를 공유하고, 쓰기가 발생한 페이지만 복사한다.

## 왜 중요한가
- 프로세스 생성 비용을 줄인다.
- 메모리 사용량을 줄인다.
- `fork()` 후 바로 `exec()`하는 패턴에서 매우 효율적이다.

## 자주 헷갈리는 점
- COW는 복사를 안 하는 것이 아니라 쓰기 전까지 복사를 미루는 것이다.
- COW의 쓰기 시점에는 page fault가 정상적으로 발생할 수 있다.
- 요구 페이징과 COW 모두 page fault를 활용해 지연 처리한다.

## 면접 포인트
요구 페이징은 필요한 페이지를 실제 접근 시점에 올리는 방식이고, COW는 복사 비용을 쓰기 시점까지 미루는 방식이다. `fork()`에서 부모와 자식이 페이지를 공유하다가 쓰기 발생 시 복사되는 예시가 대표적이다.

## 보강 설명
메모리 관리 주제는 가상 주소와 물리 주소를 분리하고, 제한된 물리 메모리를 효율적으로 쓰는 방법을 다룬다. 주소 변환, 페이지 폴트, 교체 정책, [[용어 모음집/커널|커널]] 할당자가 서로 연결된다.

## 구체적인 예시
예를 들어 프로세스가 가상 주소에 접근하면 [[용어 모음집/TLB|TLB]]나 [[용어 모음집/페이지 테이블|페이지 테이블]]을 통해 물리 프레임을 찾는다. 해당 페이지가 메모리에 없으면 페이지 폴트가 발생하고 운영체제가 디스크에서 페이지를 가져올 수 있다.
