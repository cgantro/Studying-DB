# 뮤텍스와 세마포어

## 한 줄 정의
[[용어 모음집/뮤텍스|뮤텍스]]는 **[[용어 모음집/임계 구역|임계 구역]]에 한 번에 하나의 실행 흐름만 들어가게 하는 잠금 장치**이고, [[용어 모음집/세마포어|세마포어]]는 **정해진 개수만큼의 실행 흐름이 동시에 자원에 접근하도록 허용하는 카운터 기반 [[용어 모음집/동기화|동기화]] 도구**다.

## 뮤텍스
뮤텍스는 mutual exclusion의 줄임말이다. 공유 변수, 공유 자료구조, 파일 같은 자원을 한 번에 하나의 [[용어 모음집/스레드|스레드]]만 수정하게 할 때 사용한다.

```text
lock 획득
→ 임계 구역 실행
→ lock 해제
```

예시:
- 계좌 잔액 갱신
- 전역 카운터 증가
- 공유 큐에 데이터 삽입

## 세마포어
세마포어는 내부 카운터를 가지고 있다. 카운터가 0보다 크면 접근을 허용하고, 접근할 때 카운터를 줄이며, 작업이 끝나면 다시 늘린다.

예시:
- DB 커넥션 풀 10개 제한
- 동시에 다운로드 가능한 작업 수 제한
- 생산자-소비자 버퍼의 빈 칸/찬 칸 개수 관리

## 비교표
| 구분      | 뮤텍스                     | 세마포어            |     |
| ------- | ----------------------- | --------------- | --- |
| 목적      | [[용어 모음집/상호 배제\|상호 배제]] | 자원 개수 제어        |     |
| 동시 접근 수 | 보통 1개                   | N개 가능           |     |
| 소유권     | 락을 잡은 스레드가 해제하는 구조가 일반적 | 소유권 개념이 약할 수 있음 |     |
| 예시      | 공유 변수 보호                | 커넥션 풀 제한        |     |

## 이진 세마포어와 뮤텍스
카운터가 0 또는 1인 세마포어는 뮤텍스처럼 보일 수 있다. 하지만 뮤텍스는 소유권과 상호 배제 의미가 강하고, 세마포어는 신호 전달이나 자원 개수 제어 의미가 강하다.

## 자주 헷갈리는 점
- 락을 오래 잡으면 [[용어 모음집/병렬성|병렬성]]이 떨어진다.
- 락 해제를 빼먹으면 데드락으로 이어질 수 있다.
- 세마포어는 개수를 잘못 관리하면 접근 제한이 깨진다.

## 면접 포인트
뮤텍스는 한 번에 하나만 임계 구역에 들어가게 하는 도구이고, 세마포어는 N개의 자원을 카운터로 관리하는 도구다. 뮤텍스는 상호 배제, 세마포어는 자원 개수 제어와 신호 전달에 초점을 둔다.

## 보강 설명
동기화 주제는 여러 실행 흐름이 공유 자원을 동시에 건드릴 때 생기는 오류를 막는 영역이다. 정확성을 얻는 대신 락 대기, 병렬성 감소, 데드락 같은 비용을 함께 고려해야 한다.

## 구체적인 예시
예를 들어 두 스레드가 같은 카운터를 증가시키면 읽기와 쓰기 사이에 끼어들기가 발생해 값이 하나만 증가할 수 있다. 임계 구역을 뮤텍스로 보호하면 정확성은 지킬 수 있지만 락 경합이 생길 수 있다.
