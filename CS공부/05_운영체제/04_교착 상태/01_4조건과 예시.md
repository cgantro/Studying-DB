# 4조건과 예시

## 한 줄 정의
[[용어 모음집/교착 상태|교착 상태]](deadlock)는 **여러 작업이 서로 상대가 가진 자원을 기다리며 영원히 진행하지 못하는 상태**이고, 네 가지 필요 조건이 동시에 만족될 때 발생할 수 있다.

## 교착 상태 4조건
| 조건                      | 의미                                      |     |
| ----------------------- | --------------------------------------- | --- |
| [[용어 모음집/상호 배제\|상호 배제]] | 한 자원을 한 번에 하나의 작업만 사용할 수 있음             |     |
| 점유와 대기                  | 자원을 가진 채 다른 자원을 기다림                     |     |
| 비선점                     | [[용어 모음집/운영체제\|운영체제]]가 강제로 자원을 빼앗을 수 없음 |     |
| [[용어 모음집/순환 대기\|순환 대기]] | A는 B를, B는 C를, C는 A를 기다리는 순환 구조          |     |

## 예시
```text
Thread A: lock X 획득 → lock Y 대기
Thread B: lock Y 획득 → lock X 대기
```

A는 Y가 필요하지만 B가 들고 있고, B는 X가 필요하지만 A가 들고 있다. 둘 다 상대가 락을 놓기를 기다리므로 진행이 멈춘다.

## 실생활 이해
좁은 골목에서 차 두 대가 마주 보고 들어왔고, 둘 다 뒤로 물러나지 못하는 상황과 비슷하다. 각자 길 일부를 점유한 채 상대가 비켜주기를 기다린다.

## 자원 할당 그래프
```text
P1 → R2
R1 → P1
P2 → R1
R2 → P2
```

[[용어 모음집/프로세스|프로세스]]와 자원 사이에 순환이 있으면 교착 상태 가능성을 의심할 수 있다.

## 자주 헷갈리는 점
- 교착 상태는 단순히 느린 상태가 아니라 조건이 풀리지 않으면 영원히 진행하지 못하는 상태다.
- 네 조건은 필요 조건이다. 네 조건 중 하나라도 깨면 교착 상태를 막을 수 있다.
- starvation은 특정 작업이 계속 밀리는 것이고, deadlock은 서로 기다리며 멈춘 상태다.

## 면접 포인트
교착 상태는 상호 배제, 점유와 대기, 비선점, 순환 대기 네 조건이 동시에 성립할 때 발생할 수 있다. 예시는 두 [[용어 모음집/스레드|스레드]]가 서로 다른 락을 잡고 상대 락을 기다리는 상황으로 설명하면 명확하다.

## 보강 설명
교착 상태 주제는 자원을 가진 작업들이 서로를 기다리며 멈추는 구조를 찾는 것이 핵심이다. 네 가지 필요 조건 중 하나를 깨면 교착 상태를 예방할 수 있다.

## 구체적인 예시
예를 들어 스레드 A가 lock X를 잡고 lock Y를 기다리고, 스레드 B가 lock Y를 잡고 lock X를 기다리면 둘 다 진행하지 못한다. 락 획득 순서를 항상 같게 정하면 순환 대기를 줄일 수 있다.
