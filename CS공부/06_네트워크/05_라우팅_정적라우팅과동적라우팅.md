# 정적 라우팅과 동적 라우팅

## 한 줄 정의
정적 라우팅은 **사람이 직접 경로를 넣는 방식**, 동적 라우팅은 **장비가 서로 정보를 주고받아 경로를 배우는 방식**이다.

## 정적 라우팅
### 장점
- 단순하다
- 예측 가능하다
- 작은 네트워크에 좋다

### 단점
- 변경에 약하다
- 장애 시 자동 우회가 어렵다
- 네트워크가 커지면 관리가 힘들다

## 동적 라우팅
### 장점
- 변화에 잘 대응한다
- 장애 시 우회 경로를 찾기 쉽다
- 대규모 네트워크에 적합하다

### 단점
- 설정이 복잡하다
- 오버헤드가 있다
- 잘못 설정하면 불안정해질 수 있다

## 대표 예시
- RIP
- OSPF
- BGP

## 비교 기준
- 정적 라우팅: 종이에 그린 지도
- 동적 라우팅: 실시간 내비게이션

## 체크 포인트
- [ ] 둘의 차이를 한 문장으로 말할 수 있다
- [ ] 정적 라우팅이 유리한 상황을 안다
- [ ] 동적 라우팅이 필요한 이유를 안다
