# 라우팅 테이블과 게이트웨이

## 한 줄 정의
라우팅 테이블은 **목적지별로 어디로 보낼지 적어 둔 표**이고, 기본 게이트웨이는 **서브넷 밖으로 나갈 때 처음 보내는 출구**다.

## 왜 필요한가
네트워크는 모든 목적지의 길을 한 번에 알 수 없다. 그래서 다음 홉을 정해 주는 표가 필요하다.

## 라우팅 테이블이 하는 일
- 목적지 네트워크를 찾는다
- 다음 홉을 찾는다
- 출력 인터페이스를 고른다
- 경로 비용을 비교한다
- 기본 경로를 제공한다

## 예시
```text
목적지            다음 홉           인터페이스   의미
192.168.10.0/24   직접 연결         eth0         같은 서브넷
10.0.0.0/8        192.168.1.1       eth1         내부망 우회
0.0.0.0/0         192.168.1.254     eth1         기본 경로
```

## 기본 게이트웨이
기본 게이트웨이는 내가 모르는 외부 목적지로 갈 때 먼저 보내는 장비다.

### 기억법
- 같은 동네: 직접 감
- 다른 동네: 안내소 먼저 감

## 자주 헷갈리는 점
- 게이트웨이는 목적지가 아니라 경유지다
- 라우팅 테이블은 장비마다 따로 있다
- 더 구체적인 경로가 보통 우선된다
- `0.0.0.0/0`은 모든 목적지를 포함하는 기본 경로다

## 체크 포인트
- [ ] 기본 게이트웨이의 역할을 설명할 수 있다
- [ ] 기본 경로의 의미를 안다
- [ ] 가장 구체적인 경로가 우선된다는 점을 안다
