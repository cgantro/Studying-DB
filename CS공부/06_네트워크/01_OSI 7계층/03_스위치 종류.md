# 스위치 종류

## 한 줄 정의
스위치는 네트워크에서 데이터를 적절한 방향으로 전달하는 장비이고, 어떤 계층 정보를 보고 판단하느냐에 따라 L2, L3, L4, L7 스위치처럼 나눠 부를 수 있다.

쉽게 말하면:

```text
L2 스위치 = MAC 주소를 보고 보냄
L3 스위치 = IP 주소를 보고 보냄
L4 스위치 = IP + 포트 번호를 보고 보냄
L7 스위치 = HTTP 같은 응용 계층 내용을 보고 보냄
```

---

## 왜 스위치가 여러 종류인가
원래 전통적인 스위치는 주로 [[01_OSI 7계층|OSI 7계층]]의 2계층 장비를 뜻한다. 그런데 실제 네트워크 장비는 더 높은 계층의 정보를 보고 전달 결정을 하기도 한다.

| 이름 | 주로 보는 정보 | OSI 계층 | 대표 역할 |
|---|---|---|---|
| L2 스위치 | MAC 주소 | 2계층 데이터 링크 | 같은 LAN 안에서 프레임 전달 |
| L3 스위치 | IP 주소 | 3계층 네트워크 | VLAN 간 라우팅, 빠른 라우팅 |
| L4 스위치 | IP + 포트 | 4계층 전송 | TCP/UDP 포트 기반 로드 밸런싱 |
| L7 스위치 | HTTP 헤더, URL, 쿠키 등 | 7계층 응용 | 요청 내용 기반 로드 밸런싱 |

---

## L2 스위치
L2 스위치는 MAC 주소를 보고 프레임을 전달한다.

### 동작 방식
```text
프레임 수신
   ↓
출발지 MAC 주소를 MAC 주소 테이블에 학습
   ↓
목적지 MAC 주소를 확인
   ↓
해당 포트로 프레임 전달
```

### 예시
```text
PC A ----\
PC B ----- L2 스위치 ---- 라우터
PC C ----/
```

PC A가 PC B에게 데이터를 보낼 때, L2 스위치는 목적지 MAC 주소를 보고 PC B가 연결된 포트로만 프레임을 보낸다.

### 핵심 용어
- MAC 주소
- 프레임
- MAC 주소 테이블
- 브로드캐스트
- VLAN

### 주의점
L2 스위치는 기본적으로 IP 주소를 보고 경로를 고르지 않는다. 같은 네트워크 안에서 MAC 주소 기반으로 전달한다.

---

## L3 스위치
L3 스위치는 IP 주소를 보고 패킷을 전달할 수 있는 스위치다.

### 예시
```text
VLAN 10: 192.168.10.0/24
VLAN 20: 192.168.20.0/24

VLAN 10의 PC가 VLAN 20의 서버로 갈 때
L3 스위치가 IP 기반으로 라우팅
```

### 라우터와 다른 점
둘 다 3계층 라우팅을 할 수 있다. 다만 L3 스위치는 LAN 내부에서 빠른 VLAN 간 라우팅에 많이 쓰이고, 라우터는 WAN 연결, NAT, 복잡한 라우팅 정책, 인터넷 연결에서 많이 등장한다.

### 핵심 용어
- IP 주소
- 라우팅 테이블
- VLAN 간 라우팅
- 게이트웨이

---

## L4 스위치
L4 스위치는 IP 주소와 TCP/UDP 포트 번호를 보고 트래픽을 분산하거나 전달한다.

### 예시
```text
클라이언트 요청
목적지 IP: 10.0.0.10
목적지 포트: 443
   ↓
L4 스위치
   ├─ 웹 서버 1
   ├─ 웹 서버 2
   └─ 웹 서버 3
```

### 판단 기준
L4 스위치는 흔히 5-tuple을 본다.

```text
출발지 IP
목적지 IP
출발지 포트
목적지 포트
프로토콜
```

### 장점과 한계
- L7보다 빠르고 단순하다.
- TCP/UDP 수준에서 부하 분산이 가능하다.
- URL이나 HTTP 헤더 같은 응용 계층 내용은 보지 않는다.

---

## L7 스위치
L7 스위치는 응용 계층 내용을 보고 요청을 전달한다.

### 예시
```text
GET /api/users
Host: example.com
Cookie: session=...
```

L7 스위치는 이런 정보를 보고 다음처럼 분기할 수 있다.

```text
/api/*      -> API 서버
/static/*   -> 정적 파일 서버
/admin/*    -> 관리자 서버
```

### 판단 기준
- URL 경로
- Host 헤더
- HTTP 메서드
- 쿠키
- 사용자 에이전트
- 헤더 값
- 요청 본문 일부

### 장점과 한계
- 요청 내용을 기준으로 정교하게 분산할 수 있다.
- HTTPS 종료, 인증, 압축, 캐싱 같은 기능과 함께 쓰일 수 있다.
- L4보다 처리 비용이 크다.
- TLS를 내부에서 보려면 복호화 지점이 필요하다.

---

## L4와 L7 비교

| 구분 | L4 스위치 | L7 스위치 |
|---|---|---|
| 보는 정보 | IP, 포트, TCP/UDP | HTTP 헤더, URL, 쿠키 등 |
| 속도 | 상대적으로 빠름 | 상대적으로 무거움 |
| 분산 기준 | 연결 단위 | 요청 내용 단위 |
| 예시 | 443 포트 요청을 웹 서버들로 분산 | `/api`는 API 서버, `/image`는 이미지 서버 |
| 장점 | 단순하고 빠름 | 정교한 라우팅 가능 |
| 한계 | 요청 내용을 모름 | 응용 계층 해석 비용 |

---

## 허브, 브리지, 스위치, 라우터 차이

| 장비 | 계층 | 핵심 |
|---|---|---|
| 허브 | 1계층 | 들어온 신호를 모든 포트로 뿌림 |
| 브리지 | 2계층 | MAC 주소 기반으로 네트워크 구간 연결 |
| L2 스위치 | 2계층 | MAC 주소 테이블로 프레임 전달 |
| 라우터 | 3계층 | IP 주소 기반으로 네트워크 간 경로 선택 |
| L3 스위치 | 3계층 | 스위치 장비에서 라우팅 기능 제공 |

허브는 목적지를 거의 구분하지 않고 뿌리는 장비에 가깝고, L2 스위치는 MAC 주소를 학습해서 필요한 포트로만 보낸다.

---

## 자주 하는 실수
- 스위치는 무조건 2계층 장비라고만 생각함
  - 바로잡기: 기본 스위치는 L2 장비로 이해하면 좋지만, 실무에서는 L3/L4/L7 기능을 가진 스위치나 로드 밸런서도 스위치라고 부른다.
- L3 스위치와 라우터를 완전히 다른 것으로 생각함
  - 바로잡기: 둘 다 3계층 라우팅을 할 수 있다. 다만 L3 스위치는 LAN 내부 고속 라우팅, 라우터는 네트워크 간 연결과 정책 라우팅에서 자주 등장한다.
- L4 스위치가 HTTP URL을 보고 분산한다고 생각함
  - 바로잡기: L4는 IP와 포트 같은 전송 계층 정보를 보고, URL이나 HTTP 헤더는 L7에서 본다.
- L7 스위치가 항상 더 좋다고 생각함
  - 바로잡기: L7은 정교하지만 무겁다. 단순 TCP/UDP 분산이면 L4가 더 적합할 수 있다.
- MAC 주소와 IP 주소를 같은 역할로 생각함
  - 바로잡기: MAC은 같은 링크 안에서 프레임 전달에 쓰이고, IP는 네트워크 간 경로 선택에 쓰인다.

---

## 면접 포인트
- L2 스위치는 MAC 주소를 보고 프레임을 전달한다.
- L3 스위치는 IP 주소를 보고 라우팅할 수 있다.
- L4 스위치는 IP와 포트 번호를 기준으로 로드 밸런싱한다.
- L7 스위치는 URL, HTTP 헤더, 쿠키 등 응용 계층 정보를 보고 요청을 분산한다.
- 스위치 종류는 "어느 계층 정보를 보고 판단하는가"로 구분하면 된다.

### 답변 예시
> 스위치는 어떤 계층 정보를 보고 전달하느냐에 따라 L2, L3, L4, L7로 나눌 수 있습니다. L2 스위치는 MAC 주소를 보고 같은 LAN 안에서 프레임을 전달하고, L3 스위치는 IP 주소를 보고 라우팅할 수 있습니다. L4 스위치는 IP와 포트 번호를 기준으로 트래픽을 분산하고, L7 스위치는 HTTP URL이나 헤더 같은 응용 계층 정보를 보고 더 정교하게 요청을 나눕니다.
