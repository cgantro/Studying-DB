# OSI 7계층

## 한 줄 정의
OSI 7계층은 **네트워크 통신을 7개의 역할로 나눠 이해하는 개념 모델**이다.

쉽게 말하면, 통신을 한 덩어리로 보지 않고 **"어디서 무엇을 담당하는지"**로 쪼개서 보는 지도다.

---

## 왜 배우는가
- 네트워크는 너무 복잡해서 한 번에 보면 헷갈린다.
- 계층으로 나누면 문제를 어디서 찾아야 하는지 빨리 좁힐 수 있다.
- 면접에서 OSI는 거의 기본 중의 기본이다.
- HTTP, TCP, IP, 스위치, 라우터, DNS 같은 개념을 계층에 붙여서 이해할 수 있다.
- 실무 장애도 "어느 계층에서 깨졌는지"로 보면 추적이 쉬워진다.

---

## 초보자용 한 줄 기억법
- **OSI 7계층 = 네트워크 통신을 역할별로 나눈 지도**
- **위로 갈수록 사람과 가까운 개념**
- **아래로 갈수록 물리적 전송과 가까운 개념**

---

## 전체 흐름

```text
응용 계층 (Application)
  ↓
표현 계층 (Presentation)
  ↓
세션 계층 (Session)
  ↓
전송 계층 (Transport)
  ↓
네트워크 계층 (Network)
  ↓
데이터 링크 계층 (DataLink)
  ↓
물리 계층 (Physical)
```

### 반대로 생각하면
- 아래 계층이 위 계층을 위해 기반을 제공한다.
- 사용자가 브라우저에서 버튼을 눌렀을 때, 실제로는 아래 계층까지 모두 거쳐 데이터가 이동한다.

---

## 계층을 나누는 이유
### 1) 역할 분리
각 계층이 자기 일만 하면 설계와 유지보수가 쉬워진다.

### 2) 문제 분리
어느 계층에서 문제가 생겼는지 좁히기 좋다.

### 3) 기술 교체가 쉬움
하위 계층이 바뀌어도 상위 계층은 크게 영향을 덜 받을 수 있다.

### 4) 표준화
서로 다른 장비와 소프트웨어가 약속된 방식으로 통신할 수 있다.

---

## 계층 전체 요약표

| 계층        | 주요 역할          | 대표 예시           | 기억 포인트     |
| --------- | -------------- | --------------- | ---------- |
| 7. 응용     | 사용자가 직접 쓰는 서비스 | HTTP, DNS, SMTP | 웹/이메일/DNS  |
| 6. 표현     | 데이터 표현 방식      | 인코딩, 압축, 암호화    | 형식 변환      |
| 5. 세션     | 대화 흐름 관리       | 세션 유지           | 연결의 논리적 흐름 |
| 4. 전송     | 종단 간 전달        | TCP, UDP        | 포트, 신뢰성    |
| 3. 네트워크   | 경로 선택          | IP, 라우터         | 목적지까지 길 찾기 |
| 2. 데이터 링크 | 같은 네트워크 내 전달   | MAC, 스위치        | 프레임        |
| 1. 물리     | 실제 신호 전송       | 케이블, 전파         | 비트         |

---

## 1계층: 물리 계층

### 한 줄 정의
물리 계층은 **0과 1을 실제 신호로 바꿔서 전달하는 계층**이다.

### 하는 일
- 전기 신호, 광신호, 무선 신호를 전달한다
- 케이블, 커넥터, 전압, 주파수 같은 물리적 요소를 다룬다
- 실제로 비트를 보내는 가장 아래 계층이다

### 예시
- 랜 케이블
- 광케이블
- Wi-Fi 전파
- 허브(전통적 의미에서)

### 쉽게 말하면
- 비트를 도로 위에 실제로 싣고 보내는 단계다.

### 장애 판단 기준
- 케이블이 빠졌는가?
- 신호 세기가 약한가?
- 포트 LED가 켜지는가?

---

## 2계층: 데이터 링크 계층

### 한 줄 정의
데이터 링크 계층은 **같은 네트워크 안에서 프레임을 주고받는 계층**이다.

### 하는 일
- MAC 주소 기반 전달
- 프레임 구성
- 오류 검출
- 같은 링크 내 전달
- 스위치가 주로 이 계층에서 동작한다고 이해한다

### 핵심 개념
- **프레임(frame)**: 2계층 데이터 단위
- **MAC 주소**: 네트워크 인터페이스의 물리적 식별자
- **스위치**: MAC을 보고 프레임 전달

### 예시
- 같은 사무실 내 PC끼리 통신
- 공유기 내부의 장치들
- 스위치 허브 환경

### 쉽게 말하면
- 같은 동네 안에서 실제 집까지 배달하는 단계다.

### 장애 판단 기준
- MAC 주소를 알 수 있는가?
- 스위치 포트/VLAN이 맞는가?
- 프레임 오류가 있는가?

---

## 3계층: 네트워크 계층

### 한 줄 정의
네트워크 계층은 **서로 다른 네트워크 사이에서 패킷이 어디로 갈지 정하는 계층**이다.

### 하는 일
- IP 주소 사용
- 라우팅
- 경로 선택
- 서로 다른 네트워크 연결

### 핵심 개념
- **패킷(packet)**: 3계층 데이터 단위
- **IP 주소**: 네트워크 상 논리적 주소
- **라우터**: IP를 보고 다음 경로 결정

### 예시
- 집에서 외부 서버 접속
- 사내망에서 다른 부서 네트워크로 이동
- 인터넷 전체의 길 찾기

### 쉽게 말하면
- 다른 동네로 갈 때 어떤 큰 길을 탈지 정하는 단계다.

### 장애 판단 기준
- IP가 맞는가?
- 서브넷이 맞는가?
- 라우팅 테이블에 경로가 있는가?

---

## 4계층: 전송 계층

### 한 줄 정의
전송 계층은 **프로그램과 프로그램 사이의 전달을 책임지는 계층**이다.

### 하는 일
- TCP/UDP 제공
- 포트 번호 사용
- 신뢰성, 재전송, 흐름 제어, 혼잡 제어
- 종단 간(end-to-end) 통신 관리

### 핵심 개념
- **세그먼트(segment)**: TCP 데이터 단위로 이해
- **데이터그램(datagram)**: UDP 데이터 단위로 이해
- **포트 번호**: 어떤 프로그램으로 보낼지 구분

### 예시
- 웹 브라우저와 웹 서버
- DB 클라이언트와 DB 서버
- 실시간 음성/게임 전송

### 쉽게 말하면
- 같은 건물 안의 어느 부서로 보낼지 정하는 내부 우편 시스템 같은 느낌이다.

### 장애 판단 기준
- 포트가 열려 있는가?
- TCP 핸드셰이크가 되었는가?
- UDP인지 TCP인지 맞는가?

---

## 5계층: 세션 계층

### 한 줄 정의
세션 계층은 **통신 대화의 시작, 유지, 종료를 관리하는 개념 계층**이다.

### 하는 일
- 세션 설정
- 세션 유지
- 세션 종료
- 대화 상태 관리

### 실제 구현 관점
- 실제 인터넷 기술에서 4, 7계층만큼 강하게 드러나지 않을 때도 있지만, 개념적으로 매우 중요하다.
- 로그인 세션, 대화 연결 유지 같은 개념을 이해할 때 도움이 된다.

### 쉽게 말하면
- 두 사람이 대화를 시작하고 계속 이어가고 끝내는 과정이다.

---

## 6계층: 표현 계층

### 한 줄 정의
표현 계층은 **데이터의 형식과 표현 방식을 맞추는 계층**이다.

### 하는 일
- 인코딩/디코딩
- 압축
- 암호화/복호화
- 데이터 형식 변환

### 예시
- UTF-8 인코딩
- JSON 직렬화
- gzip 압축
- TLS 암호화 개념과 연결해서 생각하기도 함

### 쉽게 말하면
- 서로 다른 언어를 쓰는 사람이 같은 내용을 이해하도록 번역하는 단계다.

---

## 7계층: 응용 계층

### 한 줄 정의
응용 계층은 **사용자가 직접 만지는 서비스가 동작하는 계층**이다.

### 하는 일
- HTTP
- DNS
- SMTP
- FTP
- SSH
- MQTT 같은 프로토콜

### 예시
- 웹 브라우저로 사이트 열기
- 도메인 이름 조회
- 이메일 주고받기

### 쉽게 말하면
- 사용자가 "실제로 쓰는 프로그램의 언어"에 가장 가까운 층이다.

---

## 계층별 데이터 단위

| 계층 | 데이터 단위 |
|---|---|
| 7~5계층 | 메시지로 묶어 생각 |
| 4계층 | 세그먼트 / 데이터그램 |
| 3계층 | 패킷 |
| 2계층 | 프레임 |
| 1계층 | 비트 |

이건 문제마다 조금 다르게 설명될 수 있지만, 기본 이해를 잡는 데 유용하다.

---

## PDU와 헤더/트레일러
각 계층에서 주고받는 데이터 단위를 **PDU(Protocol Data Unit)**라고 한다. 계층마다 PDU 이름이 다르다.

| OSI 계층 | PDU 이름 | 보통 붙는 정보 |
|---|---|---|
| 7 응용 | 데이터, 메시지 | HTTP 헤더, DNS 질의 같은 응용 프로토콜 정보 |
| 6 표현 | 데이터 | 인코딩, 압축, 암호화 형식 |
| 5 세션 | 데이터 | 세션 관리 정보 |
| 4 전송 | 세그먼트(TCP), 데이터그램(UDP) | 포트 번호, 순서 번호, 체크섬 |
| 3 네트워크 | 패킷 | 출발지/목적지 IP 주소 |
| 2 데이터 링크 | 프레임 | 출발지/목적지 MAC 주소, FCS 트레일러 |
| 1 물리 | 비트 | 전기/광/무선 신호 |

여기서 말하는 "헤더랑 테일"은 보통 **헤더(header)**와 **트레일러(trailer)**라고 부른다. 대부분의 계층은 앞에 헤더를 붙이고, 데이터 링크 계층은 뒤에 오류 검사용 트레일러도 붙일 수 있다.

---

## 캡슐화와 역캡슐화

### 캡슐화(Encapsulation)
위 계층의 데이터가 아래 계층으로 내려가며 각 계층의 헤더가 붙는 과정이다. 데이터 링크 계층에서는 뒤에 트레일러도 붙을 수 있다.

```text
7~5계층 데이터
[ Application Data ]

4계층 전송 계층
[ TCP/UDP Header ][ Application Data ]
        = Segment 또는 Datagram

3계층 네트워크 계층
[ IP Header ][ TCP/UDP Header ][ Application Data ]
        = Packet

2계층 데이터 링크 계층
[ Ethernet Header ][ IP Header ][ TCP/UDP Header ][ Application Data ][ FCS Trailer ]
        = Frame

1계층 물리 계층
010101010101010101010101...
        = Bits
```

### 역캡슐화(Decapsulation)
받는 쪽에서 아래 계층부터 헤더와 트레일러를 확인하고 제거하면서 위로 올라가는 과정이다.

```text
1계층
Bits 수신
   ↓
2계층
[ Ethernet Header ][ IP Header ][ TCP Header ][ Data ][ FCS Trailer ]
Ethernet Header와 FCS 확인 후 제거
   ↓
3계층
[ IP Header ][ TCP Header ][ Data ]
IP Header 확인 후 제거
   ↓
4계층
[ TCP Header ][ Data ]
TCP Header 확인 후 제거
   ↓
7계층
[ Data ]
응용 프로그램에 전달
```

### 핵심
- 위 계층 데이터는 아래 계층 입장에서는 payload다.
- 4계층은 포트 번호를 붙여 어느 프로그램으로 보낼지 구분한다.
- 3계층은 IP 주소를 붙여 어느 네트워크/호스트로 갈지 정한다.
- 2계층은 MAC 주소를 붙여 같은 링크 안에서 어느 장비로 보낼지 정한다.
- 2계층의 FCS(Frame Check Sequence)는 프레임이 전송 중 깨졌는지 확인하는 트레일러다.

### 웹 요청 하나로 보는 캡슐화
브라우저에서 `https://example.com`에 접속한다고 하자.

```text
HTTP 요청 데이터
   ↓
TCP 헤더 추가
출발지 포트: 임시 포트
목적지 포트: 443
   ↓
IP 헤더 추가
출발지 IP: 내 IP
목적지 IP: 서버 IP
   ↓
Ethernet/Wi-Fi 헤더 추가
출발지 MAC: 내 NIC MAC
목적지 MAC: 다음 홉 장비 MAC
   ↓
비트로 변환되어 전송
```

목적지 MAC 주소는 최종 서버의 MAC 주소가 아닐 수 있다. 같은 네트워크 밖으로 나갈 때는 보통 다음 홉인 게이트웨이/라우터의 MAC 주소를 사용한다.

### 기억법
- 보내는 쪽: 포장을 점점 더 한다
- 받는 쪽: 포장을 하나씩 벗긴다

## 계층별 주소와 장비

| 계층 | 주로 보는 주소 | 대표 장비 |
|---|---|---|
| 1 물리 | 주소보다 신호 자체 | 케이블, 리피터, 허브 |
| 2 데이터 링크 | MAC 주소 | L2 스위치, 브리지 |
| 3 네트워크 | IP 주소 | 라우터, L3 스위치 |
| 4 전송 | 포트 번호 | L4 스위치, 로드 밸런서 |
| 7 응용 | URL, HTTP 헤더, 쿠키 등 | L7 스위치, 프록시, API Gateway |

이 표 때문에 "스위치도 여러 종류가 있다"는 말이 나온다. 일반적으로 스위치라고 하면 L2 스위치를 많이 말하지만, 실제 장비나 로드 밸런서 문맥에서는 L3/L4/L7 스위치라는 표현도 쓴다.

더 자세한 정리는 [[03_스위치 종류]]에서 본다.

## OSI와 TCP/IP 모델 비교
OSI 7계층은 개념을 설명하기 좋고, TCP/IP 모델은 실제 인터넷을 설명할 때 자주 쓴다.

### TCP/IP 관점으로는 보통 이렇게 묶는다
- 응용 계층: OSI 7, 6, 5 일부
- 전송 계층: OSI 4
- 인터넷 계층: OSI 3
- 네트워크 액세스 계층: OSI 1, 2

### 비교표

| OSI 7계층 | TCP/IP 모델 | 예시 |
|---|---|---|
| 7 응용 | 응용 | HTTP, DNS |
| 6 표현 | 응용 | 인코딩, 암호화 |
| 5 세션 | 응용 | 세션 유지 |
| 4 전송 | 전송 | TCP, UDP |
| 3 네트워크 | 인터넷 | IP, 라우팅 |
| 2 데이터 링크 | 네트워크 액세스 | MAC, 스위치 |
| 1 물리 | 네트워크 액세스 | 케이블, 전파 |

## 실제 네트워크 문제를 계층별로 생각하기

### 예시 1: 케이블 문제
- 1계층 문제 가능성 높음
- 신호 자체가 안 전달됨

### 예시 2: 같은 사무실 PC끼리 안 붙음
- 2계층 문제 가능성
- MAC, 스위치, VLAN

### 예시 3: 외부 서버에 안 감
- 3계층 문제 가능성
- IP, 라우팅, 게이트웨이

### 예시 4: 웹은 열리는데 API 요청만 실패
- 4계층/7계층 문제 가능성
- 포트, TCP, HTTP, 인증 문제

### 예시 5: 한글이 깨짐
- 6계층 느낌
- 인코딩 문제

## 자주 하는 실수
- OSI 7계층을 무조건 암기만 하고 흐름을 모름
  - 바로잡기: OSI 7계층은 암기보다 송신 데이터가 상위 계층에서 하위 계층으로 캡슐화되고 수신 시 역캡슐화되는 흐름으로 이해해야 한다.
- TCP/IP 모델과 헷갈림
  - 바로잡기: TCP/IP 모델은 실제 인터넷 구현에 가까운 4계층 모델이고, OSI는 개념을 세분화한 7계층 참조 모델이다.
- 스위치와 라우터의 계층을 혼동함
  - 바로잡기: 스위치는 주로 2계층 MAC 주소 기반 전달, 라우터는 3계층 IP 주소 기반 경로 선택을 담당한다.
- HTTP가 4계층이라고 착각함
  - 바로잡기: HTTP는 애플리케이션 계층 프로토콜이고, 전송 계층에서는 보통 TCP 또는 QUIC/UDP를 사용한다.
- DNS를 3계층으로 생각함
  - 바로잡기: DNS는 도메인 이름을 IP 주소 등으로 해석하는 애플리케이션 계층 프로토콜이다.
- 세션 계층을 실제 구현과 너무 단순하게만 이해함
  - 바로잡기: OSI 세션 계층은 개념적으로 연결 세션 관리 역할을 설명하지만, 실제 인터넷 구현에서는 TCP/IP와 애플리케이션 라이브러리에 흩어져 구현되는 경우가 많다.
- 계층을 외워도 캡슐화 흐름을 못 그림
  - 바로잡기: 캡슐화는 상위 계층 데이터에 각 계층의 헤더가 붙어 내려가는 과정이다.

## 계층별 대표 키워드
- 1계층: 비트, 신호, 케이블
- 2계층: 프레임, MAC, 스위치
- 3계층: 패킷, IP, 라우터
- 4계층: 세그먼트, 포트, TCP/UDP
- 5계층: 세션
- 6계층: 인코딩, 압축, 암호화
- 7계층: HTTP, DNS, SMTP

## 디버깅할 때 보는 순서
1. 물리 연결이 살아 있는가
2. 같은 네트워크끼리 통신이 되는가
3. 라우팅이 맞는가
4. 포트가 열려 있는가
5. 프로토콜이 맞는가
6. 응용 계층에서 상태 코드나 에러가 나는가
7. 인코딩/압축/암호화 문제는 없는가

## 비교해서 이해하기

### 상위 계층 vs 하위 계층
- 상위 계층: 사람이 체감하는 서비스에 가깝다
- 하위 계층: 실제 전송과 물리 신호에 가깝다

### 응용 계층 vs 전송 계층
- 응용: HTTP, DNS 같은 서비스 규칙
- 전송: TCP/UDP, 포트, 신뢰성

### 데이터 링크 vs 네트워크
- 데이터 링크: 같은 네트워크 내부
- 네트워크: 다른 네트워크 간 이동

## 면접 포인트
면접에서는 계층 이름을 줄줄 외우는 것보다 **어떤 역할을 하는지**를 말해야 한다.

### 꼭 말할 수 있어야 하는 것
- OSI 7계층은 네트워크를 역할별로 나눈 개념 모델이다
- 아래로 갈수록 물리적 전송에 가깝고, 위로 갈수록 사용자 서비스에 가깝다
- IP는 3계층, TCP/UDP는 4계층, HTTP/DNS는 7계층이다
- 스위치는 2계층, 라우터는 3계층에서 이해하면 좋다
- 캡슐화/역캡슐화 흐름을 말할 수 있어야 한다

### 한 번에 말하는 답변 예시
> OSI 7계층은 네트워크 통신을 7개의 역할로 나눠서 보는 개념 모델입니다. 1계층은 신호 전달, 2계층은 MAC 기반 프레임 전달, 3계층은 IP와 라우팅, 4계층은 TCP/UDP와 포트, 7계층은 HTTP나 DNS 같은 응용 프로토콜을 다룹니다. 이 계층 구조를 알면 문제를 어느 단계에서 찾아야 하는지 쉽게 좁힐 수 있습니다.

### 꼬리 질문 예시
- 왜 계층을 나누나요?
- OSI와 TCP/IP 모델 차이는 뭔가요?
- 스위치와 라우터는 각각 몇 계층인가요?
- 캡슐화가 뭐예요?
- HTTP는 왜 7계층인가요?

- 아래는 신호, 위는 서비스에 가깝다
- IP는 3계층, TCP는 4계층, HTTP는 7계층으로 기억하면 쉽다
