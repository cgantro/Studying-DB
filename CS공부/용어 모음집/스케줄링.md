# 스케줄링

## 정의
준비된 작업 중 어떤 작업에 CPU를 줄지 결정하는 정책과 과정이다. 단순히 "누가 먼저 실행되는가"만 정하는 것이 아니라 응답 시간, 처리량, 공정성, 실시간 보장, 그룹별 CPU 제한까지 함께 조절한다.

## 사용 환경
여러 실행 가능한 작업 중 어떤 작업에 CPU를 줄지 결정하고 평가할 때 사용된다. 데스크톱 UI, 서버, 컨테이너, 배치 처리, [[용어 모음집/실시간 시스템|실시간 시스템]]처럼 요구사항이 서로 다른 환경에서 각각 다른 기준을 적용한다.

## 사용 조건
준비 큐, 우선순위, 예상 실행 시간, time quantum 같은 정책 입력이 필요하다. 현대 OS에서는 여기에 nice 값, virtual runtime, virtual deadline, cgroup CPU quota, 실시간 runtime/deadline/period 같은 정보도 함께 사용될 수 있다.

## 예시
Round Robin은 일정 시간마다 CPU를 다음 작업에 넘긴다. Linux의 CFS는 CPU를 상대적으로 덜 받은 작업을 우선 실행하고, EEVDF는 CPU를 받을 자격이 있는 작업 중 virtual deadline이 빠른 작업을 고른다.

## 주의점
- 평균 대기 시간을 줄이는 정책이 항상 공정한 것은 아니다.
- 응답성과 처리량은 서로 충돌할 수 있다.
- 우선순위 정책은 starvation을 만들 수 있어 aging 같은 보완책이 필요하다.
- 실제 OS는 FCFS, SJF, RR, Priority 중 하나만 단순히 쓰지 않는다.
- 실시간 정책은 "무조건 빠름"이 아니라 "정해진 시간 안에 예측 가능하게 실행됨"이 핵심이다.
