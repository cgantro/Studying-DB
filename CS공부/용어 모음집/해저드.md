# 해저드

## 정의
파이프라인 흐름을 방해하는 구조, 데이터, 제어 위험이다.

## 사용 환경
CPU가 여러 명령어의 실행 단계를 겹쳐 처리량을 높일 때 사용된다.

## 사용 조건
명령어 간 데이터 의존성, 자원 충돌, 분기 흐름을 관리할 수 있어야 한다.

## 예시
이전 명령어 결과가 아직 나오지 않았는데 다음 명령어가 그 값을 읽으려 하면 데이터 해저드가 생긴다.

## 주의점
- 파이프라이닝은 한 명령어의 지연 시간을 항상 줄이지는 않는다.
- 해저드가 생기면 stall이나 flush가 필요하다.
- 분기 예측 실패는 깊은 파이프라인에서 비용이 커진다.
