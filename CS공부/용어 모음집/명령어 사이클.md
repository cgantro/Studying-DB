# 명령어 사이클

## 정의
CPU가 명령어를 가져오고 해석하고 실행하고 결과를 기록하는 반복 흐름이다.

## 사용 환경
CPU가 프로그램을 실행할 때 매 명령어마다 반복된다. fetch, decode, execute, memory access, write-back 같은 단계로 설명한다.

## 사용 조건
PC가 유효한 명령어 주소를 가리키고, 명령어를 가져올 수 있으며, 필요한 피연산자와 실행 장치가 준비되어야 한다.

## 예시
fetch, decode, execute, memory, write-back 단계로 설명할 수 있다.

## 주의점
- 실제 CPU는 파이프라이닝 때문에 여러 명령어 사이클 단계가 겹쳐 실행될 수 있다.
- 명령어 하나의 흐름과 전체 처리량 향상 기법을 구분해야 한다.
- 분기나 인터럽트가 발생하면 PC 흐름이 바뀔 수 있다.
