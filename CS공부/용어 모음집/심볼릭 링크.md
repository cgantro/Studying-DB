# 심볼릭 링크

## 정의
다른 파일 경로를 가리키는 별도 파일이다.

## 사용 환경
파일 이름, 권한, 메타데이터, 실제 데이터 블록을 저장 장치 위에 구성할 때 사용된다.

## 사용 조건
디렉터리 엔트리와 inode, 데이터 블록, 권한 정보가 일관되게 관리되어야 한다.

## 예시
원본 경로가 사라지면 깨진 링크가 될 수 있다.

## 주의점
- 파일 이름은 보통 inode가 아니라 디렉터리 엔트리에 있다.
- 하드 링크와 심볼릭 링크는 inode 공유 여부가 다르다.
- 저널링은 파일 시스템 일관성 복구를 돕지만 모든 데이터 손실을 막지는 않는다.
