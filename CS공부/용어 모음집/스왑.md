# 스왑

## 정의
메모리 부족 시 일부 페이지를 디스크로 내보내거나 다시 가져오는 기법이다.

## 사용 환경
운영체제가 프로세스별 주소 공간을 제공하고 제한된 물리 메모리를 효율적으로 관리할 때 사용된다.

## 사용 조건
주소 변환 정보, 권한 비트, 페이지 상태, 교체 정책, 물리 프레임 관리가 함께 동작해야 한다.

## 예시
잘 쓰지 않는 페이지가 swap 영역으로 밀려날 수 있다.

## 주의점
- TLB miss와 페이지 폴트는 다른 비용과 원인을 가진다.
- 스왑이 많아지면 스래싱으로 시스템이 급격히 느려질 수 있다.
- COW와 요구 페이징은 page fault를 정상적인 지연 처리 도구로 사용한다.
