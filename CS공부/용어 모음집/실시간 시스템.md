# 실시간 시스템

## 정의
실시간 시스템(real-time system)은 작업의 계산 결과뿐 아니라 정해진 시간 안에 결과가 나오는지도 중요한 시스템이다.

## 사용 환경
자동차 제어, 항공 제어, 의료 장비, 산업 제어, 오디오/비디오 처리, 게임, 화상 회의처럼 시간 지연이 시스템 실패나 품질 저하로 이어질 수 있는 환경에서 사용된다.

## 사용 조건
작업별 deadline, runtime, period, 우선순위, 최악 실행 시간 같은 시간 제약을 알아야 한다. 평균 성능보다 deadline miss 여부와 예측 가능성이 중요하다.

## 예시
동영상 재생에서는 각 프레임이 출력 시점 안에 준비되어야 한다. 늦게 준비된 프레임은 맞는 데이터여도 이미 사용할 수 없어서 끊김이나 프레임 드롭이 생긴다.

## 주의점
- 실시간 시스템은 단순히 빠른 시스템이 아니다.
- Hard real-time은 deadline을 놓치면 실패이고, Soft real-time은 품질 저하가 발생한다.
- 처리량이 높아도 특정 작업이 deadline을 놓치면 실시간 요구사항을 만족하지 못할 수 있다.
- 일반 스케줄링의 공정성과 실시간 스케줄링의 deadline 준수는 서로 다른 목표다.
