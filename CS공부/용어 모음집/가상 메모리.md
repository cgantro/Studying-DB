# 가상 메모리

## 정의
프로세스마다 독립적이고 큰 주소 공간을 가진 것처럼 보이게 하고, 실제로는 운영체제와 MMU가 가상 주소를 물리 주소로 변환해 RAM에 접근하게 하는 메모리 관리 기법이다.

## 사용 환경
운영체제가 여러 프로세스를 동시에 실행하면서 프로세스 간 메모리 보호, 주소 공간 분리, 요구 페이징, 스왑, 효율적인 메모리 배치를 제공할 때 사용된다.

## 사용 조건
가상 주소를 물리 주소로 바꾸는 페이지 테이블, 주소 변환을 빠르게 하는 TLB, 페이지 상태와 권한 비트, 물리 프레임 관리, 페이지 폴트 처리, 필요 시 스왑 공간이 함께 동작해야 한다.

## 예시
프로세스 A와 프로세스 B가 둘 다 가상 주소 `0x1000`을 사용할 수 있다. 하지만 A의 `0x1000`은 물리 프레임 12번으로, B의 `0x1000`은 물리 프레임 30번으로 매핑될 수 있다. 그래서 두 프로세스는 같은 주소를 쓰는 것처럼 보여도 서로의 메모리를 덮어쓰지 않는다.

## 주의점
- 가상 메모리는 실제 RAM을 마법처럼 늘리는 기술이 아니다.
- 프로세스가 보는 주소는 가상 주소이고, 실제 RAM 위치는 물리 주소다.
- 페이지 폴트는 무조건 오류가 아니라 요구 페이징 과정에서 정상적으로 발생할 수 있다.
- 스왑이 많아지면 디스크 접근이 늘어 스래싱이 발생할 수 있다.
- 가상 메모리는 프로세스 주소 공간에 관한 개념이고, VM/컨테이너 가상화는 실행 환경 격리에 관한 개념이다.
