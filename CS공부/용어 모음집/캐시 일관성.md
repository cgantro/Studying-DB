# 캐시 일관성

## 정의
멀티코어에서 여러 캐시가 같은 메모리 값을 모순 없이 보게 하는 성질이다.

## 사용 환경
CPU와 메모리 사이의 속도 차이를 줄이고, 데이터 재사용 특성을 이용해 성능을 높일 때 사용된다.

## 사용 조건
최근 접근한 데이터나 가까운 주소를 다시 사용할 가능성이 있어야 캐시 효과가 커진다.

## 예시
한 코어가 값을 수정하면 다른 코어의 캐시 라인을 무효화해야 할 수 있다.

## 주의점
- 캐시 hit는 빠르지만 miss는 큰 지연을 만든다.
- 멀티코어에서는 캐시 일관성 비용이 추가된다.
- False sharing은 논리적으로 다른 데이터도 같은 캐시 라인 때문에 충돌하게 만든다.
