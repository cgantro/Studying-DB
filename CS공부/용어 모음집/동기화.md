# 동기화

## 정의
여러 실행 흐름이 공유 자원을 안전하게 사용하도록 순서와 접근을 조절하는 것이다.

## 사용 환경
여러 스레드나 프로세스가 공유 데이터에 접근하는 프로그램에서 사용된다.

## 사용 조건
공유 자원과 임계 구역을 정확히 식별하고, 접근 전후에 적절한 동기화 연산을 수행해야 한다.

## 예시
공유 카운터를 수정할 때 락으로 임계 구역을 보호한다.

## 주의점
- 동기화를 빼먹으면 race condition이 생긴다.
- 동기화를 과하게 걸면 병렬성이 줄고 응답 시간이 늘어난다.
- 락 획득 순서가 불명확하면 교착 상태가 생길 수 있다.
