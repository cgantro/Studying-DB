# 생산자 소비자 문제

## 정의
생산자가 데이터를 만들고 소비자가 데이터를 가져가는 구조에서, 제한된 버퍼를 안전하게 공유하기 위한 대표 동기화 문제다.

## 사용 환경
로그 큐, 작업 큐, 메시지 큐, 영상 프레임 버퍼, 웹 서버 요청 큐처럼 한쪽은 데이터를 넣고 다른 쪽은 데이터를 꺼내 처리하는 구조에서 사용된다.

## 사용 조건
버퍼 크기가 제한되어 있고, 여러 스레드가 같은 버퍼에 접근할 수 있어야 한다. 버퍼 자료구조 보호와 버퍼 상태 대기를 구분해야 한다.

## 예시
요청을 받는 스레드는 작업 큐에 요청을 넣고, worker 스레드는 큐에서 요청을 꺼내 처리한다. 큐가 가득 차면 생산자는 기다리고, 큐가 비면 소비자는 기다린다.

## 주의점
- 버퍼 자료구조 접근은 뮤텍스로 보호한다.
- 빈 칸 개수는 `empty` 세마포어로 관리한다.
- 찬 칸 개수는 `full` 세마포어로 관리한다.
- `mutex`를 먼저 잡고 `empty`나 `full`을 기다리면 데드락이 생길 수 있다.
- 조건 변수를 사용할 때는 깨어난 뒤에도 조건을 다시 확인해야 한다.
