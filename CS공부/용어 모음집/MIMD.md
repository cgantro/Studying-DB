# MIMD

## 정의
여러 처리 장치가 서로 다른 명령을 서로 다른 데이터에 실행하는 방식이다.

## 사용 환경
여러 작업을 동시에 처리하거나 처리량을 높이려는 시스템에서 사용된다.

## 사용 조건
작업이 충분히 분할 가능하고, 공유 자원 경합과 결과 병합 비용이 병렬화 이득보다 작아야 한다.

## 예시
멀티코어 서버에서 각 코어가 다른 요청을 처리하는 경우가 예다.

## 주의점
- 동시성은 한 코어에서도 가능하지만 병렬성은 실제 동시 실행 자원이 필요하다.
- Amdahl의 법칙 때문에 직렬 구간이 성능 향상의 한계를 만든다.
- SIMD는 같은 연산을 여러 데이터에 적용할 때 특히 효과적이다.
