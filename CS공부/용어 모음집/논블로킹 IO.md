# 논블로킹 IO

## 정의
작업이 즉시 완료되지 않아도 호출이 바로 반환되는 입출력 방식이다.

## 사용 환경
프로그램이 느린 입출력 작업을 기다리는 방식과 운영체제의 I/O 최적화를 설명할 때 사용된다.

## 사용 조건
파일 디스크립터 상태, 커널 버퍼, 이벤트 통지 방식, 장치 처리 속도에 따라 적절한 모델을 선택해야 한다.

## 예시
데이터가 없으면 에러나 상태값을 반환하고 나중에 다시 시도한다.

## 주의점
- 블로킹/논블로킹은 호출이 기다리는지의 문제이고, 동기/비동기는 완료 통지를 누가 어떻게 받는지의 문제다.
- 버퍼 캐시는 읽기 성능을 높이지만 최신성 관리가 필요하다.
- 스풀링은 느린 장치 작업을 큐에 쌓아 처리한다.
