# 운영 설정과 성능

## 관측
- Actuator health/metrics/prometheus 노출
- 요청 지연, 에러율, GC, DB pool 지표를 기본 수집

## 성능
- 커넥션 풀(HikariCP) 크기는 DB/트래픽 기준으로 조정
- 직렬화 비용과 N+1 쿼리를 우선 점검
- 캐시(Redis)와 DB 책임 분리

## 안정성
- timeout/retry/circuit-breaker 정책 명시
- graceful shutdown, readiness/liveness probe 적용

## 버전 운영
- 3.5.13 기준으로 의존성 잠금, 배포 전 회귀 테스트 자동화
