## 핵심 개념
- 용어는 한 문장 정의로 정리한다
- 헷갈리는 용어는 비교 짝을 같이 적는다
- 약어는 풀네임과 함께 적는다

## 세부 정리
- 데이터 표현: 컴퓨터가 값을 비트 패턴으로 저장하고 해석하는 규칙
- 2의 보수: 음수를 표현하기 위해 비트를 반전하고 1을 더하는 방식
- 오버플로우: 표현 가능한 범위를 넘어 값이 깨지는 현상
- 부동소수점: 실수를 제한된 비트로 근사 표현하는 방식
- 엔디언: 여러 바이트 값을 메모리에 저장하는 순서
- 시스템 콜: 사용자 프로그램이 커널 기능을 요청하는 인터페이스
- 커널 모드: 운영체제가 하드웨어와 핵심 자원에 접근할 수 있는 특권 모드
- 인터럽트: 장치나 타이머가 CPU에게 사건을 알리는 신호
- DMA: CPU 개입을 줄이고 장치와 메모리가 직접 데이터를 주고받는 방식
- CPI: 명령어 하나를 처리하는 데 필요한 평균 클럭 수
- IPC: 한 클럭에 처리할 수 있는 평균 명령어 수
- 파이프라인 hazard: 명령어 단계 겹침을 방해하는 구조/데이터/제어 위험
- PC: 다음에 실행할 명령어 주소를 가리키는 레지스터
- IR: 현재 가져온 명령어를 담는 레지스터
- namespace: 컨테이너가 보는 자원 이름 공간을 분리하는 커널 기능
- cgroup: 프로세스 그룹의 CPU/메모리 같은 자원 사용을 제한하고 측정하는 기능
- 예: 프로세스 vs 스레드
- 예: 정적 할당 vs 동적 할당
- 예: HTTP vs HTTPS
- 예: SQL vs NoSQL
- 예: 정규화 vs 인덱스
