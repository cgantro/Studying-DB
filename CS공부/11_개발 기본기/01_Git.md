# Git

## 한 줄 정의
변경 이력을 안전하게 관리하고 여러 사람이 협업할 수 있게 하는 분산 버전 관리 도구다.

## 세부 정리
### 핵심 개념
- commit은 변경 스냅샷이다
- branch는 작업 흐름을 분리한다
- merge와 rebase는 이력을 합치는 방식이 다르다

### 자주 쓰는 흐름
- 작업 전 pull
- 기능 브랜치 생성
- 커밋 누적
- PR 또는 merge

### 주의할 점
- rebase는 히스토리를 다시 쓴다
- 충돌은 내용보다 맥락을 보고 푼다
- 커밋 메시지는 나중에 읽어도 의미가 있어야 한다

## 예시
- 실험 기능을 branch에서 만들고 main에 합친다

## 면접 포인트
- merge와 rebase 차이를 이력 관점으로 설명한다

## 더 자세한 설명
- 개발 기본기는 협업과 운영을 버티게 해 주는 실전 기술이다.
- Git, 테스트, 디버깅, 로깅, 배포는 실제 프로젝트에서 매일 만나는 도구들이다.
- 기본기가 약하면 기능은 만들 수 있어도 유지보수가 어려워진다.
- 좋은 개발자는 코드를 쓰는 사람을 넘어, 시스템을 관리하는 사람에 가깝다.

## 초보자용 한 줄 기억법
- Git: Git를 "일상 언어"로 다시 바꿔 말해 보는 노트다.
## 더 깊게 보기
- 개발 기본기는 협업, 안정성, 운영을 버티게 해 주는 실전 기술이다.
- Git은 파일 저장이 아니라 "변경의 역사"를 다루는 시스템이다.
- 브랜치는 실험 공간이고, 커밋은 되돌릴 수 있는 체크포인트다.
- merge는 합치기, rebase는 이력 정리라는 이해으로 보면 쉽다.
- 협업에서는 충돌을 빨리 확인하고 작은 단위로 합치는 것이 안전하다.

## 초보자용 기억법
- Git: "이게 왜 필요한지"까지 설명할 수 있으면 이해한 것이다.
