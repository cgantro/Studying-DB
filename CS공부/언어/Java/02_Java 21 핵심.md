# Java 21 핵심

## 주요 기능
- `virtual thread`: I/O 중심 워크로드에서 동시성 처리량 개선
- `record pattern` / `switch pattern matching`: 복합 분기 가독성 향상
- `sequenced collection`: 컬렉션 API 일관성 강화

## 실무 포인트
- 블로킹 I/O가 많은 서비스는 virtual thread 검토 가치가 높다.
- 기존 thread pool 기반 코드와 혼용 시 모니터링 지표를 분리해 비교해야 한다.
- 마이그레이션은 단계적으로 진행하고 부하 테스트로 tail latency를 확인한다.

## 주의점
- CPU-bound 업무는 virtual thread 이점이 제한적일 수 있다.
- 라이브러리별 thread-local 사용 패턴을 점검해야 한다.
