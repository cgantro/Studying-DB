# 선택 기준과 실무 예시

## 선택 기준
- 정합성 요구 수준
- 읽기/쓰기 비율과 트래픽 패턴
- 질의 복잡도
- 운영/백업/복구 난이도

## 실무 예시
- 주문/결제: RDB 우선 (트랜잭션 필수)
- 세션/캐시: Key-Value
- 로그/이벤트 원문: Document 또는 Column-family

## 하이브리드 전략
- OLTP는 RDB, 캐시는 Redis, 검색은 별도 엔진
- 데이터 책임을 분리하고 동기화 경계를 명확히 설정

## 실패 사례
- 트래픽만 보고 NoSQL 선택 -> 정합성 요구 미스매치로 재설계
