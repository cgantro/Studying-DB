# Spring Boot 운영 / 외부 연동 / 커넥션 풀 - 답변 정리

---

## 1. `RestTemplate`, `RestClient`, `WebClient`의 차이는 무엇인가요?
### 답변(면접용)
- `RestTemplate`은 전통적인 동기(blocking) HTTP 클라이언트이고, 신규 개발에서는 점진적으로 대체되는 흐름이다.
- `RestClient`는 동기 방식이지만 좀 더 현대적인 API(빌더/플루언트)를 제공하는 선택지다.
- `WebClient`는 리액티브 기반(논블로킹) 클라이언트로, 대량 동시 I/O 처리나 스트리밍에 유리하다.
- 운영에서는 “리액티브 스택 도입 비용 vs 동시성 이득”을 같이 본다.

---

## 2. Spring Boot에서 외부 API 호출 시 timeout을 어디에 설정하나요?
### 답변(면접용)
- 핵심은 `connect timeout`(연결 성립)과 `read timeout`(응답 바이트 수신) 등 타임아웃 종류를 구분하는 것이다.
- 사용하는 클라이언트(`RestTemplate`/`RestClient`/`WebClient`)에 따라 설정 위치가 달라진다(커넥터/HTTP client 레벨).
- 전역 기본값을 두고, 외부 의존성별로 예외적으로 늘리거나 줄이는 식으로 관리한다.

---

## 3. 외부 API 호출 실패 시 retry를 무조건 적용하면 어떤 문제가 생기나요?
### 답변(면접용)
- 실패 원인이 “일시 장애”가 아니라 “지속 장애/부하/버그”면 재시도가 장애를 증폭시킨다(폭주).
- 타임아웃/재시도 조합이 잘못되면 스레드/커넥션 풀이 고갈되어 내 서비스까지 느려진다.
- 재시도는 백오프/지터, 최대 횟수, 실패 유형(5xx, 네트워크 에러 등) 기준을 같이 둬야 한다.

---

## 4. Circuit Breaker는 어떤 문제를 해결하기 위한 패턴인가요?
### 답변(면접용)
- 외부 의존성이 불안정할 때 “계속 호출해서 더 망가지는 상황”을 막기 위한 패턴이다.
- 실패율/타임아웃 등을 기준으로 일정 기간 호출을 차단(open)하고, 이후 일부만 허용(half-open)해 회복 여부를 확인한다.
- 목적은 장애 전파 차단, 자원 보호(스레드/커넥션), 빠른 실패로 사용자 경험을 안정화하는 것이다.

---

## 5. 서버가 외부 API에 의존할 때 장애 전파를 막으려면 어떻게 설계해야 하나요?
### 답변(면접용)
- 타임아웃/재시도/서킷브레이커를 기본으로 두고, 실패 시 동작(폴백/기능 제한)을 정의한다.
- Bulkhead(격리)로 외부 호출용 스레드/커넥션/큐를 분리해 내부 요청을 보호한다.
- 외부 의존성 상태(성공률/지연/에러)를 지표화하고, 알람과 차단 기준을 운영에 포함한다.

---

## 6. HTTP Connection Pool을 사용하는 이유는 무엇인가요?
### 답변(면접용)
- 매 요청마다 TCP/TLS 연결을 새로 맺는 비용을 줄이기 위해서다(핸드셰이크 비용).
- 커넥션 재사용으로 레이턴시를 줄이고, 동시에 처리 가능한 외부 호출 수를 제어할 수 있다.
- 대신 풀 크기/대기 시간/유휴 커넥션 관리가 필요하다(설정 미스 시 병목).

---

## 7. Connection Pool을 너무 크게 잡으면 어떤 문제가 생기나요?
### 답변(면접용)
- 커넥션 수가 늘면 메모리/파일 디스크립터/스레드 등 자원을 과도하게 소비할 수 있다.
- 외부 서버가 감당 못하면 오히려 실패율이 늘고, 재시도까지 겹치면 폭주가 된다.
- “풀을 키우면 해결”이 아니라, 병목(외부 지연/내부 처리)을 먼저 구분해야 한다.

---

## 8. HikariCP는 어떤 역할을 하나요?
### 답변(면접용)
- JDBC `DataSource`에 대한 커넥션 풀 구현체로, DB 커넥션을 생성/재사용/반납하는 역할을 한다.
- 풀의 최대 커넥션 수, 대기 시간, 유휴/수명 같은 정책을 설정해 DB 자원을 보호한다.
- 애플리케이션 성능/안정성은 “쿼리 지연 + 풀 설정 + 트랜잭션 길이”에 크게 좌우된다.

---

## 9. DB Connection Pool이 고갈되면 애플리케이션에는 어떤 증상이 나타나나요?
### 답변(면접용)
- DB 쿼리를 실행하려는 요청이 커넥션을 못 얻고 대기하면서 응답 시간이 길어진다.
- 스레드 풀(서버 worker)까지 함께 막히면 전체 API가 느려지거나 타임아웃이 연쇄적으로 발생한다.
- 로그/지표에서는 커넥션 획득 대기, 타임아웃, 요청 큐 적체 같은 형태로 나타난다.

---

## 10. Spring Boot Actuator는 운영 관점에서 어떤 용도로 사용하나요?
### 답변(면접용)
- 애플리케이션의 상태(health), 지표(metrics), 환경/설정 일부를 운영자가 확인할 수 있게 엔드포인트를 제공한다.
- 모니터링/알람(예: 지연, 에러율, 스레드/커넥션)과 연동하기 위한 관측 지점을 만든다.
- 보안상 노출 범위를 최소화하고(인증/권한/네트워크), 필요한 항목만 공개하는 식으로 운영한다.
