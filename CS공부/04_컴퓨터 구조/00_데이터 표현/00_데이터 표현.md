# 데이터 표현

## 한 줄 정의
데이터 표현은 **컴퓨터가 숫자, 문자, 명령, 메모리 값을 비트 패턴으로 저장하고 해석하는 방식**이다.

쉽게 말하면, 사람이 보는 `10`, `-3`, `3.14`, `A` 같은 값을 컴퓨터가 **0과 1의 약속**으로 바꿔 다루는 규칙이다.

---

## 왜 배우는가
- CPU와 메모리는 결국 비트만 다룬다.
- 정수 오버플로우, 부동소수점 오차, 엔디언 문제는 실무 버그로도 자주 나온다.
- 명령어, 레지스터, 메모리, 네트워크 바이트 순서를 이해하는 출발점이다.
- 면접에서 2의 보수, 오버플로우, float 오차는 기본 꼬리 질문으로 이어진다.

---

## 전체 흐름

```text
사람이 보는 값
   ↓
인코딩 규칙 적용
   ↓
비트 패턴 저장
   ↓
CPU가 연산 또는 해석
   ↓
다시 사람이 이해하는 값으로 표현
```

---

## 핵심 개념
- **비트(bit)**: 0 또는 1 하나
- **바이트(byte)**: 보통 8비트
- **진법**: 같은 값을 어떤 자리수 체계로 볼지 정하는 방식
- **2의 보수**: 음수를 표현하는 대표 방식
- **오버플로우**: 표현 가능한 범위를 넘어선 상태
- **부동소수점**: 실수를 근사해서 표현하는 방식
- **엔디언**: 여러 바이트 값을 메모리에 어떤 순서로 둘지 정하는 방식
- **정렬(alignment)**: 데이터를 효율적으로 접근하기 위해 주소 경계를 맞추는 방식

---

## 데이터 표현을 모르면 생기는 문제
- `int` 범위를 넘어 이상한 값이 나온다
- `0.1 + 0.2`가 정확히 `0.3`이 되지 않는다
- 파일/네트워크 데이터를 다른 시스템에서 잘못 해석한다
- 구조체 크기가 예상보다 커진다
- 바이너리 디버깅에서 메모리 덤프를 읽기 어렵다

---

## 비교해서 이해하기

| 주제 | 핵심 질문 |
|---|---|
| 진법 | 같은 값을 2진수/10진수/16진수로 어떻게 보나? |
| 보수 | 음수를 비트로 어떻게 표현하나? |
| 오버플로우 | 범위를 넘으면 왜 값이 돌아가나? |
| 부동소수점 | 실수를 왜 정확히 저장하지 못하나? |
| 엔디언 | 여러 바이트를 어떤 순서로 저장하나? |
| 정렬 | 왜 구조체에 빈 공간이 생기나? |

---

## 실전 예시

### 예시 1: 정수 범위 초과
```text
8비트 unsigned 정수
255 + 1 = 0
```

표현 가능한 범위를 넘어 다시 처음으로 돌아가는 것처럼 보일 수 있다.

### 예시 2: 실수 오차
```text
0.1 + 0.2 = 0.30000000000000004
```

10진 소수를 2진 부동소수점으로 정확히 표현하지 못해서 생기는 현상이다.

### 예시 3: 엔디언
```text
값: 0x12345678
big-endian:    12 34 56 78
little-endian: 78 56 34 12
```

---

## 면접 포인트
면접에서는 데이터 표현을 "2진수"로만 끝내지 말고, **범위, 해석 규칙, 시스템 간 호환성, 성능**까지 연결해 말해야 한다.

### 꼭 말할 수 있어야 하는 것
- 컴퓨터는 모든 데이터를 비트 패턴으로 저장한다
- 같은 비트라도 해석 규칙에 따라 값이 달라진다
- 2의 보수는 음수를 표현하는 대표 방식이다
- 정수는 범위가 있고, 범위를 넘으면 오버플로우가 발생한다
- 부동소수점은 실수를 근사하므로 비교에 주의해야 한다
- 엔디언은 바이트 저장 순서 문제다

### 한 번에 말하는 답변 예시
> 데이터 표현은 값을 비트 패턴으로 저장하고 해석하는 규칙입니다. 정수는 보통 2의 보수로 음수를 표현하고, 표현 범위를 넘으면 오버플로우가 발생할 수 있습니다. 실수는 부동소수점으로 근사 표현하므로 오차가 생길 수 있고, 여러 바이트 값은 엔디언에 따라 메모리 저장 순서가 달라질 수 있습니다.

---

## 체크리스트
- [ ] 2진수, 10진수, 16진수 관계를 설명할 수 있다
- [ ] 2의 보수로 음수를 표현하는 이유를 안다
- [ ] signed와 unsigned의 차이를 안다
- [ ] 정수 오버플로우가 왜 생기는지 설명할 수 있다
- [ ] 부동소수점 오차가 왜 생기는지 안다
- [ ] big-endian과 little-endian 차이를 안다
- [ ] 데이터 정렬과 패딩의 이해을 안다

---

## 초보자용 한 줄 기억법
데이터 표현은 **"컴퓨터가 0과 1을 어떤 약속으로 숫자와 글자로 읽을지 정하는 규칙"**이다.
