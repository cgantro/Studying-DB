# ALU

## 한 줄 정의
[[용어 모음집/ALU|ALU]](Arithmetic Logic Unit)는 **[[용어 모음집/CPU|CPU]] 안에서 산술 연산, 논리 연산, 비교 연산을 수행하는 계산 장치**다.

---

## 핵심 개념
- 산술 연산: 덧셈, 뺄셈 등
- 논리 연산: AND, OR, XOR, NOT 등
- 시프트 연산: 비트를 왼쪽/오른쪽으로 이동
- 비교 연산: 두 값의 크기나 같음을 판단
- 플래그 갱신: 연산 결과 상태를 상태 [[용어 모음집/레지스터|레지스터]]에 반영

---

## ALU 입력과 출력

```text
레지스터 A ─┐
            ├→ ALU → 결과 → 레지스터
레지스터 B ─┘
       ↑
   제어장치가 연산 종류 선택
```

[[용어 모음집/제어장치|제어장치]]는 [[용어 모음집/명령어|명령어]] 해석 결과에 따라 ALU가 `add`를 할지, `and`를 할지, 비교를 할지 정한다.

---

## ALU가 하는 일

| 종류 | 예시 | 쓰이는 곳 |
|---|---|---|
| 산술 연산 | add, sub | 숫자 계산, 주소 계산 |
| 논리 연산 | and, or, xor | 비트 마스크, 조건 처리 |
| 시프트 | left shift, right shift | 곱셈/나눗셈 최적화, 비트 조작 |
| 비교 | compare, test | 조건문, 반복문, 분기 |

---

## 플래그와 분기
비교 명령은 실제로 값을 저장하지 않고 플래그만 갱신하는 경우가 많다.

```text
compare r1, r2
   ↓
Zero Flag / Sign Flag 등 갱신
   ↓
조건 분기 명령이 플래그 확인
```

고급 언어의 `if`, `while`은 내부적으로 비교와 조건 분기로 바뀐다.

---

## 오버플로우와 ALU
ALU는 정해진 비트 폭 안에서 계산한다. 그래서 결과가 범위를 넘으면 오버플로우가 발생할 수 있다.

```text
8비트 signed
127 + 1 → 표현 범위 초과
```

이때 Overflow Flag나 Carry Flag 같은 상태가 중요해진다.

---

## ALU와 FPU
일반적인 ALU는 정수와 논리 연산 중심이다. 실수 연산은 보통 FPU(Floating Point Unit)나 별도 실행 유닛이 담당한다.

| 구분 | 주로 담당 |
|---|---|
| ALU | 정수, 논리, 비교, 시프트 |
| FPU | 부동소수점 실수 연산 |

---

## 자주 하는 실수
- ALU가 모든 CPU 작업을 다 한다고 생각함
  - 바로잡기: ALU는 산술/논리 연산 담당이고, 명령어 해석은 제어장치, 값 보관은 레지스터, 메모리 접근은 로드/스토어 경로가 담당한다.
- ALU와 제어장치를 섞음
  - 바로잡기: ALU는 실제 연산을 수행하고, 제어장치는 어떤 연산과 데이터 이동을 할지 지시한다.
- 비교 연산이 항상 결과 값을 저장한다고 생각함
  - 바로잡기: 비교 연산은 결과 값을 별도 저장하기보다 Zero/Sign/Carry/Overflow 같은 플래그를 바꿔 조건 분기에 쓰이는 경우가 많다.
- Carry와 Overflow를 같은 의미로 생각함
  - 바로잡기: Carry는 unsigned 연산의 자리올림/빌림과 관련되고, Overflow는 signed 표현 범위를 넘었는지와 관련된다.
- 실수 연산까지 항상 ALU가 담당한다고 생각함
  - 바로잡기: 정수 산술/논리는 ALU가 담당하고, 실수 연산은 별도의 FPU 또는 벡터 연산 장치가 담당하는 구조가 흔하다.

---

## 면접 포인트
> ALU는 CPU 내부에서 산술 연산과 논리 연산을 수행하는 장치입니다. 레지스터에서 값을 받아 덧셈, 뺄셈, AND, OR, 비교, 시프트 같은 연산을 수행하고 결과를 다시 레지스터에 저장합니다. 연산 결과에 따라 Zero, Carry, Overflow 같은 플래그가 갱신될 수 있으며, 조건 분기 명령은 이런 플래그를 활용합니다.

---

- [ ] 플래그와 조건 분기 관계를 안다
- [ ] Carry와 Overflow 차이를 이해적으로 안다

## 보강 설명
CPU 주제는 “명령어가 실제로 어떻게 실행되는가”를 기준으로 연결해서 보는 것이 좋다. [[용어 모음집/PC|PC]]가 명령어 주소를 가리키고, 제어장치가 명령어를 해석하며, 레지스터와 ALU가 값을 주고받고, 결과가 다시 레지스터나 메모리에 반영된다.

## 구체적인 예시
예를 들어 `a = b + c` 같은 코드는 메모리에서 값을 읽고, 레지스터에 올리고, ALU에서 더한 뒤, 결과를 다시 저장하는 여러 명령어로 바뀐다. [[용어 모음집/파이프라이닝|파이프라이닝]]이 있으면 여러 명령어의 단계가 겹쳐 실행되지만, 데이터 의존성이나 분기 때문에 멈출 수 있다.
