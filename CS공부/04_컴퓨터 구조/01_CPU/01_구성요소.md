# CPU 구성요소

## 한 줄 정의
[[용어 모음집/CPU|CPU]] 구성요소는 **[[용어 모음집/명령어|명령어]]를 실행하기 위해 제어, 연산, 저장, 데이터 이동을 담당하는 내부 부품들의 역할 분담**이다.

---

## 핵심 개념
- **[[용어 모음집/제어장치|제어장치]](Control Unit)**: 명령어를 해석하고 제어 신호를 만든다
- **[[용어 모음집/ALU|ALU]]**: 산술/논리 연산을 수행한다
- **[[용어 모음집/레지스터|레지스터]]**: CPU 내부에서 값을 빠르게 저장한다
- **[[용어 모음집/버스|버스]]/데이터 경로**: CPU 내부 부품 사이의 이동 통로다
- **[[용어 모음집/클럭|클럭]]**: 각 동작이 진행되는 기준 박자다
- **[[용어 모음집/캐시|캐시]]**: CPU와 메모리 사이의 속도 차이를 줄인다

---

## CPU 내부 흐름

```text
명령어 가져오기
   ↓
제어장치가 해석
   ↓
레지스터에서 값 읽기
   ↓
ALU가 계산
   ↓
레지스터에 결과 저장
   ↓
필요하면 메모리/캐시에 반영
```

---

## 구성요소별 역할

| 구성요소 | 역할           | 쉽게 말하면  |
| ---- | ------------ | ------- |
| 제어장치 | 어떤 동작을 할지 지시 | 감독      |
| ALU  | 계산과 비교 수행    | 계산기     |
| 레지스터 | 매우 빠른 임시 저장  | 손에 든 메모 |
| 버스   | 데이터 이동 통로    | 길       |
| 클럭   | 동작 타이밍 기준    | 박자      |
| 캐시   | 자주 쓰는 데이터 보관 | 가까운 서랍  |

---

## 데이터 경로 이해
CPU는 값을 그냥 "계산한다"가 아니라 정해진 경로로 옮기며 처리한다.

```text
레지스터 A ─┐
            ├→ ALU → 결과 레지스터
레지스터 B ─┘
```

명령어마다 어떤 레지스터를 읽고, ALU가 무슨 연산을 하고, 결과를 어디에 쓸지가 달라진다.

---

## 제어 신호 예시
- 레지스터 읽기
- 레지스터 쓰기
- ALU 연산 종류 선택
- 메모리 읽기
- 메모리 쓰기
- [[용어 모음집/PC|PC]] 갱신

---

## 자주 하는 실수
- ALU만 CPU의 핵심이라고 생각함
  - 바로잡기: CPU는 ALU, 제어장치, 레지스터, 캐시, 버스, 클럭이 함께 동작해야 명령어를 실행할 수 있다.
- 제어장치가 실제 계산을 한다고 착각함
  - 바로잡기: 제어장치는 계산을 직접 하기보다 명령어를 해석하고 ALU, 레지스터, 메모리 접근을 제어하는 신호를 만든다.
- 레지스터와 메모리를 같은 속도 계층으로 봄
  - 바로잡기: 레지스터는 CPU 내부의 초고속 저장 공간이고, 메모리는 CPU 밖의 큰 주기억장치다. 속도, 용량, 접근 방식이 모두 다르다.
- 버스와 캐시를 CPU 외부 개념으로만 봄
  - 바로잡기: 버스는 데이터가 이동하는 통로이고, 캐시는 자주 쓰는 데이터를 저장하는 계층이다. 둘 다 CPU 성능과 밀접하게 연결된다.
- 클럭이 모든 명령어 실행 시간을 단순히 결정한다고 생각함
  - 바로잡기: 성능은 클럭, IPC, 캐시 적중률, 분기 예측, 메모리 대기 시간, 코어 구조가 함께 결정한다.

---

## 면접 포인트
> CPU는 제어장치, ALU, 레지스터가 중심이 되어 동작합니다. 제어장치는 명령어를 해석해 각 부품에 제어 신호를 보내고, ALU는 계산과 비교를 수행하며, 레지스터는 명령 실행 중 필요한 값을 빠르게 저장합니다. 이 부품들은 내부 버스와 데이터 경로로 연결되고 클럭에 맞춰 동작합니다.

## 체크리스트

- [ ] 제어장치와 ALU 역할을 구분할 수 있다
- [ ] 레지스터가 필요한 이유를 안다
- [ ] 데이터 경로와 제어 신호의 이해을 안다

## 보강 설명
CPU 주제는 “명령어가 실제로 어떻게 실행되는가”를 기준으로 연결해서 보는 것이 좋다. PC가 명령어 주소를 가리키고, 제어장치가 명령어를 해석하며, 레지스터와 ALU가 값을 주고받고, 결과가 다시 레지스터나 메모리에 반영된다.

## 구체적인 예시
예를 들어 `a = b + c` 같은 코드는 메모리에서 값을 읽고, 레지스터에 올리고, ALU에서 더한 뒤, 결과를 다시 저장하는 여러 명령어로 바뀐다. [[용어 모음집/파이프라이닝|파이프라이닝]]이 있으면 여러 명령어의 단계가 겹쳐 실행되지만, 데이터 의존성이나 분기 때문에 멈출 수 있다.
