# 레지스터

## 한 줄 정의
[[용어 모음집/레지스터|레지스터]]는 **[[용어 모음집/CPU|CPU]] 내부에서 [[용어 모음집/명령어|명령어]] 실행 중 필요한 값과 상태를 가장 빠르게 저장하는 작은 저장 공간**이다.

---

## 왜 필요한가
CPU가 매번 [[용어 모음집/RAM|RAM]]에서 값을 읽으면 너무 느리다. 그래서 계산에 바로 필요한 값은 CPU 안의 레지스터에 둔다.

```text
메모리 → 레지스터 → ALU 계산 → 레지스터 → 메모리
```

---

## 핵심 개념
- 레지스터는 CPU 내부에 있어 매우 빠르다
- 크기와 개수가 제한되어 있다
- 명령어는 주로 레지스터를 대상으로 연산한다
- [[용어 모음집/문맥 교환|문맥 교환]] 때 레지스터 상태를 저장/복원해야 한다

---

## 대표 레지스터

| 레지스터              | 의미                   | 역할               |
| ----------------- | -------------------- | ---------------- |
| [[용어 모음집/PC\|PC]] | Program Counter      | 다음에 실행할 명령어 주소   |
| [[용어 모음집/IR\|IR]] | Instruction Register | 현재 가져온 명령어       |
| SP                | Stack Pointer        | 현재 스택의 위치        |
| FP/BP             | Frame/Base Pointer   | 함수 호출 프레임 기준     |
| FLAGS/PSW         | 상태 레지스터              | 연산 결과 상태, 조건 플래그 |
| GPR               | 범용 레지스터              | 일반 계산과 데이터 보관    |

---

## PC와 IR 흐름

```text
PC: 다음 명령어 주소
   ↓
명령어 fetch
   ↓
IR: 현재 명령어 저장
   ↓
decode/execute
   ↓
PC 갱신
```

---

## 상태 플래그
[[용어 모음집/ALU|ALU]] 연산 결과는 상태 플래그에 영향을 줄 수 있다.

- Zero Flag: 결과가 0인지
- Sign Flag: 결과가 음수인지
- Carry Flag: 자리올림/빌림이 있었는지
- Overflow Flag: signed 연산 범위를 넘었는지

조건 분기 명령어는 이런 플래그를 보고 다음 흐름을 결정할 수 있다.

---

## 레지스터와 문맥 교환
[[용어 모음집/운영체제|운영체제]]가 실행 중인 [[용어 모음집/프로세스|프로세스]]를 바꿀 때는 CPU 상태를 저장해야 한다.

```text
프로세스 A 레지스터 상태 저장
   ↓
프로세스 B 레지스터 상태 복원
   ↓
프로세스 B 실행
```

그래서 [[용어 모음집/PCB|PCB]]에는 PC, SP, 범용 레지스터, 상태 레지스터 같은 정보가 저장될 수 있다.

---

## 자주 하는 실수
- 레지스터를 [[용어 모음집/캐시|캐시]]와 같은 것으로 생각함
  - 바로잡기: 레지스터는 ALU가 바로 사용할 값과 CPU 실행 상태를 담는 아주 작은 저장소이고, 캐시는 RAM 접근을 줄이기 위해 자주 쓰는 데이터/명령어를 저장하는 계층이다.
- PC가 현재 명령어 자체를 저장한다고 생각함
  - 바로잡기: PC는 보통 다음에 가져올 명령어 주소를 가리키고, 현재 가져온 명령어 자체는 IR에 저장된다.
- SP와 PC 역할을 헷갈림
  - 바로잡기: SP는 현재 스택 위치를 가리키고, PC는 다음 명령어 주소를 가리킨다. 하나는 데이터 스택 위치, 하나는 실행 흐름 위치다.
- 플래그가 단순한 부가 정보라고만 생각함
  - 바로잡기: 플래그는 ALU 연산 결과의 상태를 담고, 조건 분기나 오버플로우 판단에 직접 사용될 수 있다.
- 문맥 교환이 메모리만 저장하는 일이라고 생각함
  - 바로잡기: 문맥 교환은 실행 대상을 바꾸는 것뿐 아니라 PC, SP, 범용 레지스터, 상태 레지스터 같은 CPU 상태를 저장하고 복원하는 작업이다.

---

## 면접 포인트
> 레지스터는 CPU 내부의 매우 빠른 작은 저장 공간으로, 명령어 실행 중 필요한 값과 상태를 저장합니다. PC는 다음 명령어 주소를, IR은 현재 명령어를, SP는 스택 위치를 가리킵니다. ALU 연산 결과는 상태 플래그에 반영될 수 있고, 운영체제의 문맥 교환에서는 이런 레지스터 상태를 저장하고 복원해야 합니다.

---

## 체크리스트
- [ ] 레지스터가 RAM보다 빠른 이유를 안다
- [ ] PC, IR, SP 역할을 구분할 수 있다
- [ ] FLAGS/PSW가 조건 분기와 연결됨을 안다
- [ ] 문맥 교환과 레지스터 저장/복원을 연결할 수 있다
---

## 보강 설명
CPU 주제는 “명령어가 실제로 어떻게 실행되는가”를 기준으로 연결해서 보는 것이 좋다. PC가 명령어 주소를 가리키고, [[용어 모음집/제어장치|제어장치]]가 명령어를 해석하며, 레지스터와 ALU가 값을 주고받고, 결과가 다시 레지스터나 메모리에 반영된다.

## 구체적인 예시
예를 들어 `a = b + c` 같은 코드는 메모리에서 값을 읽고, 레지스터에 올리고, ALU에서 더한 뒤, 결과를 다시 저장하는 여러 명령어로 바뀐다. [[용어 모음집/파이프라이닝|파이프라이닝]]이 있으면 여러 명령어의 단계가 겹쳐 실행되지만, 데이터 의존성이나 분기 때문에 멈출 수 있다.
