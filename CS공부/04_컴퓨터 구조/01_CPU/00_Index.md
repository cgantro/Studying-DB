# CPU

## 한 줄 정의
CPU는 **명령어를 가져오고 해석하고 실행하며, 컴퓨터 전체의 계산 흐름을 지휘하는 중앙 처리 장치**다.

## 핵심 흐름

```text
명령어 주소 확인
   ↓
명령어 가져오기
   ↓
명령어 해석
   ↓
레지스터/메모리에서 데이터 준비
   ↓
ALU 또는 실행 유닛에서 처리
   ↓
결과 저장
   ↓
다음 명령어로 이동
```

---

## CPU를 볼 때의 큰 그림
- **제어장치**: 어떤 일을 어떤 순서로 할지 지휘한다
- **ALU**: 산술/논리 연산을 수행한다
- **레지스터**: 명령 실행 중 필요한 값을 아주 빠르게 보관한다
- **클럭**: CPU 내부 동작의 박자를 만든다
- **[[용어 모음집/버스|버스]]/데이터 경로**: 내부 부품 사이에서 값이 이동하는 길이다
- **파이프라인**: 여러 명령어 단계를 겹쳐 [[용어 모음집/처리량|처리량]]을 높인다
- **[[용어 모음집/캐시|캐시]]**: 느린 메모리 접근을 줄여 CPU가 덜 기다리게 한다

---

## 면접 포인트
- CPU는 단순히 "계산하는 장치"가 아니라 **명령어 실행 흐름 전체를 처리하는 장치**라고 말한다
- 제어장치, ALU, 레지스터의 역할을 분리해서 설명한다
- 명령어 사이클을 `fetch → decode → execute → write-back`으로 말할 수 있어야 한다
- 성능은 클럭만이 아니라 IPC, [[용어 모음집/캐시 미스|캐시 미스]], [[용어 모음집/분기 예측|분기 예측]], 파이프라인 효율과 함께 설명한다

---

## 자주 헷갈리는 연결

| 개념 | 연결되는 질문 |
|---|---|
| 레지스터 | CPU가 왜 메모리보다 레지스터를 먼저 쓰나? |
| ALU | 계산 결과와 플래그는 어디서 만들어지나? |
| 제어장치 | 명령어마다 다른 동작을 어떻게 지시하나? |
| 명령어 사이클 | 프로그램은 CPU 안에서 어떤 순서로 실행되나? |
| 파이프라이닝 | 클럭이 같아도 처리량이 왜 달라지나? |
| IPC | 같은 클럭인데 CPU 성능이 왜 다를 수 있나? |

---

## 초보자용 기억법
CPU는 **"명령어를 읽고, 해석하고, 계산하고, 결과를 저장하는 아주 빠른 작업 관리자"**다.
---

## 보강 설명
CPU 주제는 “명령어가 실제로 어떻게 실행되는가”를 기준으로 연결해서 보는 것이 좋다. PC가 명령어 주소를 가리키고, 제어장치가 명령어를 해석하며, 레지스터와 ALU가 값을 주고받고, 결과가 다시 레지스터나 메모리에 반영된다.

## 구체적인 예시
예를 들어 `a = b + c` 같은 코드는 메모리에서 값을 읽고, 레지스터에 올리고, ALU에서 더한 뒤, 결과를 다시 저장하는 여러 명령어로 바뀐다. 파이프라이닝이 있으면 여러 명령어의 단계가 겹쳐 실행되지만, 데이터 의존성이나 분기 때문에 멈출 수 있다.
