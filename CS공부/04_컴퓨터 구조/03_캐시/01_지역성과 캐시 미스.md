# 지역성과 캐시 미스

## 한 줄 정의
[[용어 모음집/지역성|지역성]](locality)은 **프로그램이 가까운 시간이나 가까운 주소의 데이터를 반복해서 쓰는 경향**이고, [[용어 모음집/캐시 미스|캐시 미스]]는 **필요한 데이터가 캐시에 없어 더 느린 메모리에서 가져와야 하는 상황**이다.

## 지역성의 종류
### 시간 지역성
최근에 사용한 데이터나 [[용어 모음집/명령어|명령어]]를 곧 다시 사용할 가능성이 높다는 성질이다.

예시:
- 반복문에서 같은 변수 `sum`을 계속 갱신
- 함수 안의 같은 명령어를 반복 실행

### 공간 지역성
어떤 주소를 사용하면 그 주변 주소도 곧 사용할 가능성이 높다는 성질이다.

예시:
- 배열을 `arr[0]`, `arr[1]`, `arr[2]` 순서로 읽음
- 연속된 명령어를 차례대로 실행

## 캐시 미스 종류
| 종류            | 의미                | 예시                |
| ------------- | ----------------- | ----------------- |
| Cold miss     | 처음 접근해서 캐시에 없음    | 프로그램 시작 후 배열 첫 접근 |
| Capacity miss | 캐시 용량이 부족해서 밀려남   | 큰 배열을 한 번에 훑음     |
| Conflict miss | 같은 캐시 위치에 매핑되어 충돌 | 특정 간격의 주소만 반복 접근  |

## 예시
```c
for (int i = 0; i < n; i++) {
    sum += arr[i];
}
```

배열을 앞에서부터 순서대로 읽으면 공간 지역성이 좋다. 한 캐시 라인에 여러 배열 원소가 함께 올라오기 때문에 다음 원소 접근이 빨라질 수 있다.

반대로 2차원 배열을 메모리 배치와 반대 방향으로 훑으면 캐시 라인을 제대로 활용하지 못해 미스가 늘어난다.

## 캐시 미스가 비싼 이유
[[용어 모음집/CPU|CPU]]는 매우 빠르지만 메모리는 상대적으로 느리다. 캐시 미스가 나면 CPU가 필요한 데이터를 기다리게 되고, 이 시간이 성능 병목이 될 수 있다.

## 자주 헷갈리는 점
- 캐시는 프로그래머가 직접 넣고 빼는 저장소가 아니라 하드웨어가 자동으로 관리하는 경우가 많다.
- 캐시가 커지면 capacity miss는 줄 수 있지만 접근 지연이나 비용이 늘 수 있다.
- 지역성이 좋은 코드가 보통 캐시 친화적인 코드다.

## 면접 포인트
지역성은 캐시가 효과를 내는 근거다. 시간 지역성은 최근 쓴 것을 다시 쓸 가능성, 공간 지역성은 주변 주소를 쓸 가능성이다. 캐시 미스는 cold, capacity, conflict miss로 나눠 설명할 수 있다.

## 보강 설명
캐시 주제는 “빠른 CPU와 느린 메모리 사이의 간격을 어떻게 줄이는가”가 핵심이다. 지역성이 좋은 접근은 캐시 적중률을 높이고, 캐시 미스나 일관성 비용은 성능을 떨어뜨린다.

## 구체적인 예시
예를 들어 배열을 앞에서부터 순서대로 읽으면 한 캐시 라인에 들어온 주변 원소를 이어서 쓰기 때문에 빠르다. 반대로 큰 배열을 무작위로 접근하거나 여러 코어가 같은 캐시 라인을 계속 수정하면 미스와 일관성 트래픽이 늘어난다.
