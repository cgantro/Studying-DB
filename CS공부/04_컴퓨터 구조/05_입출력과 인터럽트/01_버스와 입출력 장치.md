# 버스와 입출력 장치

## 한 줄 정의
[[용어 모음집/버스|버스]]는 **[[용어 모음집/CPU|CPU]], 메모리, [[용어 모음집/입출력 장치|입출력 장치]]가 데이터와 신호를 주고받는 공용 통로**이고, 입출력 장치는 **컴퓨터 바깥 세계와 데이터를 주고받는 하드웨어**다.

---

## 핵심
- CPU는 장치를 직접 세부 제어하지 않고 보통 컨트롤러를 통해 접근한다
- 버스는 데이터, 주소, 제어 신호를 전달한다
- 장치는 CPU보다 훨씬 느린 경우가 많아 대기와 [[용어 모음집/동기화|동기화]]가 필요하다
- 장치마다 제어 방식이 달라 드라이버가 필요하다

---

## 버스 종류 이해

| 구분     | 역할                                      |
| ------ | --------------------------------------- |
| 데이터 버스 | 실제 데이터 이동                               |
| 주소 버스  | 어디에 접근할지 지정                             |
| 제어 버스  | 읽기/쓰기/[[용어 모음집/인터럽트\|인터럽트]] 같은 제어 신호 전달 |

---

## 입출력 장치 흐름

```text
CPU
 ↓ 명령
장치 컨트롤러
 ↓ 제어
입출력 장치
 ↓ 결과
장치 컨트롤러
 ↓ 상태/데이터
CPU 또는 메모리
```

---

## 컨트롤러가 필요한 이유
- 장치별 세부 동작을 숨긴다
- CPU가 읽을 수 있는 상태 [[용어 모음집/레지스터|레지스터]]를 제공한다
- 데이터 버퍼를 제공할 수 있다
- 인터럽트를 발생시켜 CPU에게 완료를 알릴 수 있다

---

## 성능 판단 기준
입출력 장치는 CPU와 속도 차이가 크다.

```text
CPU 연산: 매우 빠름
메모리 접근: 빠름
디스크/네트워크/사용자 입력: 상대적으로 느림
```

그래서 I/O에서는 대기, 버퍼링, 인터럽트, [[용어 모음집/DMA|DMA]]가 중요해진다.

---

## 면접 포인트
> 버스는 CPU, 메모리, 장치를 연결하는 통로이고, 입출력 장치는 보통 컨트롤러를 통해 CPU와 통신합니다. 컨트롤러는 장치 상태와 데이터 버퍼, 제어 레지스터를 제공하며, 장치가 느리기 때문에 CPU가 효율적으로 기다리고 처리하도록 인터럽트나 DMA 같은 기법이 함께 사용됩니다.

---

## 체크리스트
- [ ] 버스가 무엇을 연결하는지 안다
- [ ] 데이터/주소/제어 버스 역할을 구분할 수 있다
- [ ] 장치 컨트롤러가 필요한 이유를 설명할 수 있다
- [ ] CPU와 장치 속도 차이가 I/O 설계에 미치는 영향을 안다
---

## 보강 설명
입출력 주제는 CPU가 느린 장치와 어떻게 협력하는지 보는 영역이다. CPU가 계속 기다릴지, 장치가 인터럽트로 알릴지, DMA로 데이터 이동을 맡길지에 따라 효율이 달라진다.

## 구체적인 예시
예를 들어 디스크에서 큰 파일을 읽을 때 CPU가 바이트마다 직접 옮기면 비효율적이다. DMA를 사용하면 장치가 메모리로 직접 데이터를 옮기고, 완료되면 인터럽트로 CPU에게 알린다.
