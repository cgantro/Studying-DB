# DMA와 메모리 매핑 입출력

## 한 줄 정의
[[용어 모음집/DMA|DMA]]는 **장치가 [[용어 모음집/CPU|CPU]] 개입을 줄이고 메모리와 직접 데이터를 주고받게 하는 방식**이고, [[용어 모음집/메모리 매핑 입출력|메모리 매핑 입출력]]은 **장치 제어 [[용어 모음집/레지스터|레지스터]]를 메모리 주소처럼 접근하는 방식**이다.

---

## 핵심
- DMA는 대량 데이터 이동에서 CPU 부담을 줄인다
- CPU는 DMA 전송을 설정하고 완료 [[용어 모음집/인터럽트|인터럽트]]를 받는다
- memory-mapped I/O는 장치 레지스터를 주소 공간에 배치한다
- 드라이버는 이 주소에 읽기/쓰기를 하며 장치를 제어한다

---

## DMA가 없을 때

```text
장치 데이터 1개
   ↓
CPU가 읽음
   ↓
메모리에 씀
   ↓
반복
```

큰 파일이나 네트워크 패킷을 옮길 때 CPU가 지나치게 바빠진다.

---

## DMA가 있을 때

```text
CPU: 전송 위치와 크기 설정
   ↓
DMA 컨트롤러: 장치 ↔ 메모리 전송
   ↓
완료 인터럽트
   ↓
CPU: 후속 처리
```

CPU는 실제 복사를 계속 맡지 않아도 된다.

---

## DMA의 장점과 주의

| 구분  | 내용                                         |
| --- | ------------------------------------------ |
| 장점  | CPU 사용량 감소, 대량 전송 효율 향상                    |
| 주의  | [[용어 모음집/캐시 일관성\|캐시 일관성]], 권한/보호, 버퍼 위치 관리 |


---

## 메모리 매핑 입출력
장치의 제어 레지스터를 특정 메모리 주소에 연결한다.

```text
특정 주소에 쓰기 → 장치 설정 변경
특정 주소 읽기 → 장치 상태 확인
```

CPU 입장에서는 메모리를 읽고 쓰는 것처럼 보이지만, 실제로는 장치를 제어한다.

---

## Port-mapped I/O와 비교

| 방식 | 쉽게 이해하기 |
|---|---|
| memory-mapped I/O | 장치 레지스터를 메모리 주소처럼 접근 |
| port-mapped I/O | 별도 I/O 명령과 포트 공간 사용 |

---

## 운영체제와 연결
- 드라이버는 DMA 버퍼를 준비한다
- [[용어 모음집/커널|커널]]은 장치가 접근해도 되는 메모리를 관리한다
- IOMMU 같은 장치는 DMA 접근 범위를 제한해 보호를 강화한다
- 완료 인터럽트가 오면 대기 중인 작업을 깨울 수 있다

---

## 면접 포인트
> DMA는 CPU가 모든 데이터를 직접 옮기지 않고 장치와 메모리 사이의 대량 전송을 DMA 컨트롤러가 수행하게 하는 방식입니다. CPU는 전송을 설정하고 완료 인터럽트를 처리합니다. 메모리 매핑 입출력은 장치 레지스터를 메모리 주소처럼 접근해 드라이버가 장치를 제어하는 방식입니다.

---

## 체크리스트
- [ ] DMA가 CPU 부담을 줄이는 이유를 설명할 수 있다
- [ ] DMA 완료 인터럽트 흐름을 안다
- [ ] memory-mapped I/O가 무엇인지 안다
- [ ] DMA와 캐시 일관성/보호 문제가 연결됨을 안다
---

## 보강 설명
입출력 주제는 CPU가 느린 장치와 어떻게 협력하는지 보는 영역이다. CPU가 계속 기다릴지, 장치가 인터럽트로 알릴지, DMA로 데이터 이동을 맡길지에 따라 효율이 달라진다.

## 구체적인 예시
예를 들어 디스크에서 큰 파일을 읽을 때 CPU가 바이트마다 직접 옮기면 비효율적이다. DMA를 사용하면 장치가 메모리로 직접 데이터를 옮기고, 완료되면 인터럽트로 CPU에게 알린다.
