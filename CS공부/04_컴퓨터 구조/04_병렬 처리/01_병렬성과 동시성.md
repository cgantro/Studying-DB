# 병렬성과 동시성

## 한 줄 정의
[[용어 모음집/동시성|동시성]](concurrency)은 **여러 작업을 번갈아 진행해 동시에 처리되는 것처럼 구성하는 성질**이고, [[용어 모음집/병렬성|병렬성]](parallelism)은 **여러 작업이 실제로 같은 시간에 실행되는 성질**이다.

## 핵심 차이
| 구분 | 동시성 | 병렬성 |
|---|---|---|
| 핵심 | 작업을 잘 쪼개고 전환함 | 작업을 실제 동시에 실행함 |
| 코어 수 | 한 코어에서도 가능 | 보통 여러 코어 필요 |
| 목적 | 응답성, 구조화, [[용어 모음집/대기 시간|대기 시간]] 활용 | 처리 속도, [[용어 모음집/처리량|처리량]] 향상 |
| 예시 | 싱글 코어 이벤트 루프 | 멀티코어 병렬 계산 |

## 예시
### 동시성 예시
웹 서버가 요청 A의 DB 응답을 기다리는 동안 요청 B를 처리한다. 실제 [[용어 모음집/CPU|CPU]]는 한 번에 하나의 코드를 실행할 수 있어도, 대기 시간을 활용해 여러 작업이 동시에 진행되는 것처럼 보인다.

### 병렬성 예시
이미지 파일을 4등분해 4개 코어가 각각 필터를 적용한다. 이 경우 여러 계산이 물리적으로 동시에 실행된다.

## 둘은 함께 쓰일 수 있다
실제 서버는 동시성과 병렬성을 같이 쓴다. 여러 요청을 동시적으로 관리하면서, 여러 CPU 코어에서 병렬로 실행한다.

```text
요청 관리: 동시성
여러 코어 실행: 병렬성
```

## 자주 헷갈리는 점
- 멀티[[용어 모음집/스레드|스레드]]라고 항상 병렬인 것은 아니다. 싱글 코어에서는 스레드가 번갈아 실행된다.
- 비동기 I/O는 동시성을 높이는 기술이지, CPU 계산을 자동으로 병렬화하는 기술은 아니다.
- 병렬성은 속도를 높일 수 있지만 [[용어 모음집/동기화|동기화]] 비용 때문에 항상 빨라지지는 않는다.

## 면접 포인트
동시성은 여러 작업을 다루는 구조의 문제이고, 병렬성은 여러 작업을 실제 동시에 실행하는 물리적 실행의 문제다. 한 코어에서도 동시성은 가능하지만 병렬성은 보통 여러 실행 자원이 필요하다.

## 보강 설명
병렬 처리 주제는 “작업을 나눴을 때 실제로 빨라지는가”를 따져야 한다. 나눌 수 없는 직렬 구간, 동기화 비용, 결과 합치기 비용이 전체 성능의 한계를 만든다.

## 구체적인 예시
예를 들어 이미지 필터는 영역별로 나누면 여러 코어가 동시에 처리할 수 있다. 하지만 마지막에 결과를 합치는 단계가 길거나 공유 락을 자주 잡으면 기대만큼 빨라지지 않는다.
