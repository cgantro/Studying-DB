# SIMD와 MIMD

## 한 줄 정의
[[용어 모음집/SIMD|SIMD]]는 **하나의 [[용어 모음집/명령어|명령어]]로 여러 데이터를 동시에 처리하는 방식**이고, [[용어 모음집/MIMD|MIMD]]는 **여러 처리 장치가 서로 다른 명령어를 서로 다른 데이터에 대해 실행하는 방식**이다.

## Flynn 분류 이해
| 분류 | 의미 | 예시 |
|---|---|---|
| SISD | 하나의 명령, 하나의 데이터 | 전통적인 단일 코어 실행 |
| SIMD | 하나의 명령, 여러 데이터 | 벡터 연산, 이미지 픽셀 처리 |
| MISD | 여러 명령, 하나의 데이터 | 드문 구조 |
| MIMD | 여러 명령, 여러 데이터 | 멀티코어 [[용어 모음집/CPU|CPU]], 분산 처리 |

## SIMD 예시
```text
데이터: [1, 2, 3, 4]
연산: 각 원소에 10 더하기
결과: [11, 12, 13, 14]
```

같은 연산을 여러 데이터에 반복 적용하므로 SIMD에 잘 맞는다. 이미지 밝기 조정, 행렬 연산, 벡터 계산, 딥러닝 연산에서 자주 활용된다.

## MIMD 예시
멀티코어 서버에서 한 코어는 네트워크 요청을 처리하고, 다른 코어는 로그 압축을 수행하며, 또 다른 코어는 DB 결과를 가공할 수 있다. 각 코어가 서로 다른 명령 흐름을 실행하므로 MIMD에 가깝다.

## SIMD가 잘 맞는 작업
- 배열 전체에 같은 연산 적용
- 이미지/영상 픽셀 처리
- 행렬 곱셈
- 벡터 내적
- 암호화나 압축의 일부 연산

## MIMD가 잘 맞는 작업
- 여러 사용자 요청 처리
- 서로 다른 서비스 작업 병렬 실행
- [[용어 모음집/프로세스|프로세스]]/[[용어 모음집/스레드|스레드]] 기반 병렬 처리
- 분산 시스템의 작업 분담

## 자주 헷갈리는 점
- SIMD는 스레드를 많이 만드는 것과 다르다. 하나의 명령이 여러 데이터 lane에 적용되는 방식이다.
- GPU는 SIMD와 비슷한 대량 데이터 병렬 처리에 강하지만, 내부 모델은 제품과 아키텍처마다 차이가 있다.
- 일반적인 멀티코어 병렬 프로그램은 대부분 MIMD 관점으로 설명할 수 있다.

## 면접 포인트
SIMD는 같은 명령을 여러 데이터에 동시에 적용하는 데이터 병렬 방식이고, MIMD는 여러 실행 단위가 각자 다른 명령 흐름을 처리하는 방식이다. 배열, 픽셀, 행렬처럼 같은 연산이 반복되는 작업은 SIMD에 잘 맞는다.

## 보강 설명
병렬 처리 주제는 “작업을 나눴을 때 실제로 빨라지는가”를 따져야 한다. 나눌 수 없는 직렬 구간, [[용어 모음집/동기화|동기화]] 비용, 결과 합치기 비용이 전체 성능의 한계를 만든다.

## 구체적인 예시
예를 들어 이미지 필터는 영역별로 나누면 여러 코어가 동시에 처리할 수 있다. 하지만 마지막에 결과를 합치는 단계가 길거나 공유 락을 자주 잡으면 기대만큼 빨라지지 않는다.

## 헷갈릴 때 구분 기준
- 이름이 비슷한 용어는 “무엇을 관리하는가”, “언제 발생하는가”, “어떤 비용을 줄이는가”로 구분한다.
- 성능 관련 개념은 항상 병목과 오버헤드를 같이 본다.
- [[용어 모음집/운영체제|운영체제]] 개념은 사용자 프로그램 관점과 [[용어 모음집/커널|커널]] 내부 관점을 나누어 설명한다.
