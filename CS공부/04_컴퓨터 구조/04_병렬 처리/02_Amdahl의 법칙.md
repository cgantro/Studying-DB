# Amdahl의 법칙

## 한 줄 정의
[[용어 모음집/Amdahl의 법칙|Amdahl의 법칙]]은 **프로그램에서 병렬화할 수 없는 직렬 부분이 전체 성능 향상의 최대 한계를 결정한다**는 법칙이다.

## 핵심 공식
```text
Speedup = 1 / ((1 - P) + P / N)
```

- `P`: 병렬화 가능한 비율
- `N`: 병렬 처리에 사용하는 코어 또는 처리 장치 수
- `1 - P`: 반드시 직렬로 실행해야 하는 비율

## 예시
전체 작업의 80%를 병렬화할 수 있고, 4개 코어를 쓴다고 하자.

```text
Speedup = 1 / (0.2 + 0.8 / 4)
        = 1 / (0.2 + 0.2)
        = 2.5
```

코어를 4개 쓴다고 4배 빨라지는 것이 아니라 최대 2.5배 정도가 된다. 남은 20%의 직렬 부분이 전체 성능을 제한하기 때문이다.

## 직관
```text
전체 작업
├─ 직렬 부분: 한 명만 처리 가능
└─ 병렬 부분: 여러 명이 나눠 처리 가능

직렬 부분이 길면 전체 시간은 계속 남는다.
```

## 실무에서 보는 지점
- 작업 분할 비용
- 결과 합치기 비용
- 락 [[용어 모음집/대기 시간|대기 시간]]
- I/O 대기
- 순서가 반드시 필요한 로직

이런 부분은 병렬화해도 남거나 오히려 늘어날 수 있다.

## 자주 헷갈리는 점
- 코어 수를 늘리면 무조건 선형으로 빨라진다고 생각하면 안 된다.
- 병렬화 가능한 비율이 높아도 [[용어 모음집/동기화|동기화]]와 통신 비용이 크면 실제 속도 향상은 더 작다.
- Amdahl의 법칙은 "병렬화를 하지 말라"는 말이 아니라 병목을 먼저 찾아야 한다는 말에 가깝다.

## 면접 포인트
Amdahl의 법칙은 병렬 처리의 한계를 설명할 때 사용한다. 병렬화할 수 없는 직렬 구간이 남아 있으면 코어를 계속 늘려도 속도 향상은 일정 수준에서 멈춘다.

## 보강 설명
병렬 처리 주제는 “작업을 나눴을 때 실제로 빨라지는가”를 따져야 한다. 나눌 수 없는 직렬 구간, 동기화 비용, 결과 합치기 비용이 전체 성능의 한계를 만든다.

## 구체적인 예시
예를 들어 이미지 필터는 영역별로 나누면 여러 코어가 동시에 처리할 수 있다. 하지만 마지막에 결과를 합치는 단계가 길거나 공유 락을 자주 잡으면 기대만큼 빨라지지 않는다.
