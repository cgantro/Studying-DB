# ISA와 어셈블리

## 한 줄 정의
[[용어 모음집/ISA|ISA]](Instruction Set Architecture)는 **[[용어 모음집/CPU|CPU]]가 이해할 수 있는 [[용어 모음집/명령어|명령어]]의 종류, [[용어 모음집/레지스터|레지스터]], 데이터 형식, 메모리 접근 방식 등을 정한 약속**이고, [[용어 모음집/어셈블리|어셈블리]]는 그 ISA 명령어를 사람이 읽기 쉽게 표현한 언어다.

## 핵심 개념
- **ISA**: 하드웨어와 소프트웨어 사이의 계약
- **기계어(machine code)**: CPU가 직접 해석하는 0과 1의 명령
- **어셈블리(assembly)**: 기계어와 거의 1:1로 대응되는 저수준 언어
- **어셈블러(assembler)**: 어셈블리 코드를 기계어로 바꾸는 프로그램
- **[[용어 모음집/RISC|RISC]]/[[용어 모음집/CISC|CISC]]**: ISA를 설계하는 대표적인 두 방향

## ISA가 정하는 것
- 어떤 명령어가 있는가: `add`, `load`, `store`, `jump` 같은 연산
- 어떤 레지스터를 쓰는가: 범용 레지스터, [[용어 모음집/PC|PC]], SP 등
- 명령어 형식은 어떤가: opcode, register 번호, immediate 값의 위치
- 메모리에 어떻게 접근하는가: 주소 지정 방식, 정렬 규칙
- 함수 호출 규약은 어떻게 맞추는가: 인자 전달, 반환값, 스택 사용 방식

## 예시
```asm
add r1, r2, r3
```

위 명령은 보통 `r1 = r2 + r3`이라는 의미다. 어셈블리에서는 사람이 `add`라고 쓰지만, 실제 CPU는 이 명령을 특정 비트 패턴의 기계어로 읽는다.

## RISC와 CISC
| 구분 | RISC | CISC |
|---|---|---|
| 명령어 | 단순하고 길이가 비교적 일정 | 복잡하고 다양한 명령어 |
| 설계 의도 | 단순한 명령을 빠르게 실행 | 한 명령으로 많은 일을 처리 |
| 예시 | ARM, RISC-V | x86 계열 |
| 장점 | [[용어 모음집/파이프라이닝|파이프라이닝]]에 유리 | 코드 밀도가 좋을 수 있음 |

## 자주 헷갈리는 점
- 어셈블리는 CPU마다 다르다. x86 어셈블리와 ARM 어셈블리는 명령어와 레지스터 이름이 다르다.
- C 코드가 같아도 컴파일 대상 ISA가 다르면 생성되는 기계어는 달라진다.
- ISA는 구현 그 자체가 아니다. 같은 ISA를 따르는 CPU라도 내부 파이프라인, [[용어 모음집/캐시|캐시]], 실행 유닛 설계는 다를 수 있다.

## 면접 포인트
ISA는 "CPU가 어떤 명령어를 어떻게 실행할지 정한 인터페이스"라고 설명하면 좋다. 어셈블리는 그 ISA를 사람이 읽을 수 있게 표현한 것이고, 기계어는 CPU가 실제로 실행하는 비트열이다.

## 보강 설명
명령어 주제는 소프트웨어와 하드웨어의 경계다. 고급 언어 코드는 컴파일을 거쳐 ISA가 정의한 명령어와 레지스터 사용 규칙에 맞춰 실행된다.

## 구체적인 예시
예를 들어 `x = arr[i]`는 배열 시작 주소와 인덱스를 이용해 실제 메모리 주소를 계산하고, load 명령으로 값을 레지스터에 가져오는 흐름으로 바뀔 수 있다.
