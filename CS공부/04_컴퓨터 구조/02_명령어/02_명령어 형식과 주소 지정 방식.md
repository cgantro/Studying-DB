# 명령어 형식과 주소 지정 방식

## 한 줄 정의
[[용어 모음집/명령어|명령어]] 형식은 **명령어 비트열을 opcode, [[용어 모음집/레지스터|레지스터]], 주소, 즉시값 같은 필드로 나누는 방식**이고, 주소 지정 방식은 **피연산자를 어디에서 어떻게 찾을지 정하는 규칙**이다.

## 명령어 형식
명령어는 보통 다음 정보로 구성된다.

```text
[ opcode ][ operand1 ][ operand2 ][ operand3 또는 immediate ]
```

- **opcode**: 어떤 연산을 할지 나타냄
- **operand**: 연산 대상
- **register field**: 사용할 레지스터 번호
- **immediate**: 명령어 안에 직접 들어 있는 상수 값
- **address field**: 메모리 주소 또는 주소 계산에 필요한 값

## 예시
```asm
addi r1, r2, 10
```

의미는 `r1 = r2 + 10`이다. 여기서 `addi`는 opcode, `r1`과 `r2`는 레지스터 피연산자, `10`은 immediate 값이다.

## 주소 지정 방식
| 방식 | 의미 | 예시로 이해하기 |
|---|---|---|
| 즉시 주소 지정 | 명령어 안의 값 자체를 사용 | `r1 = r2 + 10` |
| 레지스터 주소 지정 | 레지스터에 있는 값을 사용 | `r1 = r2 + r3` |
| 직접 주소 지정 | 명령어가 메모리 주소를 직접 가리킴 | `load r1, [1000]` |
| 간접 주소 지정 | 레지스터/메모리에 들어 있는 주소를 따라감 | `load r1, [r2]` |
| 베이스+오프셋 | 기준 주소에 offset을 더해 접근 | `load r1, 8(sp)` |

## 스택 접근 예시
```asm
load r1, 8(sp)
```

이 명령은 스택 포인터 `sp`가 가리키는 주소에서 8바이트 떨어진 위치의 값을 `r1`로 읽는다는 의미다. 함수 호출 시 지역 변수나 저장된 레지스터를 꺼낼 때 이런 방식이 자주 쓰인다.

## 왜 중요한가
- 컴파일러가 변수와 배열 접근을 어떻게 기계어로 바꾸는지 이해할 수 있다.
- 포인터, 배열, 구조체 접근 비용을 더 잘 이해할 수 있다.
- [[용어 모음집/CPU|CPU]] 파이프라인과 [[용어 모음집/캐시|캐시]] 동작을 설명할 때 메모리 접근 명령을 구분할 수 있다.

## 자주 헷갈리는 점
- 주소 지정 방식은 "값"을 정하는 게 아니라 "값을 어디서 찾을지" 정하는 방식이다.
- immediate 값은 메모리에 접근하지 않아 빠르지만 표현 가능한 범위가 명령어 비트 수에 제한된다.
- 메모리 접근 명령은 보통 레지스터 연산보다 비용이 크다.

## 면접 포인트
명령어는 opcode와 operand로 구성되고, 주소 지정 방식은 피연산자를 찾는 방법이다. 예를 들어 immediate는 명령어 안의 값을 바로 쓰고, register indirect는 레지스터 안의 주소를 따라가 메모리에 접근한다.

## 보강 설명
명령어 주제는 소프트웨어와 하드웨어의 경계다. 고급 언어 코드는 컴파일을 거쳐 ISA가 정의한 명령어와 레지스터 사용 규칙에 맞춰 실행된다.

## 구체적인 예시
예를 들어 `x = arr[i]`는 배열 시작 주소와 인덱스를 이용해 실제 메모리 주소를 계산하고, load 명령으로 값을 레지스터에 가져오는 흐름으로 바뀔 수 있다.
