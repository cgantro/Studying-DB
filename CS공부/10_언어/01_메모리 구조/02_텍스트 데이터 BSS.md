# 텍스트 데이터 BSS

## 메모리 세그먼트
- Text(Code): 실행 명령어 저장, 일반적으로 읽기 전용
- Data: 초기화된 전역/정적 변수
- BSS: 초기화되지 않은 전역/정적 변수(실행 시 0으로 초기화)

## 왜 분리하나
- 보안: 코드 영역 쓰기 방지로 공격면 감소
- 효율: BSS는 파일에 실제 0 데이터를 저장하지 않아 바이너리 크기 절감
- 성능: 로더가 세그먼트 특성에 맞춰 매핑 최적화 가능

## 실무 이슈
- 대형 전역 배열을 BSS에 두면 실행 시 메모리 급증
- static mutable 상태는 테스트/동시성 버그 원인
- 컨테이너 메모리 제한 환경에서 전역 메모리 사용량 관리 필요

## 체크리스트
- 전역/정적 변수 최소화
- 변경 가능한 전역 상태는 캡슐화
- 큰 버퍼는 필요 시점 동적 할당 + 수명 관리
