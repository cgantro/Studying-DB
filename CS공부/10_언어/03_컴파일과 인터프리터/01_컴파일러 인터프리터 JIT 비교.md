# 컴파일러 인터프리터 JIT 비교

## 컴파일러
- 소스 코드를 사전에 기계어(또는 중간 산출물)로 변환한다.
- 장점: 실행 성능, 배포 단순성(런타임 의존 적음)
- 단점: 플랫폼별 빌드/배포 관리 필요

## 인터프리터
- 코드를 읽으면서 즉시 실행한다.
- 장점: 빠른 실험/디버깅
- 단점: 반복 경로 성능 한계

## JIT
- 실행 중 hotspot 코드를 동적으로 최적화한다.
- 장점: 실제 실행 패턴 기반 최적화 가능
- 단점: 워밍업 비용, 메모리 사용량 증가

## 실무 선택 포인트
- 짧은 배치: startup 비용 작은 방식 우선
- 장시간 서버: JIT 최적화 이점 큼
- 저지연 서비스: 워밍업/컴파일 스파이크까지 고려
