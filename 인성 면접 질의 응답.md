### 1. 1분 자기소개

저는 프로젝트를 진행하며 두 가지 역량을 길러왔습니다.  

첫 번째는 **병목을 분석하고 구조적으로 개선하는 역량**입니다. RobotPal 프로젝트에서는 카메라 스트리밍 과정에서 렌더링과 인코딩 처리가 맞물리며 프레임 드랍이 발생하는 문제를 확인했고, 이를 생산자-소비자 구조로 분리해 처리 흐름을 개선한 경험이 있습니다.

두 번째는 **팀이 같은 기준으로 일할 수 있게 만드는 협업 역량**입니다. 프로젝트를 진행하며 Jira 관리 기준, API 명세, 커밋·브랜치 네이밍 규칙 등을 정리해 팀원들이 같은 기준으로 개발할 수 있도록 했습니다. 최근에는 AI Agent를 활용할 때도 팀의 코드 컨벤션과 작업 규칙을 따를 수 있도록 AGENTS.md 문서화를 진행했습니다.

이러한 경험을 바탕으로 삼성SDS에서도 안정적이고 운영 가능한 소프트웨어를 만드는 데 기여하겠습니다..

### 2. 지원동기
키워드 : 실시간 데이터 처리, 상태 불일치 방지, 협업 기준 정리(특히 중요)
경험 : 팀 문서 협업 (좀 더 구체적으로)
제품군들을 보ㅇ

저는 프로젝트를 진행하며 실시간 데이터 처리, 상태 불일치 방지, 협업 기준 정리의 중요성을 경험했습니다. 특히 API 명세, Jira 기준, 커밋·브랜치 규칙을 정리하면서 협업의 핵심은 단순한 소통이 아니라, 팀이 같은 기준과 맥락으로 일할 수 있는 구조를 만드는 것이라고 느꼈습니다.

이후 삼성SDS의 Brity 제품군을 보며, 기업의 업무 흐름을 연결하고 반복 업무와 정보 불일치를 줄이는 시스템에 관심을 갖게 되었습니다. 저도 삼성SDS에서 기업 고객의 업무가 더 안정적이고 효율적으로 이어질 수 있도록, 협업과 업무 흐름을 시스템적으로 개선하는 개발자로 성장하고 싶습니다.

### 3. 장/단점
### 직무적 장점

**병목을 감으로 판단하지 않고, 구조와 흐름을 나눠서 분석하는 점**

> 제 직무적 장점은 문제를 감으로 해결하기보다, 데이터 흐름과 처리 구조를 나눠 원인을 찾는 점입니다.  
> RobotPal 프로젝트에서 카메라 스트리밍 중 프레임 드랍이 발생했을 때, 단순히 성능이 부족하다고 보기보다 렌더링, 인코딩, 큐 적재 흐름을 나눠 확인했습니다. 그 결과 렌더링과 인코딩이 같은 흐름에서 맞물리는 것이 병목이라는 점을 확인했고, 생산자-소비자 구조로 분리해 개선했습니다.  
> 저는 이런 식으로 문제를 구조적으로 쪼개고, 병목 지점을 찾아 개선하는 데 강점이 있습니다.

핵심 키워드:

- 병목 분석
- 데이터 흐름 분리
- 구조적 개선
- 측정 기반 개선

### 직무적 단점

**처음에는 구조와 기준을 먼저 잡으려 해서 실행이 늦어질 때가 있음**

> 제 직무적 단점은 처음 문제를 마주했을 때 구조와 기준을 먼저 명확히 잡으려는 성향이 있어, 초반 실행 속도가 늦어질 때가 있다는 점입니다.  
> 최근엔 AI AGENT를 활용하여 계획/
> 예전에는 전체 구조를 충분히 이해한 뒤 개발을 시작하려 했지만, 프로젝트에서는 빠르게 검증해야 하는 상황도 많았습니다. 그래서 최근에는 먼저 작은 단위로 구현하거나 테스트해보고, 이후 기준과 문서를 보완하는 방식으로 바꾸고 있습니다.  
> 예를 들어 팀 프로젝트에서도 처음부터 모든 규칙을 완성하려 하기보다, Git 브랜치 규칙, Jira 기준, API 명세처럼 충돌이 자주 나는 부분부터 우선 정리했습니다.

이 단점이 좋은 이유:

- 실제 단점처럼 들림
- 개발자 성향과 자연스럽게 연결됨
- 보완 방식이 구체적임
- SDS에서 “느린 사람”으로 보이지 않게 방어 가능

### 인성적 장점

**팀의 흐름을 보고 기준을 맞추는 점**

> 제 인성적 장점은 팀 안에서 각자의 일이 잘 연결되도록 흐름을 맞추는 점입니다.  
> 개발 프로젝트에서는 Jira, API 명세, 커밋·브랜치 네이밍 규칙을 정리해 팀원들이 같은 기준으로 일할 수 있도록 했습니다. 또한 밴드 활동에서도 각자 원하는 음악 방향이 달랐을 때, 각자의 장점을 살릴 수 있도록 곡의 방향과 역할을 조율한 경험이 있습니다.  
> 저는 앞에서 강하게 끌고 가기보다는, 팀원들이 같은 방향을 보고 움직일 수 있도록 기준을 정리하고 조율하는 편입니다.

핵심 키워드:

- 조율
- 기준 정리
- 팀 흐름 파악
- 협업 안정화

### 인성적 단점

**설명을 충분히 하려다 답변이 길어질 때가 있음**

> 제 인성적 단점은 상대가 충분히 이해하길 바라는 마음에 설명이 길어질 때가 있다는 점입니다.  
> 특히 프로젝트 경험이나 문제 해결 과정을 설명할 때 배경부터 자세히 말하려다 보니, 핵심이 늦게 전달되는 경우가 있었습니다.  
> 이를 보완하기 위해 최근에는 먼저 결론을 말하고, 이후에 근거를 덧붙이는 방식으로 말하려고 노력하고 있습니다. 예를 들어 면접 답변이나 팀 회의에서도 “문제는 무엇인지, 제가 한 일은 무엇인지, 결과가 무엇인지” 순서로 정리하려고 하고 있습니다.

이 단점이 좋은 이유:

- 실제 피드백과 맞음
- 면접장에서 바로 개선된 모습을 보여줄 수 있음
- 치명적이지 않음
- “커뮤니케이션 개선 노력”으로 연결 가능
