### 1. 1분 자기소개

저는 프로젝트를 진행하며 두 가지 역량을 길러왔습니다.  

첫 번째는 **병목을 분석하고 구조적으로 개선하는 역량**입니다. RobotPal 프로젝트에서는 카메라 스트리밍 과정에서 렌더링과 인코딩 처리가 맞물리며 프레임 드랍이 발생하는 문제를 확인했고, 이를 생산자-소비자 구조로 분리해 처리 흐름을 개선한 경험이 있습니다.

두 번째는 **팀이 같은 기준으로 일할 수 있게 만드는 협업 역량**입니다. 프로젝트를 진행하며 Jira 관리 기준, API 명세, 커밋·브랜치 네이밍 규칙 등을 정리해 팀원들이 같은 기준으로 개발할 수 있도록 했습니다. 최근에는 AI Agent를 활용할 때도 팀의 코드 컨벤션과 작업 규칙을 따를 수 있도록 AGENTS.md 문서화를 진행했습니다.

이러한 경험을 바탕으로 삼성SDS에서도 안정적이고 운영 가능한 소프트웨어를 만드는 데 기여하겠습니다..

### 2. 지원동기
키워드 : 실시간 데이터 처리, 상태 불일치 방지, 협업 기준 정리(특히 중요)
경험 : 팀 문서 협업 (좀 더 구체적으로)
제품군들을 보ㅇ

저는 프로젝트를 진행하며 실시간 데이터 처리, 상태 불일치 방지, 협업 기준 정리의 중요성을 경험했습니다. 특히 API 명세, Jira 기준, 커밋·브랜치 규칙을 정리하면서 협업의 핵심은 단순한 소통이 아니라, 팀이 같은 기준과 맥락으로 일할 수 있는 구조를 만드는 것이라고 느꼈습니다.

이후 삼성SDS의 Brity 제품군을 보며, 기업의 업무 흐름을 연결하고 반복 업무와 정보 불일치를 줄이는 시스템에 관심을 갖게 되었습니다. 저도 삼성SDS에서 기업 고객의 업무가 더 안정적이고 효율적으로 이어질 수 있도록, 협업과 업무 흐름을 시스템적으로 개선하는 개발자로 성장하고 싶습니다.
