# AGENTS.md - 프로젝트 하위 지침

적용 범위: `프로젝트/`

## 목적
- 모든 면접 답변의 근거가 되는 원본 DB를 유지한다.

## 필수 항목
1. 프로젝트 상태(진행중/미완성/완료)
2. 한 줄 정의
3. 내 역할(팀 역할과 분리)
4. 구조/흐름
5. 기술 선택 이유 + 대안 + 트레이드오프
6. 문제 -> 원인 -> 해결 -> 결과 -> 배운 점
7. CS 연결

## 수치 규칙
- 수치에는 측정 조건/시점/방법을 같이 기록한다.
- 없으면 수치 단정 금지 + `확인 필요` 표시

## 문체
- 과장 금지
- 완료되지 않은 항목은 계획/가설로 명시
