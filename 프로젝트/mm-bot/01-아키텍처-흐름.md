# mm-bot - 아키텍처/흐름

## 전체 구조(요약)
- Mattermost 채널 데이터 → (배치) OCR/임베딩 → pgvector 저장
- Mattermost Slash Command → (실시간) 의도 분류 → 검색/조회 → LLM → Mattermost 응답

```mermaid
sequenceDiagram
  participant MM as "Mattermost"
  participant N8N as "n8n(Webhook)"
  participant DB as "DB(pgvector)"
  participant LLM as "LLM"

  MM->>N8N: Slash Command POST
  N8N-->>MM: 즉시 응답(처리 중...)
  N8N->>N8N: 의도 분류(meal/domain_rag/general_chat)
  alt 최신 공지/실시간 필요
    N8N->>MM: 최근 메시지 조회(REST)
  else 일반 질의
    N8N->>DB: 벡터 검색(topK)
  end
  N8N->>LLM: context + question
  N8N->>MM: 답변 POST(/api/v4/posts)
```

## 설계 포인트
- “질의 경로”는 빠르게 끝내야 하므로 읽기 중심.
- “적재 경로”는 느려도 되지만, 중복 방지/체크포인트가 핵심.
