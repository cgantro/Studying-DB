# 스티커(STICKY) - 배포 스크립트 가드레일

## 목표
- “조용히 실패하는 배포”를 줄인다.

## 적용한 가드레일(요약)
- 필수 변수 누락 시 즉시 실패: `require_var`
- `${...}` 같은 미해결 플레이스홀더 감지 시 실패: `ensure_not_placeholder`
- staging DB reset은 강한 조건을 만족할 때만 허용:
  - staging 환경, 예상된 compose 프로젝트명, 명시적 ACK 값

## 대안
- CI 단계에서 `.env`/변수 스키마를 정적으로 검증(예: JSON schema, dotenv lint):
  - 장점: 배포 전에 더 빨리 잡힘
  - 단점: 운영 변수/마스킹 변수까지 포함한 설계가 필요
