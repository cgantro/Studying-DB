# 스티커(STICKY) - 헬스체크와 보안 경계

## 문제
- 배포 스크립트는 “배포 후 정상 기동”을 확인해야 한다.
- 그런데 인증이 걸린 엔드포인트로 healthcheck를 하면 배포가 불안정해진다.

## 해결
- `/actuator/health/**`는 `permitAll`로 열었다.
- 대신 나머지 사용자 자원(API)은 `authenticated`로 유지해서 보안 경계를 유지했다.

## 왜 이 선택이 맞았나
- 헬스체크는 외부 노출이 되더라도, 반환 내용이 최소(UP/DOWN)로 제한되면 위험이 상대적으로 낮다.
- 배포 안정성(기동 확인 실패로 인한 롤백/재배포 반복) 비용이 더 컸다.

## 대안
- 내부 네트워크 전용 health endpoint(보안그룹/프록시 레벨 차단):
  - 장점: 외부 노출 최소화
  - 단점: 배포/라우팅 설정이 더 복잡해짐
