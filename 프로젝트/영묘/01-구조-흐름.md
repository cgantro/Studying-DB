# 영묘 - 구조/흐름

## 서버 구성
- Lobby: WebSocket(uwebsockets) 기반
- Voice: UDP 기반 릴레이 서버

## Voice packet 포맷(요약)
- 고정 헤더 + 가변 payload
  - packetType: Join/VoiceData/Leave
  - roomCode: 16 bytes
  - speakerName: 32 bytes
  - payloadSize: uint32

## 데이터 흐름(요약)
```mermaid
sequenceDiagram
  participant UE as "Unreal Client"
  participant VS as "VoiceServer(UDP)"
  participant CM as "VoiceClientManager"

  UE->>VS: Join(roomCode,speakerName)
  VS->>CM: addClient/joinRoom
  UE->>VS: VoiceData(roomCode,speakerName,opus_payload)
  VS->>CM: getRoomClientsSnapshot(roomCode)
  VS->>VS: sender 제외하고 UDP relay
  VS->>UE: VoiceData(...)
  UE->>VS: Leave(roomCode,speakerName)
  VS->>CM: leaveRoom/removeClient
```

## RoomCode 스코프
- 클라 수신 메시지/이벤트는 RoomCode가 다르면 무시.
- 같은 맵을 여러 방에서 쓰더라도, RoomCode 기반 월드 오프셋을 적용해 “물리적 겹침”을 피한다.
