# 영묘 - RoomCode 기반 논리적 격리

## 문제
- “방” 개념이 없는 상태에서 멀티플레이 메시지/이벤트가 섞이면,
  - 다른 방의 UI 업데이트가 보이거나
  - 다른 방 이벤트가 내 게임플레이에 영향을 주는 문제가 생긴다.

## 해결: RoomCode를 스코프 키로 강제
### 1) 메시지 필터링(클라이언트)
- 수신 메시지에 roomCode가 있고, 현재 roomCode와 다르면 무시한다.

### 2) PlayerState/이벤트에도 RoomCode를 붙임
- PlayerState에 RoomCode를 복제(replication)하고, 이벤트는 instigator의 RoomCode를 들고 다닌다.

### 3) 월드 오프셋으로 물리적 격리
- RoomCode를 해시해서 슬롯을 정하고, 큰 X 오프셋(step=5,000,000)을 적용해 같은 맵을 “다른 위치”에 로드한다.
- 결과: 같은 레벨을 여러 방이 쓰더라도 시야/충돌이 겹치지 않게 된다.

## 대안
- 방마다 독립된 서버/월드 인스턴스:
  - 장점: 격리 강함
  - 단점: 리소스/운영 비용 증가, 인스턴스 관리 복잡
