# 영묘 (Unreal 멀티플레이/보이스)

## 한 줄 정의
WebSocket 기반 보이스를 UDP+Opus로 재설계하고, Unreal/외부 C++ 서버 경계를 정리한 멀티플레이 구조 개선 프로젝트.

## 내 역할
- 보이스 통신 구조 개편(WebSocket -> UDP)
- 코덱 체계 재구축(libOpus)
- 서버/클라이언트 경계 리팩터링

## 핵심 구조
- 외부 C++ 서버: 로비/채팅/보이스/게임시작 관리
- Unreal 서버: 인게임 authoritative gameplay
- 보이스: UDP 릴레이 + Opus 인코딩/디코딩

## 기술 선정 이유
- UDP: 최신성 우선, HoL blocking 회피
- Opus: 대역폭 절감 + 실시간 음성 품질
- 핸들러/라우터 구조: 책임 분리와 유지보수성 향상

## 개선 경험
### 1) WebSocket 지연/끊김
- 해결: UDP 릴레이 전환
- 효과: 음성 밀림 감소

### 2) Raw PCM 대역폭 과다
- 해결: Opus 코덱 전환
- 효과: 전송량 대폭 감소(요약 기준 약 10분의 1)

### 3) 버퍼/노이즈/하이톤 문제
- 해결: PCM 누적 버퍼, 주파수 기준 통일(16k)
- 효과: 음질/자연스러움 회복

## CS 연결
- 네트워크: UDP, 지터/로스, 백프레셔
- 운영체제: 이벤트루프, 스레드 모델
- 소프트웨어공학: 경계 설계, 모듈화, 운영 안정성
