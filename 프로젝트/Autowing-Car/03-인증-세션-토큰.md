# Autowing Car - 인증/세션/토큰

## 목표
- API는 Stateless(JWT)로 단순하게 유지한다.
- Refresh Token은 서버가 회수/만료를 통제한다.

## 구현 요약
- Access Token: JWT
- Refresh Token: Redis 저장(`RT:{userId}`), TTL 기반 만료

## 문제 -> 원인 -> 대응
### 로그인 비용(BCrypt)
- 문제: 동시 접속이 늘면 로그인 구간이 CPU 병목이 된다.
- 원인: BCrypt는 설계상 비용이 큰 해시(보안 비용).
- 대응:
  - 시연/MVP 조건에서는 strength를 낮춰 성능 실험이 가능하게 했다.
  - 운영 전환 시에는 “strength 상향 + 수평 확장 + rate limit”을 같이 묶어서 본다.

## 대안
- Argon2id:
  - 장점: 메모리 기반 비용 조절로 GPU 공격에 강함
  - 단점: 운영 환경/튜닝 경험이 필요
