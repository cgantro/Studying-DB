# Autowing Car - 맵 리소스 로딩(Docker/JAR)

## 문제
- 로컬에서는 되던 “맵 파일 로딩”이 컨테이너/JAR 배포에서 실패했다.

## 원인
- classpath 리소스는 일반 파일 경로가 아니다.
- 컨테이너에서는 마운트/권한/경로가 다르다.

## 해결(요약)
- classpath/file 패턴 기반으로 리소스 후보를 수집한다.
- 외부 맵(file)이 있으면 외부 맵을 우선한다(운영 교체 가능).
- 일부 메타데이터(MapInfo)는 파일 I/O에 강결합하면 취약해서, 실패 시 안전한 fallback 값을 둔다.

## 대안
- 맵 메타데이터/노드/엣지를 DB 마이그레이션으로 적재하고, 파일은 초기 적재 전용으로만 사용:
  - 장점: 배포 형태(Docker/JAR)에 덜 민감
  - 단점: 적재/버전관리/검증 파이프라인이 필요
